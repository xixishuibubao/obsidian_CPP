# 1. C/C++ 与 Python 互调用

C/C++ 负责性能与硬件访问，Python 负责编排与快速迭代。互调用本质是**跨语言边界传递数据**、共享进程地址空间或启动子进程。工程案例：[[C-Linux生态/07.Telescope1126实战/4. 驱动插件与pybind11动态加载|驱动插件与 pybind11 动态加载]]。

## 【核心精简提纲】

### 记忆主干

**Python→C/C++**：ctypes/cffi 或 pybind11 编译 `.so` 后 `import`。**C++→Python**：pybind11 embed 或 popen 子进程。交叉编译须 Python ABI、sysroot 与 [[C-Linux生态/01.Linux环境/3. rpath与链接路径|rpath]] 一致。

### 核心要点清单

- `PYBIND11_MODULE(foo, m)` → Python `import foo`（文件名 `foo*.so`）
- 交叉编译：Python 版本、ABI tag（`cpython-312`）、sysroot 头文件/库一致
- 部署：`sys.path`、[[C-Linux生态/01.Linux环境/3. rpath与链接路径|rpath]]、`LD_LIBRARY_PATH`
- ctypes：C 须 `extern "C"`；签名错 → segfault，try/except 捕不到
- pybind11：C++ 异常 → Python；长计算用 `py::gil_scoped_release`
- 编译链：`gcc -shared -fPIC` 或 `pybind11_add_module`（CMake）

### 方案速查

| 方案 | 方向 | 编译 | 本项目 |
|------|------|------|--------|
| ctypes | Py→C | 否 | FITS/libcfitsio |
| pybind11 扩展 | Py→C++ | 是 | **驱动层主选** |
| pybind11 embed | C++→Py | 是 | 备选宿主模式 |
| popen/subprocess | C→Py 脚本 | 否 | 一次性任务 |

## 1.1 两个方向

```plain
┌─────────────────────────────────────────────────────────────┐
│  Python 调 C/C++（Extension / FFI）                          │
│  ctypes / cffi  →  已有 .so，运行时加载                      │
│  pybind11 / C API  →  编译出 .so 扩展模块，import 加载       │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  C/C++ 调 Python（Embedding / 进程外）                       │
│  pybind11 embed   →  scoped_interpreter + import             │
│  popen / system   →  子进程跑 python 脚本                     │
└─────────────────────────────────────────────────────────────┘
```

| 方向 | 典型场景 | 开发成本 |
|------|----------|----------|
| Python → C/C++ | 硬件驱动、数值计算 | ctypes 低 · pybind11 中 |
| C/C++ → Python | 脚本配置、插件 | embed 中 · popen 低 |

## 1.2 方案总览

| 方案 | 编译 | 类型安全 | 适用 |
|------|------|----------|------|
| ctypes | 否（仅 C 库） | 弱 | 已有 `.so`、快速胶水 |
| cffi | ABI 否 / API 是 | 中 | 比 ctypes 易维护 |
| **pybind11** | 是 | 强 | **C++ 互调首选** |
| CPython C API | 是 | 手动 refcnt | 底层扩展 |
| popen/subprocess | 否 | 进程隔离 | 无共享内存 |

本项目：驱动 **pybind11 `.so` + importlib**；FITS 用 **ctypes 调 libcfitsio**，见 [[C-Linux生态/07.Telescope1126实战/9. FITS天文图像与后处理|FITS 天文图像与后处理]]。

## 1.3 Python 调用 C：ctypes

```python
from ctypes import CDLL, c_double, POINTER, byref

lib = CDLL("./libmath.so")
lib.add.argtypes = [c_double, c_double, POINTER(c_double)]
lib.add.restype = None
result = c_double()
lib.add(1.5, 2.5, byref(result))
```

C 侧：

```c
void add(double a, double b, double *out) { *out = a + b; }
```

编译（[[B-构建与脚本/01.构建工具/1. g++|g++]]）：

```bash
gcc -shared -fPIC math.c -o libmath.so
```

| 话题 | 说明 |
|------|------|
| 结构体 | `ctypes.Structure` 对齐须与 C 一致 |
| GIL | ctypes 调用期间持有 GIL |
| 错误 | 返回值检查；segfault 不可捕获 |

## 1.4 Python 调用 C++：pybind11

### 最小示例

`example.cpp`：

```cpp
#include <pybind11/pybind11.h>
namespace py = pybind11;
int add(int a, int b) { return a + b; }
PYBIND11_MODULE(example, m) {
    m.def("add", &add, "Add two integers");
}
```

`CMakeLists.txt`（[[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake]]）：

```cmake
cmake_minimum_required(VERSION 3.15)
project(pybind_example LANGUAGES CXX)
find_package(Python3 COMPONENTS Interpreter Development REQUIRED)
find_package(pybind11 CONFIG REQUIRED)
pybind11_add_module(example example.cpp)
```

```bash
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
python3 -c "import example; print(example.add(1, 2))"
```

### 绑定 C++ 类

```cpp
PYBIND11_MODULE(camera, m) {
    py::class_<Camera>(m, "Camera")
        .def(py::init<const std::string &>())
        .def("connect", &Camera::connect)
        .def("start_exposure", &Camera::start_exposure,
             py::arg("exposure_ms"), py::arg("raw_path"));
}
```

| 能力 | API |
|------|-----|
| 释放 GIL | `py::gil_scoped_release` |
| Python 回调 | `py::function`，注意生命周期 |

### 模块名与部署

- `PYBIND11_MODULE(foo, m)` → `import foo`；产物 `foo.cpython-312-*.so`
- 交叉编译：板端 Python 版本与 ABI 一致；`CMAKE_TOOLCHAIN_FILE` + `Python3_ROOT_DIR`
- 依赖 `.so`：[[C-Linux生态/01.Linux环境/3. rpath与链接路径|rpath]] 或 `LD_LIBRARY_PATH`

工程细节：[[C-Linux生态/07.Telescope1126实战/6. 交叉编译与DriverSO部署|交叉编译与 DriverSO 部署]]。

## 1.5 cffi（简述）

ABI 模式类似 ctypes；API 模式编译时生成扩展，类型检查更好。

## 1.6 C/C++ 调用 Python：Embedding

| 方式 | 特点 |
|------|------|
| pybind11 embed | RAII、`py::module_::import` |
| CPython C API | `Python.h`，冗长 |
| popen | 进程隔离，[[C-Linux生态/02.Linux系统编程/8. popen 与 system 的区别|popen 与 system]] |

### pybind11 embed 最小示例

`embed_main.cpp`：

```cpp
#include <pybind11/embed.h>
namespace py = pybind11;

int main() {
    py::scoped_interpreter guard{};
    py::module_::import("sys").attr("path").attr("append")(".");
    auto calc = py::module_::import("calc");
    int sum = calc.attr("add")(1, 2).cast<int>();
    return 0;
}
```

CMake：`target_link_libraries(embed_demo PRIVATE pybind11::embed)`。

### 扩展 vs 内嵌模块

| 宏 | 加载方式 |
|----|----------|
| `PYBIND11_MODULE` | 独立 `.so`，`import` |
| `PYBIND11_EMBEDDED_MODULE` | 链入可执行文件 |

Telescope1126 采用**扩展模块**（Python 主服务 + `import` `.so`）。

### GIL

非 Python 线程调 Python API：`py::gil_scoped_acquire`。长 C++ 计算：`py::gil_scoped_release`。

## 1.7 数据与 ABI

| 模式 | 内存归属 |
|------|----------|
| ctypes buffer | Python 分配，C 只写 |
| C `malloc` 返回 | 需提供 `free` |
| pybind11 string/bytes | 自动拷贝 |

异常：pybind11 映射 C++ 异常；ctypes segfault 不可捕获。

## 1.8 选型决策

```plain
Python 调 C/C++？
  ├── 已有 C .so、API 少 → ctypes/cffi
  ├── C++ 类/STL → pybind11（本项目驱动）
  └── 海量 API → SWIG

C/C++ 调 Python？
  ├── 偶尔脚本 → popen
  ├── C++ 主程序频繁互调 → pybind11 embed
  └── 纯 C → Python.h
```
