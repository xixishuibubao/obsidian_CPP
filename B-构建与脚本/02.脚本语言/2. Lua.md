# 1. Lua 脚本语言

Lua 是轻量 **嵌入式脚本语言**，单文件解释器、C API 成熟，常用于游戏逻辑、Nginx OpenResty、Redis 脚本、配置 DSL。本知识库主栈以 Python/Shell 为主；Lua 作扩展阅读。

## 【核心精简提纲】

### 记忆主干

**单线程 VM + table 一统数组/字典 + `local` 控作用域**；宿主通过 `lua_State` 与 C API 互调；体积极小，适合嵌入，生态弱于 Python。

### 核心要点清单

- 变量默认全局；脚本内用 `local` 避免污染
- `table`：`t[1]` 数组 + `t.key` 字典；`ipairs` / `pairs` 迭代
- C 嵌入：`luaL_newstate` → `luaL_openlibs` → `luaL_dostring` / `lua_pcall`
- 链接 `-llua`（`liblua-dev`）
- 本项目：Telescope1126 用 **Python + pybind11**，未用 Lua

### 选型速查

| 维度 | Lua | Python |
|------|-----|--------|
| 体积 | 极小 | 较大 |
| 嵌入 | C API 一等公民 | C API 重（CPython） |
| 生态 | 游戏/网关/Redis | 科学计算/Web/AI |
| 本 vault | 扩展阅读 | [[B-构建与脚本/02.脚本语言/3. Python|主栈]] |

## 【完整拓展详情】

## 1.1 常用场景

| 场景 | 说明 |
|------|------|
| **嵌入 C/C++** | 通过 `lua.h` / `lauxlib.h` 在宿主程序内跑脚本 |
| **OpenResty / Nginx** | 高性能网关逻辑，JIT（LuaJIT）场景 |
| **游戏 / 工具链** | 热更新逻辑、Wireshark 插件 |
| **Redis** | `EVAL` / `EVALSHA` 原子脚本 |

### 适用边界

| 适合 | 不适合 |
|------|--------|
| 固件/游戏内几 KB 级脚本 VM | 需要 NumPy/pandas 等科学栈 |
| 配置 DSL、规则引擎 | 团队已统一 Python 且体积不敏感 |
| Nginx 请求级逻辑 | 复杂 AI/ML 管线 |

**本项目**：Telescope1126 以 **Python + pybind11** 为脚本层；若仅需极简 glue 且镜像体积敏感，可评估 Lua embed 替代部分 Python。

## 1.2 基础语法

```lua
-- 变量默认全局；local 局部
local n = 42
local s = "hello"

-- table = 数组 + 字典（1-based 数组下标惯例）
local t = { 1, 2, name = "cam" }
print(t[1], t.name)

local function add(a, b)
    return a + b
end

for i, v in ipairs(t) do
    print(i, v)
end

for k, v in pairs(t) do
    print(k, v)
end
```

### 易混概念

- **Lua table 不是 JavaScript 对象**：元表（metatable）才提供运算符重载与 OOP 惯用法
- **数组从 1 开始**：`ipairs` 遇 `nil` 停止；稀疏数组行为需小心
- **只有 `false` 与 `nil` 为假**；`0` 与 `""` 均为真

### 典型错误

- 忘记 `local` → 全局污染，并发/重入时互相覆盖
- 用 `#t` 测长度：仅对连续序列可靠
- 字符串拼接在循环中用 `..` → 性能差；大量拼接用 `table.concat`

## 1.3 C 嵌入最小示例

```c
#include <lua.h>
#include <lauxlib.h>
#include <lualib.h>

int main(void) {
    lua_State *L = luaL_newstate();
    luaL_openlibs(L);
    if (luaL_dostring(L, "print('hello from lua')") != LUA_OK) {
        const char *err = lua_tostring(L, -1);
        fprintf(stderr, "lua error: %s\n", err);
        lua_close(L);
        return 1;
    }
    lua_close(L);
    return 0;
}
```

链接：`-llua`（需安装 `liblua-dev` 或自编译静态库）。

### 扩展：注册 C 函数给 Lua

```c
static int l_add(lua_State *L) {
    int a = luaL_checkinteger(L, 1);
    int b = luaL_checkinteger(L, 2);
    lua_pushinteger(L, a + b);
    return 1;  /* 返回值个数 */
}

lua_register(L, "add", l_add);
/* Lua: print(add(1, 2)) */
```

栈模型是 C 嵌入核心：入参/返回值均通过 `lua_push*` / `lua_to*` 操作栈顶。

## 1.4 与 Python 选型（展开）

| 维度 | Lua | Python |
|------|-----|--------|
| 运行时体积 | ~200KB 级（可裁库） | CPython + 标准库 MB 级 |
| 启动速度 | 快 | 相对慢 |
| GIL | 单线程 VM，无 GIL 概念 | CPython GIL |
| 与本项目 | 未采用 | Tornado · pybind11 · 见 [[B-构建与脚本/02.脚本语言/3. Python|Python 笔记]] |
| 互操作 | C API 直接 | pybind11 / ctypes |

OpenResty 选 Lua 因 **请求级隔离 + 极低延迟**；Telescope1126 选 Python 因 **驱动生态、Tornado、团队栈**。
