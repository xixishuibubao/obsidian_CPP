# 1. 进阶特性

CMake 3.0+ 以 `target_*` 命令为核心，精准控制每个可执行文件/库的属性。本篇覆盖复杂项目组织、依赖管理与交叉编译集成；入门见 [[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake 入门]]。

## 【核心精简提纲】

### 记忆主干

**目标级配置 + 依赖可见性**：`target_include_directories` / `target_link_libraries` / `target_compile_options` 替代全局命令；`PRIVATE` 内部用、`PUBLIC` 向上传递、`INTERFACE` 仅头文件库。复杂项目按模块 `add_subdirectory`，外部依赖用 `find_package` / `FetchContent`。

### 核心要点清单

- `target_compile_options(app PRIVATE -Wall)` 传递 [[B-构建与脚本/01.构建工具/1. g++|g++ 选项]]
- `CMAKE_EXPORT_COMPILE_COMMANDS ON` → `compile_commands.json`（IDE/clangd）
- `option(BUILD_TEST ...)` 控制可选子目录
- 对象库 `OBJECT` 减少中间 `.a` 产物
- 交叉编译：`CMAKE_TOOLCHAIN_FILE` + `CMAKE_SYSROOT`
- pybind11 多目标：`pybind11_add_module` + `target_link_libraries` PRIVATE 依赖链

## 【完整拓展详情】

## 1.1 目标属性（`target_*` 系列）

| 命令 | 作用 |
|------|------|
| `target_include_directories` | 头文件目录 |
| `target_link_libraries` | 链接库 |
| `target_compile_options` | 编译选项（`-Wall`、`-march=…`） |
| `target_compile_definitions` | 宏（`DEBUG=1`） |

```cmake
target_compile_options(math_app PRIVATE -Wall -Werror)
target_compile_definitions(math_app PRIVATE DEBUG=1)
```

### 可见性关键字

| 关键字 | 含义 | 场景 |
|--------|------|------|
| `PRIVATE` | 仅当前目标 | 内部子模块 |
| `PUBLIC` | 当前 + 传递给依赖方 | 对外 API 库 |
| `INTERFACE` | 仅传递给依赖方 | header-only 库 |

elseif(UNIX)

    # GCC 特有

## 1.3 外部依赖（`find_package`）

```cmake
find_package(Boost 1.70 REQUIRED COMPONENTS program_options)

```

## 1.5 FetchContent

```cmake
include(FetchContent)
FetchContent_Declare(fmt GIT_REPOSITORY https://github.com/fmtlib/fmt.git GIT_TAG 10.1.1)
FetchContent_MakeAvailable(fmt)
target_link_libraries(my_app PRIVATE fmt::fmt)
```

## 1.6 复杂项目目录规范

```plain
project_root/
├── CMakeLists.txt
├── cmake/              # FindXXX.cmake
├── src/
│   ├── core/           # 静态库
│   ├── utils/
│   └── app/
├── include/
├── 3rdparty/
└── test/
```

**设计原则**：

1. 模块化：`add_subdirectory` 分模块
2. 目标级 `target_*`，避免全局 `include_directories`
3. `option` 开关（`BUILD_TEST`）
4. `find_package` / `FetchContent` 管理第三方

**根模板片段**：

```cmake
cmake_minimum_required(VERSION 3.15)
project(ComplexProject LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_EXPORT_COMPILE_COMMANDS ON)
option(BUILD_TEST "Build test" ON)
add_subdirectory(src)
if(BUILD_TEST)
    enable_testing()
    add_subdirectory(test)
endif()
```

## 1.7 库内多层子目录（依赖隔离）

内部模块 `PRIVATE` 头文件与链接，对外库 `PUBLIC` 仅暴露接口头：

```cmake
# network 对外主库
add_library(network STATIC network.cpp)
target_include_directories(network
    PUBLIC  ${CMAKE_CURRENT_SOURCE_DIR}
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/../core
)
target_link_libraries(network PRIVATE network_core network_protocol)
```

上层 app 只需：

```cmake
target_link_libraries(network_app PRIVATE network)
```

### 对象库优化

```cmake
add_library(network_core OBJECT buffer.cpp)
add_library(network STATIC network.cpp $<TARGET_OBJECTS:network_core>)
```

减少中间静态库文件。

## 1.8 交叉编译集成

toolchain 文件示例：

```cmake
set(CMAKE_SYSTEM_NAME Linux)
set(CMAKE_C_COMPILER aarch64-linux-gnu-gcc)
set(CMAKE_CXX_COMPILER aarch64-linux-gnu-g++)
set(CMAKE_SYSROOT /opt/sysroot)
set(CMAKE_FIND_ROOT_PATH ${CMAKE_SYSROOT})
set(CMAKE_FIND_ROOT_PATH_MODE_PROGRAM NEVER)
set(CMAKE_FIND_ROOT_PATH_MODE_LIBRARY ONLY)
set(CMAKE_FIND_ROOT_PATH_MODE_INCLUDE ONLY)
```

- `CMAKE_SYSROOT` 对应 [[B-构建与脚本/01.构建工具/1. g++|g++ --sysroot]]
- 板端 Python 头文件/库须在 sysroot 或 `Python3_ROOT_DIR` 指向板端 staging

详见 [[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|交叉编译优化选项]]、[[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|交叉编译部署]]。
