# 1. Makefile

GNU Make 用「目标 : 依赖 + Tab 命令」描述文件依赖与增量编译。适合中小项目、内核/服务器风格构建；大型跨平台项目更常用 [[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake]] 生成 Makefile。

## 【核心精简提纲】

### 记忆主干

**依赖驱动增量构建**：Make 比较时间戳，变更的依赖触发命令；**Tab 缩进**是语法要求。变量 `$(CC)`、自动变量 `$@`/`$^`、模式规则 `%.o: %.c` 是工业 Makefile 骨架。

### 核心要点清单

- 规则：`target : 依赖` + Tab + 命令（Tab 不可换成空格）
- 流程：检查依赖时间戳 → 过期则执行命令 → 链接成 target
- 伪目标：`.PHONY: clean` 不检查文件时间戳
- 变量：`CC := gcc`，使用 `$(CC)`；`CFLAGS` / `CXXFLAGS` 传选项给 [[B-构建与脚本/01.构建工具/1. g++|g++]]
- 函数：`$(wildcard *.c)`、`$(patsubst %.c,%.o,$(Srcs))`
- 进阶：模式规则、自动依赖 `.d`、并行 `-j` → [[B-构建与脚本/01.构建工具/5. 编译脚本设计|编译脚本设计]]

## 【完整拓展详情】

## 1.1 语法格式

```makefile
target : 依赖1 依赖2
	gcc 源文件 -c 目标.o [附加选项]    # 行首必须是 Tab
```

- `target` 可无依赖或命令（仅作占位）
- 命令行实际是调用 [[B-构建与脚本/01.构建工具/1. g++|gcc/g++]]，选项写在 `CFLAGS`/`CXXFLAGS` 或命令行

## 1.2 执行流程

1. 自顶向下解析目标依赖图
2. 比较 target 与各依赖的时间戳
3. 依赖更新或不存在 → 执行对应命令
4. 全部依赖就绪后链接生成 target

## 1.3 伪目标 `.PHONY`

不对应磁盘文件，每次执行命令：

```makefile
.PHONY: clean rebuild
clean:
	rm -f main main.o
rebuild: clean main
```

## 1.4 常见变量

赋值：`val := 内容`，引用 `$(val)`。

**保留变量**：

| 变量 | 默认 | 用途 |
|------|------|------|
| `CC` | `cc` | C 编译器 |
| `CXX` | `g++` | C++ 编译器 |
| `CFLAGS` / `CXXFLAGS` | 空 | 编译选项 |
| `LDFLAGS` | 空 | 链接选项（rpath 等） |
| `AR` | `ar` | 静态库打包 |

**自动变量**（无需 `$(...)` 包裹变量名本身）：

| 符号 | 含义 |
|------|------|
| `$@` | 当前 target |
| `$^` | 全部依赖 |
| `$<` | 第一个依赖 |
| `$?` | 比 target 新的依赖 |

**函数**：

```makefile
Srcs := $(wildcard *.c)
Objs := $(patsubst %.c, %.o, $(Srcs))
```

模式规则中 `%` 类似通配符 `*`。

## 1.5 与 CMake 的关系

CMake **生成** Makefile（Linux 默认），不替代 Make 的增量逻辑。复杂项目用 [[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake 入门]] + [[B-构建与脚本/01.构建工具/4. CMake进阶与复杂项目|CMake 进阶]] 维护 `CMakeLists.txt`，本地仍执行 `cmake .. && make -j`。

## 1.6 交叉编译 Makefile 片段

```makefile
CROSS_COMPILE ?= aarch64-linux-gnu-
CC      := $(CROSS_COMPILE)gcc
CXX     := $(CROSS_COMPILE)g++
SYSROOT := /opt/sysroot
CFLAGS  += --sysroot=$(SYSROOT)
```

详见 [[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|交叉编译优化选项]]。pybind11 模块 Makefile 通常交给 CMake `pybind11_add_module`，见 [[3. C-C++与Python互调用|互调用笔记]]。
