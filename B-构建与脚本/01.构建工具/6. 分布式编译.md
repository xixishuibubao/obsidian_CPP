# 1. 分布式编译

Ubuntu 双机 x86_64（9950X + 12900H）上用 **distcc** / **Icecream（icecc）** 把 C/C++ **compile** 分发到远程 helper，加速 **native** 与 **x86→ARM/ARM64 交叉** 构建。链接仍在主编译机本地；**工具链与 sysroot 路径一致** 依赖 [[C-Linux生态/01.Linux环境/8. NFS 网络文件系统|NFS]]。构建选项见 [[B-构建与脚本/01.构建工具/1. g++|g++]]；交叉 `-march` 见 [[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|交叉编译优化选项]]。

## 【核心精简提纲】

### 记忆主干

**分布式 = 远程 `-c`，本地 link**。CMake 用 `CMAKE_*_COMPILER_LAUNCHER=distcc`（compiler 仍是 gcc/交叉 gcc）；Make 用 `CC="distcc gcc"`。**交叉编译**：编译器与 sysroot 在 NFS 同路径（如 `/srv/distcc`）。禁止 `-march=native`（9950X vs 12900H 微架构不同）。

### 核心要点清单

- 前置：[[C-Linux生态/01.Linux环境/8. NFS 网络文件系统|NFS]] 部署完成，§1.2 checklist 通过
- distcc：`DISTCC_HOSTS="localhost/16 192.168.1.11/20"`，`-j` ≈ slot 总和（约 36~48）
- icecc：一台跑 `icecc-scheduler`，各节点 `iceccd` + `ICECREAM_SCHEDULER_HOST`
- native：`CMAKE_C_COMPILER_LAUNCHER=distcc`；交叉：toolchain 路径指向 NFS + 同上 launcher
- 监控：`distccmon-text 1`；叠加缓存：`CCACHE_PREFIX=distcc`
- 不加速：link、硬编码 `/usr/bin/gcc` 的规则、LTO 全阶段（建议关闭或仅 link 开 `-flto`）

### 构建方式速查表

| 方式 | distcc native | distcc 交叉 |
|------|---------------|-------------|
| gcc/clang 直编 | `distcc gcc -c ...` | `distcc aarch64-linux-gnu-gcc -c ...`（NFS 路径） |
| Make | `export CC="distcc gcc"` | `CC="distcc ${CROSS_COMPILE}gcc"` |
| CMake/Ninja | `CMAKE_*_COMPILER_LAUNCHER=distcc` | `TOOLCHAIN_FILE`（NFS 路径）+ launcher |

### 构建链定位

```plain
NFS 同路径（[[C-Linux生态/01.Linux环境/8. NFS 网络文件系统|NFS]]）→ distcc/icecc launcher → gcc/Make/CMake → .o（远程）→ link（本地）
```

## 【完整拓展详情】

## 1.1 原理与边界

### 分发什么

| 阶段 | 是否分发 | 说明 |
|------|----------|------|
| 预处理 + 编译 + 汇编（→ `.o`） | **是** | distcc/icecc 核心 |
| 链接（→ 可执行文件 / `.so`） | **否** | 需要全部 `.o`，在主编译机本地 |
| `ar` / `ranlib` | 一般否 | 视构建系统 |
| 代码生成（protobuf、bison 等） | 否 | 仍在本地 |

### launcher vs compiler（CMake 最易踩坑）

| 角色 | 正确 | 错误 |
|------|------|------|
| **compiler** | `gcc`、`clang`、`/srv/distcc/toolchain/.../aarch64-linux-gnu-gcc` | 把 `distcc` 设为 compiler |
| **launcher** | `distcc` 或 `icecc` | 仅设 `CC=distcc gcc` 传给 **CMake**（应拆成 compiler + launcher） |

Make / 直调 gcc 习惯 **`CC="distcc gcc"`**（一体 wrapper）；CMake 3 推荐 **launcher 属性**，二者不要混用语义。

### 不覆盖

Bazel 远程执行、Kubernetes 构建农场、Golang、ROS/colcon。内核 `Kbuild` 可用 distcc 但规则特殊，见 [[C-Linux生态/03.Linux开发/1. 讯为RK3588|RK3588 内核模块]]，本篇不展开。

## 1.2 环境前提与 NFS 联用 checklist

### 通用前提

- 各节点 **GCC/G++ 主版本一致**（native 时）；Ubuntu 同 major 为宜
- **禁止** `-march=native`（9950X Zen5 vs 12900H Alder Lake）——见 [[B-构建与脚本/01.构建工具/1. g++|g++ -march 规则]]
- `DISTCC_HOSTS` 保留 **localhost slot**，远程不可达时 fallback
- 远程节点需能 `#include` 到与主编译机 **相同绝对路径** 的头文件 → **NFS**

### 交叉编译（NFS 为核心）

- helper 仍是 **x86_64**，执行 **交叉 gcc**（非本机 `/usr/bin/gcc`）
- **工具链 + sysroot** 位于 `/srv/distcc/...`，各节点 NFS 挂载同路径
- `CMAKE_TOOLCHAIN_FILE` 内路径 **必须** 使用 NFS 挂载点
- **icecc-create-env**：可选，从 NFS 上 g++ 打 env；**不替代** NFS 上的 sysroot 单一来源

### 联用 checklist（NFS 细节见专文）

- [ ] 各节点 `mount | grep /srv/distcc`
- [ ] helper：`test -x /srv/distcc/toolchain/.../g++`（或 apt 交叉 gcc 在 PATH 且版本一致）
- [ ] helper：`ls /srv/distcc/sysroot/usr/include`（或项目实际 sysroot 路径）
- [ ] `distccd`/`iceccd` 运行；防火墙 **3632**（distcc/icecc）、**2049**（NFS）内网通
- [ ] `DISTCC_HOSTS` 或 `ICECREAM_SCHEDULER_HOST` 已 export

## 1.3 distcc：native 双机部署

### 安装

```bash
sudo apt install distcc
```

### distccd（各节点）

Ubuntu 常用 `/etc/default/distcc`：

```bash
STARTDISTCC="true"
ALLOWEDNETS="192.168.1.0/24"
ZEROCONF="false"
# 并发 slot（9950X 约 28~32，12900H 约 16~20，留核给系统）
JOBS="32"
```

```bash
sudo systemctl enable --now distcc
ss -lntp | grep 3632
```

### 客户端

```bash
export DISTCC_HOSTS="localhost/16 192.168.1.11/20"
# 格式：host/slots，空格分隔；localhost 必留
distcc --show-hosts
```

**`-j` 建议**：≈ 各节点 `JOBS` 之和（双机约 **36~48**），见 §1.9。

### 验证

```bash
# 终端 1
distccmon-text 1

# 终端 2
export CC="distcc gcc"
distcc gcc -c -O2 foo.c -o foo.o
# distccmon 应显示 remote 计数增加
```

## 1.4 distcc：x86 → ARM/ARM64

### 工具链来源（均经 NFS 同路径）

| 类型 | 做法 |
|------|------|
| Ubuntu apt | Server 安装 `gcc-aarch64-linux-gnu`，同步到 `/srv/distcc` 或各节点同版本 apt + NFS sysroot |
| Rockchip SDK | 工具链在 `/srv/distcc/toolchain/...`，见 [[C-Linux生态/03.Linux开发/2. 交叉编译优化选项]] |
| masquerade | `DISTCC_DIR` 下为交叉 triple 建链，distccd 允许该编译器名 |

### 命令行 / Make

```bash
export DISTCC_HOSTS="localhost/16 192.168.1.11/20"
export CROSS_COMPILE=aarch64-linux-gnu-
# 或 NFS 绝对路径：
# export CC="distcc /srv/distcc/toolchain/aarch64-rockchip1240-linux-gnu/bin/aarch64-rockchip1240-linux-gnu-gcc"
export CC="distcc ${CROSS_COMPILE}gcc"
export CXX="distcc ${CROSS_COMPILE}g++"
make -j36 ARCH=arm64
```

### CMake 交叉

```bash
cmake -B build -G Ninja \
  -DCMAKE_TOOLCHAIN_FILE=/path/to/aarch64.toolchain.cmake \
  -DCMAKE_C_COMPILER_LAUNCHER=distcc \
  -DCMAKE_CXX_COMPILER_LAUNCHER=distcc
```

toolchain 内：

```cmake
set(CMAKE_SYSROOT /srv/distcc/sysroot)
set(CMAKE_C_COMPILER /srv/distcc/toolchain/aarch64-linux-gnu/bin/aarch64-linux-gnu-gcc)
# 不要写 distcc；launcher 由命令行注入
```

### distcc-pump（可选）

大工程且预处理依赖 sysroot 大量头文件时，**pump** 将 include 路径一并分发，配置较重；NFS 已挂载 sysroot 时 **多数项目不必 pump**。

### 验证交叉产物

```bash
file build/**/*.o | head
# 应含 ARM aarch64，而非 x86-64
```

## 1.5 Icecream：native 双机部署与扩展

### 架构

- **icecc-scheduler**：一台（如 9950X `192.168.1.10`）
- **iceccd**：各节点，连同一 scheduler

```bash
sudo apt install icecc
```

`/etc/default/icecc`（或 icecc 包等效配置，路径因版本略异）：

```bash
ICECREAM_SCHEDULER_HOST="192.168.1.10"
ICECREAM_SCHEDULER_PORT="8765"
ICECREAM_MAX_JOBS="32"
START_ICECC="true"
START_ICECC_SCHEDULER="false"   # 仅 scheduler 节点设 true
```

scheduler 节点：

```bash
# START_ICECC_SCHEDULER="true"
sudo systemctl enable --now icecc-scheduler iceccd
```

### 使用

```bash
export ICECC_SCHEDULER_HOST=192.168.1.10
cmake -B build -DCMAKE_C_COMPILER_LAUNCHER=icecc -DCMAKE_CXX_COMPILER_LAUNCHER=icecc
cmake --build build -j36

# 或不改 CMake，外包一层：
icecc --build make -j36
```

### 扩展第三台

新机器：装 `icecc` → 指向同一 `ICECREAM_SCHEDULER_HOST` → NFS 挂载 `/srv/distcc` → 增加 `DISTCC_HOSTS` 或 icecc 自动调度。

## 1.6 Icecream 与 NFS（create-env 可选）

NFS 已提供持久同路径 SDK 时，icecc 可直接用 NFS 上的编译器；可选 **create-env** 加速 icecc 内部分发：

```bash
/usr/lib/icecc/icecc-create-env \
  /srv/distcc/toolchain/aarch64-rockchip1240-linux-gnu/bin/aarch64-rockchip1240-linux-gnu-g++ \
  --sysroot /srv/distcc/sysroot
```

SDK 升级：更新 NFS 内容 → 视情况重建 env。sysroot 仍以 NFS 为准，避免双轨维护。

与 `build_all.sh`：在 [[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|交叉编译优化选项]] 流程外加 `COMPILER_LAUNCHER=icecc` 或 `icecc --build ./build_all.sh`。

export CCACHE_PREFIX=distcc

## 1.8 构建方式接入矩阵

### 1.8.1 直调 gcc / clang

```bash
distcc gcc -c -O2 -o foo.o foo.c
distcc clang++ -c -std=c++17 -o bar.o bar.cpp
```

多文件临时并行仍建议 Make/CMake；若脚本：

```bash
find src -name '*.cpp' -print0 | xargs -0 -I{} -P 32 \
  distcc g++ -c -O2 -o {}.o {}
```

| 维度 | 直调 gcc/clang | Make | CMake |
|------|----------------|------|-------|
| 并行 | 手动 `xargs -P` | `make -j` | Ninja + `-j` |
| distcc 接入 | 命令前缀 | `CC`/`CXX` | `COMPILER_LAUNCHER` |
| 改 distcc | 即时 | 即时 | **重新 cmake** |
| 交叉 | NFS 上 triple 绝对路径 | `CROSS_COMPILE` | `TOOLCHAIN_FILE` |

**clang / LTO**：远程须同版本 `clang`；**`-flto` 与分布式常冲突**——建议分布式阶段关 LTO，或仅在 **link 行** 加 `-flto`。

### 1.8.2 Makefile

见 §1.3、§1.4。注意：

- 规则内 **硬编码** `/usr/bin/gcc` 不会被 distcc 拦截 → 改为 `$(CC)`
- `export CC CXX` 供递归 Make 继承
- 第三方上游 Makefile：`icecc --build make -j36` 零修改

### 1.8.3 CMake / Ninja

native 配置：

```bash
cmake -B build -G Ninja \
  -DCMAKE_C_COMPILER=gcc \
  -DCMAKE_CXX_COMPILER=g++ \
  -DCMAKE_C_COMPILER_LAUNCHER=distcc \
  -DCMAKE_CXX_COMPILER_LAUNCHER=distcc
cmake --build build -j36
```

或：

```bash
export CMAKE_BUILD_PARALLEL_LEVEL=36
ninja -C build
```

| 问题 | 处理 |
|------|------|
| 改 launcher 无效 | 删 `build/` 或 `cmake --fresh` |
| `try_compile` 失败 | 配置阶段暂关 launcher，或 `-DCMAKE_TRY_COMPILE_TARGET_TYPE=STATIC_LIBRARY` |
| Unity build | 单 TU 过大，分发粒度粗；可 `-DCMAKE_UNITY_BUILD=OFF` |
| `ExternalProject` | 子项目单独传 launcher |

详见 [[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake 入门]]、[[B-构建与脚本/01.构建工具/4. CMake进阶与复杂项目|CMake 进阶]]。

### 1.8.4 总览表

| | gcc/clang | Make | CMake/Ninja |
|--|-----------|------|-------------|
| distcc native | 前缀 | `export CC/CXX` | `COMPILER_LAUNCHER` |
| distcc 交叉 | NFS triple | `CROSS_COMPILE+distcc` | TOOLCHAIN+launcher |
| icecc | `icecc --build` | `icecc --build make` | launcher=icecc |
| 重新配置 | 否 | 否 | **是** |
| 主要风险 | 手动并行 | 硬编码 CC | try_compile / LTO |

## 1.9 9950X + 12900H 参数建议

| 节点 | 逻辑线程 | distccd `JOBS` / slot | 备注 |
|------|----------|------------------------|------|
| 9950X | 32 | 28~32 | 常开 NFS Server + scheduler |
| 12900H | 20 | 16~20 | 笔记本注意散热降频 |

- **`make -j` / `CMAKE_BUILD_PARALLEL_LEVEL`**：≈ 两机 slot 之和，**36~48** 起调，负载高再略增
- **`DISTCC_HOSTS`** 示例：`localhost/16 192.168.1.10/28 192.168.1.11/18`（三台时）
- 12900H 外出：仅从 `DISTCC_HOSTS` 去掉对端或关 distcc，本地 `-j` 降为本地核数

## 1.10 故障排查

| 现象 | 方向 |
|------|------|
| `connection refused` | distccd 未启、防火墙 3632、ALLOWEDNETS |
| `preprocessor failed` / 缺头文件 | **helper 上**查 NFS 路径；非只看主编译机 |
| wrong architecture `.o` | 误用 native gcc；查 `CROSS_COMPILE` / toolchain |
| 全在 localhost | `DISTCC_HOSTS` 空、slot 0、网络不通 |
| icecc 全本地 | scheduler 地址错、iceccd 未连 |
| 版本 / ABI 怪错 | gcc 主版本不一致；Ubuntu 差 major |
| NFS + distcc 同时问题 | 先 §1.2 checklist，再 distccmon |

## 1.11 distcc vs Icecream 选型

| 维度 | distcc | Icecream |
|------|--------|----------|
| 上手 | 低 | 中 |
| 双机 | 够用 | 够用 |
| 多机调度 | 轮询 | scheduler，扩展顺 |
| CMake | launcher | launcher |
| x86→ARM | NFS + distcc；可选 pump | NFS + icecc；create-env 可选 |
| 推荐 | **2 机快速试** | **计划加第三台+** |

**路径一致**：一律 [[C-Linux生态/01.Linux环境/8. NFS 网络文件系统|NFS]]；本篇 checklist，NFS 篇部署。
