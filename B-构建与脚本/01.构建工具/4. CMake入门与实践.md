# 1. CMake 核心认知

## 1.1 基本概念

- **CMake 是什么**：不是直接编译代码，而是生成跨平台构建文件（如 Linux 的 Makefile、Windows 的 VS 工程、Mac 的 Xcode 工程）的工具。
- **核心优势**：跨平台、模块化、可扩展，适配从单文件小程序到大型分布式 C++ 项目的构建需求。
- **核心文件**：
  - `CMakeLists.txt` — CMake 的配置文件，描述项目结构、编译规则、依赖等。
  - `CMakeCache.txt` — 缓存文件，存储编译配置（编译器、编译选项），首次配置后生成。
  - `build` 目录 — 建议的构建目录（out-of-source build），分离源码和构建产物，避免污染源码。

## 1.2 环境准备

```bash
# Linux
sudo apt install cmake       # Debian/Ubuntu
sudo yum install cmake       # CentOS/RHEL

# Windows
# 官网下载安装包 https://cmake.org/download/，配置环境变量

# Mac
brew install cmake

# 验证（建议 3.15+）
cmake --version
```

# 2. 快速入门：单文件到多文件项目

## 2.1 单文件 C++ 项目

### 2.1.1 项目结构

```
hello/
├── CMakeLists.txt
└── main.cpp
```

### 2.1.2 main.cpp

```cpp
#include <iostream>
int main() {
    std::cout << "Hello CMake!" << std::endl;
    return 0;
}
```

### 2.1.3 CMakeLists.txt 核心模板

```cmake
# 指定 CMake 最低版本（避免语法兼容问题）
cmake_minimum_required(VERSION 3.15)

# 定义项目名称和语言（PROJECT_NAME 为全局变量）
project(HelloCMake LANGUAGES CXX)

# 设置 C++ 标准（跨平台统一标准，避免编译器差异）
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)   # 强制要求，不降级
set(CMAKE_CXX_EXTENSIONS OFF)         # 禁用编译器扩展（如 GNU 扩展）

# 生成可执行文件
# add_executable(可执行文件名 源码文件)
add_executable(hello main.cpp)
```

### 2.1.4 编译运行（通用流程）

```bash
# 1. 创建 build 目录并进入（out-of-source build，必做）
mkdir build && cd build

# 2. 生成构建文件（.. 表示 CMakeLists.txt 在上级目录）
cmake ..

# 3. 编译（-j 后接 CPU 核心数，加速编译）
make -j4

# 4. 运行
./hello          # Linux/Mac
# hello.exe      # Windows
```

## 2.2 多文件多目录项目

### 2.2.1 项目结构

```
calc/
├── CMakeLists.txt          # 根目录 CMakeLists.txt
├── src/
│   ├── CMakeLists.txt      # 子目录 CMakeLists.txt
│   ├── main.cpp
│   ├── add.cpp
│   └── subtract.cpp
└── include/
    ├── add.h
    └── subtract.h
```

### 2.2.2 头文件示例（`include/add.h`）

```cpp
#pragma once
int add(int a, int b);
```

### 2.2.3 根目录 CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.15)
project(Calculator LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

# 添加头文件目录（全局可见，或在子目录中单独设置）
include_directories(${PROJECT_SOURCE_DIR}/include)

# 添加子目录（执行子目录的 CMakeLists.txt）
add_subdirectory(src)
```

### 2.2.4 子目录 `src/CMakeLists.txt`

```cmake
# 收集当前目录下的所有源文件（避免手动罗列）
file(GLOB SRC_FILES ${CMAKE_CURRENT_SOURCE_DIR}/*.cpp)

# 生成可执行文件
add_executable(calc ${SRC_FILES})

# 可选：指定可执行文件输出路径（如 build/bin）
set(EXECUTABLE_OUTPUT_PATH ${PROJECT_BINARY_DIR}/bin)
```

### 2.2.5 编译运行

```bash
mkdir build && cd build
cmake ..
make -j4
./bin/calc
```

# 3. 进阶核心：库编译与链接

复杂项目必然涉及**库**（静态库/动态库），这是 CMake 最核心的能力之一。参见 [[5. CMake进阶与复杂项目|CMake 进阶]] 了解更多高级用法。

## 3.1 项目结构

```
math_lib/
├── CMakeLists.txt
├── lib/
│   ├── CMakeLists.txt
│   ├── math.cpp
│   └── math.h
└── app/
    ├── CMakeLists.txt
    └── main.cpp
```

## 3.2 头文件与源文件

`lib/math.h`：

```cpp
#pragma once
int multiply(int a, int b);
int divide(int a, int b);
```

`lib/math.cpp`：

```cpp
#include "math.h"
int multiply(int a, int b) { return a * b; }
int divide(int a, int b) { return b != 0 ? a / b : 0; }
```

## 3.3 根目录 CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.15)
project(MathLibDemo LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# 添加库目录和应用目录
add_subdirectory(lib)
add_subdirectory(app)
```

## 3.4 lib/CMakeLists.txt（编译库）

```cmake
# 编译静态库（STATIC）/ 动态库（SHARED）
# add_library(库名 库类型 源码文件)
add_library(math_lib STATIC math.cpp)

# 设置库的头文件目录（供链接者使用）
target_include_directories(math_lib PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})

# 可选：设置库输出路径
set(LIBRARY_OUTPUT_PATH ${PROJECT_BINARY_DIR}/lib)
```

- `PUBLIC`：头文件目录对库本身和链接该库的目标都可见（还有 `PRIVATE` / `INTERFACE`，参见 [[5. CMake进阶与复杂项目|目标属性]]）。

## 3.5 app/CMakeLists.txt（链接库）

```cmake
# 生成可执行文件
add_executable(math_app main.cpp)

# 链接库（核心命令）
target_link_libraries(math_app PRIVATE math_lib)

# 可选：可执行文件输出路径
set(EXECUTABLE_OUTPUT_PATH ${PROJECT_BINARY_DIR}/bin)
```

## 3.6 编译运行

```bash
mkdir build && cd build
cmake ..
make -j4
./bin/math_app      # 输出：3*4=12；8/2=4
```

> **总结**：CMake 的核心是 `CMakeLists.txt`，遵循 out-of-source build 模式。核心命令包括 `cmake_minimum_required`、`project`、`add_executable` / `add_library`、`target_link_libraries`。推荐优先使用 `target_*` 命令替代全局命令，精确控制每个目标的属性。

# 4. CMake 补充知识

## 4.1 真假值判断

CMake 的 `if()` 中优先解析为变量名，真假规则如下：

- **假值**：空字符串、`0`、`OFF`、`NO`、`FALSE`、`N`、`IGNORE`、`NOTFOUND`（或后缀为 `_NOTFOUND`）
- **真值**：除假值外的所有情况（包括非空字符串、`1`、`ON`、`YES`、`TRUE` 等）

以上规则对大小写不敏感。

## 4.2 输出路径设置

```cmake
# 全局库输出目录
set(LIBRARY_OUTPUT_PATH ${PROJECT_BINARY_DIR}/lib)

# 全局可执行文件输出目录
set(EXECUTABLE_OUTPUT_PATH ${PROJECT_BINARY_DIR}/bin)
```

> 使用 `target_*` 命令时，也可在子目录中按目标单独设置。

## 4.3 调用外部环境变量

```cmake
# 获取系统环境变量（如 PATH、HOME）
message(STATUS "HOME: $ENV{HOME}")
```

- 普通变量用 `${VAR_NAME}`
- 环境变量必须用 `$ENV{VAR_NAME}`

## 4.4 CMakeLists.txt 完整模板

适用于中等规模项目的实用模板（含源码自动扫描与过滤）：

```cmake
cmake_minimum_required(VERSION 3.17)
project(my_project)

# C++ 标准
set(CMAKE_CXX_STANDARD 20)

# 输出路径（自动创建）
set(LIBRARY_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/lib)
set(EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/bin)
file(MAKE_DIRECTORY ${LIBRARY_OUTPUT_PATH} ${EXECUTABLE_OUTPUT_PATH})

# 编译选项
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -g -fPIC -pthread")

# 公开头文件路径
set(PUBLIC_INCLUDE_DIR ${PROJECT_SOURCE_DIR}/include)
include_directories(${PUBLIC_INCLUDE_DIR})

# 递归扫描源文件
file(GLOB_RECURSE SRC_LIST
    CONFIGURE_DEPENDS
    ${PROJECT_SOURCE_DIR}/*.c
    ${PROJECT_SOURCE_DIR}/*.cpp
    ${PROJECT_SOURCE_DIR}/src/*.c
    ${PROJECT_SOURCE_DIR}/src/*.cpp
)

# 排除无关目录
list(FILTER SRC_LIST EXCLUDE REGEX "${PROJECT_SOURCE_DIR}/build/.*")
list(FILTER SRC_LIST EXCLUDE REGEX "${PROJECT_SOURCE_DIR}/third_party/.*")

# 编译可执行文件
add_executable(my_project ${SRC_LIST})
```

> **注意**：`file(GLOB_RECURSE)` 在大型项目中可能遗漏新增文件，推荐配合 `CONFIGURE_DEPENDS` 使用。手动罗列源文件是更可靠的方式。
