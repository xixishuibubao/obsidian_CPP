# 1. 进阶特性

## 1.1 目标属性（`target_*` 系列命令）

CMake 3.0+ 推荐使用 `target_*` 命令（而非全局命令），精准控制每个目标（可执行文件/库）的属性，是复杂项目的核心规范。

| 命令 | 作用 | 作用域 |
|------|------|--------|
| `target_include_directories` | 设置头文件目录 | 目标级 |
| `target_link_libraries` | 链接库 | 目标级 |
| `target_compile_options` | 设置编译选项（如 `-Wall`） | 目标级 |
| `target_compile_definitions` | 设置宏定义（如 `-DDEBUG`） | 目标级 |

**示例**：

```cmake
# 给 math_app 添加编译选项和宏定义
target_compile_options(math_app PRIVATE -Wall -Werror)   # 开启警告并视为错误
target_compile_definitions(math_app PRIVATE DEBUG=1)     # 定义宏 DEBUG=1
```

### 1.1.1 可见性关键字

| 关键字 | 含义 | 典型场景 |
|--------|------|----------|
| `PRIVATE` | 仅当前目标使用，不传递给上层 | 内部子模块间依赖 |
| `PUBLIC` | 当前目标使用，且传递给上层 | 对外暴露接口的依赖 |
| `INTERFACE` | 当前目标不使用，仅传递给上层 | 纯头文件库 |

## 1.2 条件编译（跨平台适配）

复杂项目需适配不同系统/编译器，用 `if-else` 实现：

```cmake
if(WIN32)
    # Windows 特有配置
    target_compile_definitions(math_lib PUBLIC WINDOWS_PLATFORM)
elseif(UNIX)
    # Linux/Mac 特有配置
    target_compile_options(math_lib PRIVATE -fPIC)
endif()

# 检测编译器
if(CMAKE_CXX_COMPILER_ID MATCHES "GNU")
    # GCC 特有选项
elseif(CMAKE_CXX_COMPILER_ID MATCHES "Clang")
    # Clang 特有选项
elseif(MSVC)
    # MSVC 特有选项（如 /W4）
endif()
```

## 1.3 外部依赖管理（`find_package`）

用 `find_package` 自动查找第三方库（如 Boost、OpenCV、Qt）：

```cmake
# 查找 Boost（要求 1.70+，只查找 program_options 组件）
find_package(Boost 1.70 REQUIRED COMPONENTS program_options)

# 链接（Boost::program_options 是 CMake 的"导入目标"，自动处理头文件和链接库）
target_link_libraries(my_app PRIVATE Boost::program_options)
```

## 1.4 安装规则（`install`）

```cmake
# 安装可执行文件到 /usr/local/bin（Linux）
install(TARGETS math_app RUNTIME DESTINATION bin)

# 安装库文件到 /usr/local/lib
install(TARGETS math_lib ARCHIVE DESTINATION lib)   # 静态库用 ARCHIVE

# 安装头文件到 /usr/local/include/math
install(FILES ${CMAKE_CURRENT_SOURCE_DIR}/math.h DESTINATION include/math)
```

```bash
# 执行安装
cd build
make install                         # Linux/Mac（需 sudo）
# cmake --install .                  # 通用命令（跨平台）
```

## 1.5 子模块/外部项目（`FetchContent`）

现代 CMake 推荐用 `FetchContent` 拉取外部依赖（如 GitHub 上的开源库）：

```cmake
include(FetchContent)

FetchContent_Declare(
    fmt
    GIT_REPOSITORY https://github.com/fmtlib/fmt.git
    GIT_TAG        10.1.1            # 指定版本
)

FetchContent_MakeAvailable(fmt)

# 链接 fmt 库
target_link_libraries(my_app PRIVATE fmt::fmt)
```

# 2. 复杂项目 CMake 设计

## 2.1 目录结构规范

```
project_root/
├── CMakeLists.txt          # 根配置：版本、项目名、全局编译标准
├── cmake/                  # 自定义 cmake 模块（FindXXX.cmake）
├── src/                    # 源码目录
│   ├── CMakeLists.txt      # 源码配置：添加子目录、链接库
│   ├── core/               # 核心模块（静态库）
│   ├── utils/              # 工具模块（静态库）
│   └── app/                # 可执行文件
├── include/                # 公开头文件
├── 3rdparty/               # 第三方依赖（本地库）
├── test/                   # 测试代码（如 Google Test）
└── docs/                   # 文档
```

## 2.2 核心设计原则

1. **模块化**：每个功能模块单独编译为库，通过 `add_subdirectory` 组织，降低耦合。
2. **目标级配置**：优先使用 `target_*` 命令，避免全局命令（如 `include_directories`）导致的冲突。
3. **跨平台兼容**：用 `if-else` 适配不同系统/编译器，避免硬编码路径/编译选项。
4. **可配置性**：通过 `option` 提供编译开关（如 `option(BUILD_TEST "Build test" ON)`）。
5. **清晰的依赖管理**：优先用 `find_package` / `FetchContent` 管理外部依赖，避免手动拷贝库文件。

## 2.3 根 CMakeLists.txt 模板

```cmake
cmake_minimum_required(VERSION 3.15)
project(ComplexProject LANGUAGES CXX)

# 全局配置
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)
set(CMAKE_EXPORT_COMPILE_COMMANDS ON)   # 生成 compile_commands.json（供 IDE/工具使用）

# 编译选项
option(BUILD_TEST "Build test cases" ON)
option(BUILD_SHARED_LIBS "Build shared libraries" OFF)

# 添加自定义 cmake 模块路径
list(APPEND CMAKE_MODULE_PATH ${PROJECT_SOURCE_DIR}/cmake)

# 添加源码目录
add_subdirectory(src)

# 可选：编译测试
if(BUILD_TEST)
    enable_testing()
    add_subdirectory(test)
endif()
```

# 3. 库内部多层子目录组织

## 3.1 场景定义

当库（lib）内部包含多层子目录，且这些子目录是该库的**内部依赖**（对外不可见）时，核心思路是**分层管理、私有依赖隔离**，确保子目录的代码仅服务于当前库，不对外暴露。

### 3.1.1 以 `network` 库为例

```
network_lib/
├── CMakeLists.txt
└── src/
    ├── CMakeLists.txt                # network 库的总入口
    ├── network/                      # 对外接口
    │   ├── CMakeLists.txt
    │   ├── network.h                 # 对外头文件
    │   └── network.cpp
    ├── core/                         # 内部核心模块（仅 network 使用）
    │   ├── CMakeLists.txt
    │   ├── buffer.h                  # 内部头文件
    │   └── buffer.cpp
    └── protocol/                     # 内部协议模块（仅 network 使用）
        ├── CMakeLists.txt
        ├── http_parser.h             # 内部头文件
        └── http_parser.cpp
```

## 3.2 分层编写方案

### 3.2.1 根目录 CMakeLists.txt

仅做全局基础配置：

```cmake
cmake_minimum_required(VERSION 3.15)
project(NetworkLibDemo LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

add_subdirectory(src)
```

### 3.2.2 src/CMakeLists.txt（库总入口）

```cmake
# 先添加内部子模块（顺序不重要，CMake 会处理依赖）
add_subdirectory(core)
add_subdirectory(protocol)

# 添加对外的 network 模块
add_subdirectory(network)
```

### 3.2.3 src/core/CMakeLists.txt（内部核心模块）

关键点：头文件目录用 `PRIVATE`，仅自身可见：

```cmake
add_library(network_core STATIC buffer.cpp buffer.h)

# PRIVATE：仅 core 自身可见
target_include_directories(network_core PRIVATE ${CMAKE_CURRENT_SOURCE_DIR})
target_compile_options(network_core PRIVATE -Wall -Werror)
```

### 3.2.4 src/protocol/CMakeLists.txt（内部协议模块）

```cmake
add_library(network_protocol STATIC http_parser.cpp http_parser.h)

target_include_directories(network_protocol
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/../core
)

# PRIVATE：依赖不传递给上层
target_link_libraries(network_protocol PRIVATE network_core)
```

### 3.2.5 src/network/CMakeLists.txt（对外主库）

对外头文件用 `PUBLIC`，内部依赖用 `PRIVATE`：

```cmake
add_library(network STATIC network.cpp network.h)

target_include_directories(network
    PUBLIC  ${CMAKE_CURRENT_SOURCE_DIR}          # 对外暴露
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/../core  # 内部可见
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/../protocol
)

# PRIVATE：内部依赖不泄露给上层
target_link_libraries(network
    PRIVATE network_core
    PRIVATE network_protocol
)
```

## 3.3 使用该库（验证隔离性）

### app/CMakeLists.txt

```cmake
add_executable(network_app main.cpp)

# 仅需链接对外的 network 库，无需关心内部 core/protocol
target_link_libraries(network_app PRIVATE network)
target_include_directories(network_app PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/../src/network)
```

### app/main.cpp

```cpp
#include "network.h"    // 仅包含对外头文件
// 无需包含 core/buffer.h 或 protocol/http_parser.h

int main() {
    NetworkClient client;
    client.connect("127.0.0.1", 8080);
    return 0;
}
```

## 3.4 优化：对象库（Object Library）

如果内部子模块不需要单独编译为库，可用**对象库**（`OBJECT`）替代静态库，减少编译产物：

```cmake
# src/core/CMakeLists.txt — 对象库
add_library(network_core OBJECT buffer.cpp buffer.h)
target_include_directories(network_core PRIVATE ${CMAKE_CURRENT_SOURCE_DIR})
```

```cmake
# src/protocol/CMakeLists.txt — 对象库
add_library(network_protocol OBJECT http_parser.cpp http_parser.h)
target_include_directories(network_protocol
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/../core
)
target_link_libraries(network_protocol PRIVATE network_core)
```

```cmake
# src/network/CMakeLists.txt — 引入对象文件
add_library(network STATIC
    network.cpp network.h
    $<TARGET_OBJECTS:network_core>         # 生成器表达式：引入对象文件
    $<TARGET_OBJECTS:network_protocol>
)

target_include_directories(network
    PUBLIC  ${CMAKE_CURRENT_SOURCE_DIR}
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/../core
    PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/../protocol
)
```

## 3.5 核心注意事项

1. **依赖可见性控制**：`PRIVATE` 仅当前目标使用；`PUBLIC` 传递给上层；`INTERFACE` 仅传递给上层（纯头文件库）。
2. **头文件隔离**：内部头文件用 `PRIVATE`，对外头文件用 `PUBLIC`，避免上层代码依赖内部实现。
3. **目录职责单一**：每层 CMakeLists.txt 只负责当前目录的构建，不跨目录操作。
4. **避免 GLOB 收集源文件**：复杂项目中 `file(GLOB)` 可能遗漏新增文件，建议手动罗列或用 `GLOB_RECURSE` 并配合 `CMAKE_CONFIGURE_DEPENDS`。
