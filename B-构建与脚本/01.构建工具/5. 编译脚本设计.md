# 1. 编译脚本设计

工业级 C/C++ 项目常在 CMake 之外保留**自研 Makefile + configure**（Nginx、部分嵌入式）。本笔记汇总学习路径与避坑；日常构建优先 [[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake]]，理解 Makefile 有助于读生成产物与裁剪构建。

## 【核心精简提纲】

### 记忆主干

**Makefile 本质 = 依赖图 + 增量规则**。进阶靠模式规则、自动依赖 `.d`、`wildcard` 收集源文件、并行 `-j` 与第三方库探测脚本（Nginx `auto/`）。与 CMake 互补：CMake 生成规则，本篇学**手写规则**的工业技巧。

### 核心要点清单

- 必读：GNU Make 手册（Pattern Rules、Variables、Functions）
- 实战：陈皓《跟我一起写 Makefile》— 15 个渐进案例
- 架构参考：Nginx `objs/Makefile`、`configure` + `auto/` 探测
- 关键技巧：`.d` 自动依赖、`order-only`（`|`）、`make -n` 展开调试
- 编译命令来自 [[B-构建与脚本/01.构建工具/1. g++|g++]]，规则写在 [[B-构建与脚本/01.构建工具/2. makefile|Makefile]] 语法层
- 交叉编译：`CROSS_COMPILE` 前缀 + `SYSROOT` 传入 `CC`/`CFLAGS`

### 构建链定位

```plain
手写 Makefile（本篇 + Nginx 风格）  ←→  CMake 生成 Makefile（[[3. CMake入门与实践]]）
         均调用 g++（[[1. g++]]）
```

- 渐进案例：头文件依赖、模块化编译、`.d` 自动生成

## 1.2 进阶实践：工业技巧

### Nginx 构建系统（1.25.1）

- `objs/Makefile`：`wildcard` 收集源文件
- `auto/`：Shell 探测编译器、PCRE、OpenSSL
- `configure` 参数 → Makefile 变量（`--with-http_ssl_module`）
- 工具：`make -n` 查看命令展开

### O'Reilly《Linux 程序员的 Makefile 指南》

- 自定义构建规则、大型项目扁平化、GNU Make 陷阱（递归 Make）

### GitHub：simple-makefile-examples

- 自动 `.d` 依赖、模块化 `--add-module` 风格、跨平台路径

## 1.3 避坑指南

| 问题 | 方案 |
|------|------|
| `-j` 并行竞态 | `order-only` 依赖 `\|` |
| 头文件路径硬编码 | 变量 + `$(SRCROOT)` |
| 第三方库探测失败 | `shell` 容错 + 明确报错 |
| 头文件改漏编 | `gcc -MMD -MP` 生成 `.d` |

Nginx 邮件组（nginx.org/pipermail/nginx/）搜索 `Makefile`、`configure` 有社区讨论。

## 1.7 与 CMake / pybind11 的选型

| 场景 | 推荐 |
|------|------|
| 跨平台、多依赖、IDE 集成 | [[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake]] |
| 极简服务器、可控构建、无 CMake 依赖 | 自研 Makefile（本篇） |
| Python 扩展模块 | CMake + `pybind11_add_module`（[[B-构建与脚本/02.脚本语言/4. C-C++与Python互调用|互调用]]），非手写 Makefile |

中小型 C 项目（10 万行内）自研 Makefile 可比 Autotools 更轻；超此规模或需 pybind/交叉编译 toolchain，CMake 维护成本更低。
