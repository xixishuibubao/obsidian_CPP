# 1. CMake 核心认知

CMake 不直接编译代码，而是**生成**跨平台构建文件（Linux Makefile、Windows VS、Xcode）。本笔记覆盖入门流程与库链接；进阶见 [[B-构建与脚本/01.构建工具/4. CMake进阶与复杂项目|CMake 进阶]]。编译选项底层仍由 [[B-构建与脚本/01.构建工具/1. g++|g++]] 执行。

## 【核心精简提纲】

### 记忆主干

**out-of-source build**：`mkdir build && cmake .. && cmake --build .`。核心命令 `project`、`add_executable`/`add_library`、`target_link_libraries`；现代写法优先 `target_*` 而非全局 `include_directories`。

### 核心要点清单

- 配置文件：`CMakeLists.txt`；缓存：`CMakeCache.txt`（首次 configure 后）
- 标准：`set(CMAKE_CXX_STANDARD 17)` + `REQUIRED ON` + `EXTENSIONS OFF`
- 库可见性：`PUBLIC` / `PRIVATE` / `INTERFACE`（详见进阶笔记）
- 真假值：`OFF`/`FALSE`/`0` 为假，其余为真（大小写不敏感）
- pybind11：`find_package(pybind11)` + `pybind11_add_module`，见 [[B-构建与脚本/02.脚本语言/4. C-C++与Python互调用|互调用]]
- 交叉编译：`CMAKE_TOOLCHAIN_FILE` 或 toolchain 设 `CMAKE_SYSROOT`、编译器前缀

## 【完整拓展详情】

## 1.1 基本概念

| 概念 | 说明 |
|------|------|
| CMake | 元构建系统，生成 Makefile / Ninja / VS 工程等 |
| `CMakeLists.txt` | 项目配置：目标、源文件、依赖、选项 |
| `CMakeCache.txt` | 缓存编译器路径、选项，修改后常需删 build 重配 |
| out-of-source | 源码与 `build/` 分离，避免污染仓库 |

## 1.2 环境准备

```bash
# Linux
sudo apt install cmake    # Debian/Ubuntu
cmake --version             # 建议 3.15+，pybind11 常用 3.17+
```

## 1.3 单文件 C++ 项目

**结构**：

```plain
hello/
├── CMakeLists.txt
└── main.cpp
```

**CMakeLists.txt**：

```cmake
cmake_minimum_required(VERSION 3.15)
project(HelloCMake LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

add_executable(hello main.cpp)
```

**编译运行**：

```bash
mkdir build && cd build
cmake ..
cmake --build . -j4
./hello
```

等价于手写 `g++ -std=c++17 main.cpp -o hello`（见 [[B-构建与脚本/01.构建工具/1. g++|g++]]）。

## 1.4 多文件多目录项目

```plain
calc/
├── CMakeLists.txt
├── src/
│   ├── CMakeLists.txt
│   ├── main.cpp
│   ├── add.cpp
│   └── subtract.cpp
└── include/
    ├── add.h
    └── subtract.h
```

**根 CMakeLists.txt**：

```cmake
cmake_minimum_required(VERSION 3.15)
project(Calculator LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

include_directories(${PROJECT_SOURCE_DIR}/include)
add_subdirectory(src)
```

**src/CMakeLists.txt**：

```cmake
file(GLOB SRC_FILES ${CMAKE_CURRENT_SOURCE_DIR}/*.cpp)
add_executable(calc ${SRC_FILES})
set(EXECUTABLE_OUTPUT_PATH ${PROJECT_BINARY_DIR}/bin)
```

> 大型项目避免裸 `GLOB`，改用手动列表或 `CONFIGURE_DEPENDS`；见 [[B-构建与脚本/01.构建工具/4. CMake进阶与复杂项目|CMake 进阶]]。

## 1.5 库编译与链接

```plain
math_lib/
├── CMakeLists.txt
├── lib/
│   ├── CMakeLists.txt
│   ├── math.cpp
│   └── math.h
└── app/
    ├── CMakeLists.txt
    └── main.cpp
```

**lib/CMakeLists.txt**：

```cmake
add_library(math_lib STATIC math.cpp)
target_include_directories(math_lib PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})
```

**app/CMakeLists.txt**：

```cmake
add_executable(math_app main.cpp)
target_link_libraries(math_app PRIVATE math_lib)
```

动态库：`add_library(math_lib SHARED math.cpp)`，需 `-fPIC`（CMake 默认处理）。

## 1.6 CMake 补充知识

### 真假值

`if()` 中假值：空、`0`、`OFF`、`NO`、`FALSE`、`N`、`IGNORE`、`NOTFOUND`；其余为真。

### 输出路径

```cmake
set(LIBRARY_OUTPUT_PATH ${PROJECT_BINARY_DIR}/lib)
set(EXECUTABLE_OUTPUT_PATH ${PROJECT_BINARY_DIR}/bin)
```

现代写法：`set_target_properties` + `RUNTIME_OUTPUT_DIRECTORY`。

### 环境变量

```cmake
message(STATUS "HOME: $ENV{HOME}")   # 环境变量用 $ENV{}
message(STATUS "SRC: ${PROJECT_SOURCE_DIR}")  # CMake 变量用 ${}
```

### 中等规模模板要点

- `CMAKE_CXX_FLAGS` 追加 `-g -fPIC -pthread`（对应 [[B-构建与脚本/01.构建工具/1. g++|g++ 选项]]）
- `file(GLOB_RECURSE ... CONFIGURE_DEPENDS)` 扫描源文件
- `list(FILTER ... EXCLUDE REGEX)` 排除 `build/`、`third_party/`

### 交叉编译简述

通过 toolchain 文件设置：

```cmake
set(CMAKE_SYSTEM_NAME Linux)
set(CMAKE_C_COMPILER aarch64-linux-gnu-gcc)
set(CMAKE_CXX_COMPILER aarch64-linux-gnu-g++)
set(CMAKE_SYSROOT /opt/sysroot)
```

配置：`cmake -DCMAKE_TOOLCHAIN_FILE=../toolchain.cmake ..`

实例：[[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|交叉编译优化选项]]。
