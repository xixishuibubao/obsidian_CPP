# 1. g++ 编译

GCC/G++ 是 C/C++ 编译链前端：预处理 → 编译 → 汇编 → 链接。本笔记覆盖常用选项、符号可见性、rpath、交叉编译与工具链辅助；构建编排见 [[B-构建与脚本/01.构建工具/2. makefile|Makefile]]、[[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake 入门]]。

## 【核心精简提纲】

### 记忆主干

**四阶段**：`-E` 预处理 → `-S` 汇编 → `-c` 目标文件 → 链接可执行文件/`.so`。选项通过 `-D` 宏、`-O` 优化、`-g` 调试、`-fvisibility` 符号、`-Wl,-rpath` 运行时库路径、`-march`/`--sysroot` 交叉编译落地。

### 核心要点清单

- 基本：`gcc/g++ <源文件> -o <目标> [-Wall -g -O2]`
- 宏：` -DABC` / `-DABC=123` 等价 `#define`
- 动态库：`gcc -shared -fPIC foo.c -o libfoo.so`
- 符号：` -fvisibility=hidden` 对 `.so` 会隐藏导出，调用方需 `extern "C"` + 显式导出
- rpath：`-Wl,-rpath,'$ORIGIN/lib'`（单引号防 shell 吃掉 `$ORIGIN`）
- 交叉编译：`--sysroot` + `-march`/` -mcpu`；禁用 `-march=native`
- pybind11 扩展模块：与目标板 Python ABI 一致，见 [[B-构建与脚本/02.脚本语言/4. C-C++与Python互调用|C/C++ 与 Python 互调用]]

## 1.1 gcc/g++ 编译命令

```bash
gcc [-控制参数] <原始文件> -o [目标文件名] [可选附加]
```

| 阶段 | 命令 | 产物 |
|------|------|------|
| 预处理 | `gcc -E a.c -o a.i` | `.i` |
| 编译 | `gcc -S a.c -o a.s` | `.s` |
| 汇编 | `gcc -c a.c -o a.o` | `.o` |
| 链接 | `gcc a.c -o a` | 可执行文件 |

常用附加选项：

| 选项 | 含义 |
|------|------|
| `-Wall` | 开启常见告警 |
| `-g` | 携带 gdb 调试信息，见 [[C-Linux生态/04.调试与优化/1. gdb|gdb]] |
| `-O0`~`-O3` / `-Os` | 优化等级 |
| `-DABC` / `-DABC=123` | 宏定义 |
| `-fPIC` | 位置无关代码，生成 `.so` 时常用 |

## 1.2 编译选项

### 符号可见性

```bash
-fvisibility=hidden
```

对直接编译和 `.a`：减少符号表，一般不影响调用。对 `.so`：未显式导出的符号外部不可见；ctypes/pybind11 调 C API 时需 `extern "C"` 与正确导出。

### 运行时库路径（rpath）

`-Wl,-rpath` 指定动态链接器运行时搜索 `.so` 的路径。

相关笔记：[[C-Linux生态/01.Linux环境/3. rpath与链接路径|rpath 与链接路径]]

**单路径**（逗号或等号均可）：

```bash
-Wl,-rpath,/usr/lib
-Wl,-rpath=/usr/lib
```

**多路径**：冒号 `:` 分隔。

```bash
-Wl,-rpath,/usr/lib:/usr/local/lib
-Wl,-rpath,/usr/lib -Wl,-rpath,/usr/local/lib   # 等效
```

**`$ORIGIN`（推荐）**：可执行文件/so 所在目录，嵌入式打包首选。

```bash
-Wl,-rpath,'$ORIGIN/lib'    # 必须单引号
```

**交叉编译**：rpath 写**目标板**路径，勿写主机 `/home/...`。

**验证**：`readelf -d your_app | grep RPATH`

**CMake 写法**（分号分隔）：

```cmake
set_target_properties(target PROPERTIES
    BUILD_RPATH "$ORIGIN;$ORIGIN/lib"
    INSTALL_RPATH "$ORIGIN;$ORIGIN/lib"
)
```

见 [[B-构建与脚本/01.构建工具/3. CMake入门与实践|CMake 入门]]。

| 坑点 | 正确做法 |
|------|----------|
| 多路径用逗号 | 用冒号 `:` |
| `$ORIGIN` 无引号 | shell 解析为空 |
| 交叉编译写主机路径 | 写目标板路径 |
| 裸 `./lib` | 换工作目录失效，用 `$ORIGIN` |

### 交叉编译

在主机上生成目标板机器码。实例：[[C-Linux生态/03.Linux开发/3. 交叉编译优化选项|RK3588 交叉编译优化选项]]、[[C-Linux生态/07.Telescope1126实战/6. 交叉编译与DriverSO部署|DriverSO 部署]]。

**基本参数**：

```bash
gcc \
  --sysroot=/opt/.../sysroots/cortexa7hf-neon-poky-linux-gnueabi \
  -march=armv7ve -mfpu=neon -mfloat-abi=hard -mcpu=cortex-a7
```

| 参数 | 说明 |
|------|------|
| `--sysroot` | 目标标准库根（交叉编译必选项） |
| `-march` | 指令集架构 |
| `-mfpu` / `-mfloat-abi` | ARM32 浮点 |
| `-mcpu` | `-march` + `-mtune` 简写 |

**`-march` 规则**：

| 写法 | 结果 |
|------|------|
| `-march=native` | 交叉编译**禁用**（识别主机 x86） |
| `armv8.2+` | 不规范，应为 `armv8.2-a` |
| `armv8-a` 及以上 | 默认带 SIMD/NEON |

**`-mtune` vs `-march` vs `-mcpu`**：

| 参数 | 控制 |
|------|------|
| `-march` | 可用指令集 |
| `-mtune` | 流水线调度 |
| `-mcpu` | 二者简写 |

**常用优化组合**：

```bash
-O3 -march=armv8.2-a -mtune=cortex-a76 -ffast-math -fopenmp -fno-rtti -fno-exceptions
```

**验证 SIMD**：`aarch64-linux-gnu-objdump -d prog | grep -E "vadd|vmul|ld1"`

**pybind11 交叉编译**：扩展模块须与板端 Python 版本、ABI tag（如 `cpython-312`）一致，通常用板端 `python3-config --includes` 与 CMake `find_package(Python3 ...)`，见 [[B-构建与脚本/02.脚本语言/4. C-C++与Python互调用|互调用笔记]]。

## 1.3 工具链辅助

```bash
objcopy     # 分离/合并调试信息
objdump -t  # 符号表
objdump -d  # 反汇编
objdump -p  # 链接库
readelf -d  # 动态段（RPATH 等）
strip       # 去除 debug 符号
```

**编译缓存**：`ccache` 缓存 `.o`，配合 [[B-构建与脚本/01.构建工具/2. makefile|Makefile]] `-j` 并行效果显著。
