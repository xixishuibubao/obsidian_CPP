# 1. ARM 与 X86 架构速览

对比 **ARM（RISC）** 与 **X86（CISC）** 的汇编习惯与内存模型，为 [[D-系统与架构/04.MCU嵌入式/1. mcu开发基础|MCU 开发]]、[[C-Linux生态/03.Linux开发/3. 交叉编译优化选项|交叉编译]]、[[C-Linux生态/07.Telescope1126实战/6. 交叉编译与DriverSO部署|DriverSO 部署]] 打基础。

## 【核心精简提纲】

### 记忆主干

**X86 = CISC，操作数顺序「目标, 源」，可直接访存；ARM = RISC，Load/Store，立即数常带 `#`，须先载入寄存器再运算。** ARM64 寄存器 `X0–X30`，`Wn` 为低 32 位且写 W 清零高 32 位。

### 核心要点清单

- Intel 语法：`mov rax, rbx`（目标在左）；内存 `[addr]`
- ARM64：`mov x0, #42`；无 `X31` 称谓（保留/SP 用途）
- ARM32：`R0–R12` 通用；`R13` SP、`R14` LR、`R15` PC；Thumb LSB=1
- CISC 微码复杂，分支预测与锁竞争常是瓶颈；RISC 对 **cache miss** 更敏感
- RK3588 / 1126B 属 ARM64；MCU 裸机多见 ARM32 Thumb

### 关键对照

| 维度 | X86-64 | ARM32 | ARM64 |
|------|--------|-------|-------|
| 风格 | CISC | RISC | RISC |
| 立即数 | 多数无 `#` | 通常 `#` | 多数无 `#` |
| 访存 | 指令可操作内存 | Load/Store | Load/Store |
| 生态 | PC/服务器 | MCU/嵌入式 Linux | 手机/板端 |

## 【完整拓展详情】

## 1.1 架构对照

| 维度 | X86 / X86-64 | ARM32 | ARM64 (AArch64) |
|------|--------------|-------|-----------------|
| 指令风格 | CISC，复杂寻址 | RISC，Load/Store | RISC，固定 32-bit 编码 |
| 通用寄存器 | EAX…R15（64-bit） | R0–R15 | X0–X30，W0–W30 为低 32 位 |
| 立即数 | 多数不需 `#` | 通常需要 `#` | 多数不需 `#` |
| 典型生态 | PC · 服务器 | MCU · 嵌入式 Linux | 手机 · RK3588 · 1126B |
| 内存模型 | 允许较多内存操作数 | 须先 Load 到寄存器再运算 | 同 ARM32 思路 |

### 前置依赖

- 理解 **寄存器 vs 内存**、**指令集 vs 微架构**（`-mtune` 调度的对象）
- 交叉编译时目标 ISA 由 `-march` 决定，见 [[B-构建与脚本/01.构建工具/1. g++|g++]]、[[C-Linux生态/03.Linux开发/3. 交叉编译优化选项|交叉编译优化]]

### 易混概念

- **ARM32 ≠ ARM64**：指令集不兼容，二进制不可混跑；工具链 triple 不同（`arm-linux-gnueabihf` vs `aarch64-linux-gnu`）
- **AArch64 是 ARMv8 的 64 位执行状态**，与 32 位 ARM 用户态是两套 ISA

## 1.2 汇编语法

### 1.2.1 X86 Intel 语法

> Intel 语法是一种书写格式，AMD 与 Intel CPU 均支持。

- 操作数顺序：`指令 目标, 源`
- 注释：`;`
- 内存寻址：`[addr]` 表示解引用

```nasm
; rax = rbx + 1
mov rax, rbx
add rax, 1
mov rax, [rbx]      ; 从内存加载
add [rbx], 1        ; 直接写内存（CISC 特征）
```

**自测**：ARM32 写立即数常加 `#`，X86 Intel 通常不加——因汇编器语法约定不同，非硬件绝对限制。

### 1.2.2 ARM64

1. `X0~X30` 为 64 位通用寄存器；`X0–X7` 常用于参数/返回值（AAPCS64）
2. `W0~W30` 为对应 `X` 的低 32 位；**写 W 会将高 32 位清零**
3. 无 `X31` 称谓，该槽位为保留/特殊用途（如 SP 在另一套编码）
4. ARM64 是 ARM32 的超集概念上的 64 位扩展，但用户态 ISA 独立

```arm
mov x0, #42
add x0, x0, #1
ldr x1, [x2]        ; Load
str x1, [x3]        ; Store
```

### 1.2.3 ARM32（Thumb）

- `R0–R12` 通用；`R13` SP；`R14` LR；`R15` PC
- 嵌入式常见 **Thumb-2** 混合 16/32 位指令
- **Cortex-M 仅 Thumb**：函数指针 LSB 必须为 1，见 [[D-系统与架构/04.MCU嵌入式/2. ARM32内存注入（UART）|ARM32 内存注入]]

```arm
movs r0, #42
adds r0, #1
```

## 1.3 内存模型与性能

### 1.3.1 X86（CISC）

- 指令可直接操作内存，微码在 CPU 内拆解为 μops
- 性能瓶颈常在：**分支预测**、锁竞争、缓存一致性（多核）
- 对程序员「方便」，但优化仍需关注数据局部性

### 1.3.2 ARM（RISC）

- **Load → 运算 → Store**；编译器/手写汇编都需显式安排寄存器
- 对 **cache miss、DMA 一致性、对齐** 更敏感
- 板端图像管线（MPP/ISP）需关注 buffer 对齐与 cache 维护，见 [[C-Linux生态/07.Telescope1126实战/10. Rockchip MPP与ISP预览管线|MPP 与 ISP]]

### 方案对比：何时关心 ISA 差异

| 场景 | 关注点 |
|------|--------|
| 应用层 C/C++ | mostly 编译器处理；调 `-march`/`-mtune` |
| 内联汇编 / 裸机 | 寄存器约定、Thumb bit、内存屏障 |
| 性能调优 | ARM 上 SIMD/NEON 需显式开启 march；X86 上 AVX 同理 |
