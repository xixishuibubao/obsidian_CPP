# 1. ARM 与 X86 架构速览

## 1.1 架构对照

| 维度 | X86 / X86-64 | ARM32 | ARM64 (AArch64) |
|------|--------------|-------|-----------------|
| 指令风格 | CISC，复杂寻址 | RISC，Load/Store | RISC，固定 32-bit 编码 |
| 通用寄存器 | EAX…R15（64-bit） | R0–R15 | X0–X30，W0–W30 为低 32 位 |
| 立即数 | 多数不需 `#` | 通常需要 `#` | 多数不需 `#` |
| 典型生态 | PC · 服务器 | MCU · 嵌入式 Linux | 手机 · RK3588 · 1126B |
| 内存模型 | 允许较多内存操作数 | 须先 Load 到寄存器再运算 | 同 ARM32 思路 |

## 1.2 汇编语法

### 1.2.1 X86 Intel 语法

> Intel 语法是一种书写格式，AMD 与 Intel CPU 均支持。

- 操作数顺序：`指令 目标, 源`
- 注释：`;`（类似 C++ 的 `//`）
- 内存寻址：`[addr]` 表示解引用

```nasm
; Intel 示例：rax = rbx + 1
mov rax, rbx
add rax, 1
```

### 1.2.2 ARM64

1. `X0~X30` 为 64 位通用寄存器。
2. `W0~W30` 为对应 `X` 的低 32 位；写 `W` 会清零高 32 位。
3. 无 `X31` 称谓，该槽位为保留/特殊用途（如 SP）。
4. ARM64 是 ARM32 的超集；ARM32 寄存器为 `R0~R15`。

```arm
// ARM64 示例
mov x0, #42
add x0, x0, #1
```

### 1.2.3 ARM32（Thumb）

- `R0–R12` 通用；`R13` SP；`R14` LR；`R15` PC。
- 嵌入式常见 **Thumb-2** 混合 16/32 位指令，见 [[D-系统与架构/04.MCU嵌入式/2. ARM32内存注入（UART）|ARM32 内存注入]]。

## 1.3 内存模型与性能

### 1.3.1 X86（CISC）

- 指令可直接操作内存，微码复杂。
- 性能瓶颈常在：**分支预测**、锁竞争、缓存一致性。

### 1.3.2 ARM（RISC）

- **Load → 运算 → Store**，对缓存与 **cache miss** 更敏感。
- 嵌入式需关注 DMA 一致性、对齐（见驱动与 MPP 管线笔记）。
