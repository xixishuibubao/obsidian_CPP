# 1. MCU 开发基础

Keil/JFlash 调试、Cortex-M 异常栈帧、boot/app 跳转与存储特性的嵌入式速查。

## 【核心精简提纲】

### 记忆主干

**halt 异常**：硬件自动把 `R0–R3,R12,LR,PC,XPSR` 压栈，SP 指向 R0；用 **LR−1** 回溯 Thumb 调用链。**boot→app**：关中断、`DeInit` 时钟、重设 VTOR，勿重复初始化 RCC。

### 核心要点清单

- Keil 中文路径 → unicode 映射报错；`fromelf` 生成 dis/bin
- 栈帧低→高：`R0,R1,R2,R3,R12,LR,PC,XPSR`（32 字节）
- Flash 页擦除 vs EEPROM 字擦；boot 在片内 flash，FS 常挂外设
- AC5 `at(0x08008000)` / AC6 `section(".ARM.__at_...")` 定位变量
- `printf` 走 microlib + 重定向 `fputc`；或 `vsnprintf` 自定义输出

## 【完整拓展详情】

## 1.1 Keil5 与反汇编

### 1.1.1 常见报错

- **无 unicode 映射字符**：工程路径含中文，Keil 编译过但界面 X 标记 → 换英文路径。

### 1.1.2 反汇编与导出

```bash
fromelf --text -a -c --output name.dis name.axf   # axf 名与路径一致
fromelf.exe --bin -o "$L@L.bin" "#L"
fromelf.exe --text -a -c -o "$L@L.dis" "#L"
```

- 未重定向 `printf` 且链标准库时，可能断在 `sys_write`/`sys_open` 卡死 → 用 microlib 或自定义 `vsprintf` 输出。

## 1.2 JFlash 与 halt 异常

### 1.2.1 导出 RAM 快照

`j-flash-commander` 顺序：`usb` → `halt` → 选芯片/接口 →  
`savebin e:/ram.bin 0x20000000 0x10000`

### 1.2.2 栈帧布局

SP 指向 **R0**；地址递增依次为 `R0,R1,R2,R3,R12,LR,PC,XPSR`（8×4 字节）。其后为函数栈局部变量；**LR** 可用于回溯调用者（Thumb 下 LR 末位为 1，实际地址 −1）。

### 1.2.3 寄存器速查

| 寄存器 | 用途 |
|--------|------|
| R0–R3 | 参数 / 返回值 |
| R4–R11 | 局部变量（AAPCS 需 callee 保存） |
| R12 (IP) | 子程序 scratch |
| LR | 返回地址 |
| PC | 当前指令（Thumb 末位 0） |

RTOS PendSV 切换：先入栈 `XPSR,PC,LR,R12,R3–R0`，再入栈 `R11–R4`，符合 AAPCS。

## 1.3 简单汇编

| 指令 | 含义 |
|------|------|
| `BL` | 带链接跳转 |
| `B` | 无条件跳转 / 死循环 |
| `LDR/STR` | 加载/存储；`[R4]` 间接寻址 |
| `BX LR` | 弹栈返回（非单纯 mov PC） |

变量赋值 `a=5`：`LDR R0,a` → `MOV R1,#5` → `STR R1,[R0]`。

## 1.4 存储与 boot

- **EEPROM**：字擦写；**Flash/NOR**：页擦，适合 seldom-change 数据
- Boot 在片内 flash；文件系统多在外部 SD 等
- 合并 hex：boot 去掉 `:00000001FF`，拼接 app hex，再烧 `0x08000000`

## 1.5 Bootloader 与 App

### 1.5.1 定位编译（Keil AC5/AC6）

```cpp
#if __ARMCOMPILER_VERSION >= 6000000
#define PLACE_AT(addr) __attribute__((used, section(".ARM.__at_" #addr)))
#else
#define PLACE_AT(addr) __attribute__((at(addr), used))
#endif
```

GCC 不兼容上述写法；复杂项目用 scatter/scatter 文件。

### 1.5.2 C/C++ 混编

`.h` 中函数声明包 `extern "C" { ... }` + `#ifdef __cplusplus`  guard。

### 1.5.3 boot_to_app 踩坑

| 要点 | 说明 |
|------|------|
| 关中断 | boot 跳转前 `__disable_irq()`；跳转后 app 入口 `__enable_irq()` |
| 勿 HAL_Delay | 关中断后 SysTick 不跑 |
| VTOR | app 起始设中断向量偏移 |
| RCC | **勿** boot 与 app 各配一次时钟；app 先 `HAL_RCC_DeInit()` 再 `SystemClock_Config()` |
| microlib | 重定向 `fputc` 或 `vsnprintf` + 串口发送，避免半主机 syscalls |

```cpp
void App_Entry(void) {
    __disable_irq();
    HAL_RCC_DeInit();
    SystemClock_Config();
    HAL_Init();
    __enable_irq();
    // MX_xxx_Init(); while(1) { ... }
}
```

### 1.5.4 自定义日志（vsnprintf）

用 `vsnprintf` 格式化到缓冲区再 `uart_send`，避免 `printf` 直接拉半主机；带 tag 的对齐输出可用 `%-12s` 固定标签宽度（见原笔记 `myprintf_with_format` 完整实现）。
