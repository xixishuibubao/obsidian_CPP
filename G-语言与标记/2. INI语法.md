# 1. INI 语法

## 【核心精简提纲】

### 记忆主干

INI（Initialization）是一种 段（section）+ 键值对 的配置格式，语法简单、人类可读，长期用于应用配置与 Windows 生态。与 8. YAML语法 同属结构化配置/数据标记语言。

### 核心要点清单

- **结构**：`[section]` + `key = value`；无原生嵌套
- **类型**：值均为字符串，布尔/整数由应用解析
- **适用**：`server.ini`、嵌入式板端、Git 风格配置
- **不适用**：深层嵌套、数组列表 → 用 YAML/JSON
- **Python**：`configparser`；本项目 **ConfigObj** 读 `hardware.ini`
- **C++**：inih 回调；返回 0 终止解析

### 学习路径

1. 认 `[section]` + `key = value` 基本形
2. Python：`configparser` / 本项目 `ConfigObj`
3. C++：inih 回调式解析
4. 与 [[3. YAML语法|YAML]] 对比选型

### 关键结论 / 流程

```plain
运维改几行配置 → INI
K8s/CI 嵌套列表 → YAML
```

## 【完整拓展详情】

INI（Initialization）是一种 **段（section）+ 键值对** 的配置格式，语法简单、人类可读，长期用于应用配置与 Windows 生态。与 [[3. YAML语法|YAML]] 同属结构化配置/数据标记语言。

> 本 vault 笔记 **不使用 YAML frontmatter**（见 [[G-语言与标记/1. Markdown|Markdown 笔记]]），但项目运行配置（如 `server.ini`）仍常用 INI。

## 1.1 基本结构

```ini
; 行注释（部分解析器也支持 #）
# 这也是注释

[section_name]
key = value
another_key = 123
flag = true

[database]
host = 127.0.0.1
port = 3306
```

| 元素 | 语法 | 说明 |
|------|------|------|
| Section | `[name]` | 逻辑分组，类似命名空间 |
| 键值对 | `key = value` | `=` 两侧空格可选 |
| 注释 | `;` 或 `#` | 依解析器而定；Python `configparser` 默认 `;` |
| 空行 | 忽略 | 用于可读性 |

**无原生嵌套**：INI 只有「段 + 扁平键」，不能像 YAML 那样缩进嵌套对象。复杂结构常用 **键名带点号** 或 **多段模拟**：

```ini
[paths]
FitsSavePath = /data/fits
RawSavePath = /data/raw

[preview]
width = 1920
height = 1080
```

或约定 `database.host = 127.0.0.1`（需解析器支持或应用层自行拆分）。

## 1.2 值类型与引号

INI **没有标准类型系统**，值一律是字符串，由应用或库做转换：

```ini
[system]
config = TestFake
version = 1.2.3
enabled = true          ; 读出来仍是 "true" 字符串
count = 42
path = /oem/hardware.ini
quoted = "含空格的 value"
```

| 类型 | 处理方式 |
|------|----------|
| 整数 / 布尔 | 应用 `int()`、`lower() == 'true'` 等自行解析 |
| 列表 | 逗号分隔 `hosts = a,b,c`，或重复键（非所有解析器支持） |
| 多行文本 | 不标准；少见，复杂文本更倾向 YAML/JSON |

## 1.3 常用场景

| 场景 | 典型文件 | 为何用 INI |
|------|----------|------------|
| **应用运行时配置** | `server.ini`、`hardware.ini` | 运维可直接改、diff 清晰 |
| **Windows / 传统软件** | `desktop.ini`、游戏设置 | 历史生态 |
| **Git 本地配置** | `~/.gitconfig`（INI 风格） | `[user]` / `[core]` 段 |
| **嵌入式 / 板端** | `/oem/hardware.ini` | 体积小、解析器轻（C/Python 均有） |
| **构建工具部分配置** | 某些 IDE、旧版工具链 | 简单键值足够 |

**典型用法**：嵌入式/服务端用 **ConfigObj** 或标准库读 `server.ini` 与硬件配置，见 [[2. Python|Python 笔记]]。

**不太适合 INI 的场景**：

- 深层嵌套结构（Kubernetes 级配置 → 用 YAML）
- 数组/对象列表（CI matrix、多服务编排 → YAML/JSON）
- 跨语言 Schema 校验（OpenAPI、JSON Schema → JSON/YAML）

## 1.4 Python 解析

依赖：`configparser` 为标准库；本项目用 `configobj`（`pip install configobj`）。

### 1.4.1 读取：configparser

```python
import configparser
from pathlib import Path

cfg = configparser.ConfigParser()
# read 接受多个路径，后者覆盖前者
read_ok = cfg.read(["server.ini", "/oem/hardware.ini"], encoding="utf-8")
if not read_ok:
    raise FileNotFoundError("no ini loaded")

fits_path = cfg["path"]["FitsSavePath"]
port = cfg.getint("server", "port", fallback=8999)
enabled = cfg.getboolean("system", "enabled", fallback=False)

# 遍历某段
for key, value in cfg["database"].items():
    print(key, value)
```

### 1.4.2 读取：ConfigObj（本项目）

```python
from configobj import ConfigObj

cfg = ConfigObj("server.ini", encoding="utf-8")
fits_path = cfg["path"]["FitsSavePath"]

# hardware.ini 示例字段
startup_key = cfg["system"]["config"]  # → StartUpConfig 查找用
```

见 [[2. Python|Python 笔记]] 中 ConfigObj 示例。

### 1.4.3 写入与类型转换

```python
import configparser

cfg = configparser.ConfigParser()
cfg["system"] = {
    "config": "TestFake",
    "version": "1.0",
}
cfg["path"] = {"FitsSavePath": "/data/fits"}

with open("out.ini", "w", encoding="utf-8") as f:
    cfg.write(f)

# INI 值本质是字符串，布尔/整数需自己转
raw = cfg["system"]["enabled"]  # "true"
enabled = raw.lower() in ("1", "true", "yes", "on")
```

| API | 用途 |
|-----|------|
| `cfg.read([paths])` | 加载一个或多个文件 |
| `cfg["sec"]["key"]` | 读字符串 |
| `cfg.getint` / `getboolean` | 带 fallback 的类型转换 |
| `cfg.write(f)` | 写回 INI |

## 1.5 C++ 解析

INI 无 C++ 标准库支持，常用第三方库。**嵌入式 / 轻量** 选 **inih**（单头文件 + 可选 `ini.c`）；功能更全可选 **SimpleIni**、**iniparser**。

### 1.5.1 inih：回调式解析

示例 `server.ini`：

```ini
[path]
FitsSavePath = /data/fits

[server]
port = 8999
```

```cpp
#include <iostream>
#include <string>
#include <unordered_map>
#include "ini.h"  // https://github.com/rxi/ini

using SectionMap = std::unordered_map<std::string, std::string>;
using IniStore = std::unordered_map<std::string, SectionMap>;

static int ini_handler(void* user, const char* section,
                       const char* name, const char* value) {
    if (!section || !name || !value) return 1;
    auto* store = static_cast<IniStore*>(user);
    (*store)[section][name] = value;
    return 1;  // 返回 0 终止解析
}

int main() {
    IniStore store;
    const int err = ini_parse("server.ini", ini_handler, &store);
    if (err < 0) {
        std::cerr << "ini_parse failed: " << err << '\n';
        return 1;
    }

    const std::string& fits = store["path"]["FitsSavePath"];
    const int port = std::stoi(store["server"]["port"]);
    std::cout << fits << " port=" << port << '\n';
    return 0;
}
```

```bash
# 将 ini.h / ini.c 放入工程，或 CMake FetchContent
g++ -std=c++17 main.cpp ini.c -o read_ini
```

### 1.5.2 库选型

| 库 | 语言 | 特点 |
|----|------|------|
| **inih** | C/C++ | 极小、无依赖，适合 MCU / 驱动 |
| **iniparser** | C | Debian 常见 `libiniparser-dev` |
| **SimpleIni** | C++ | 头文件模板，支持 Unicode |
| **Boost.PropertyTree** | C++ | `read_ini`，与 Boost 绑定 |

写入 INI 时多数库只提供读；写回可用 `fprintf` 按 `[section]\nkey = value\n` 格式拼接，或继续用 Python 生成部署配置。

## 1.6 INI vs YAML 速查

| 维度 | INI | YAML |
|------|-----|------|
| 结构 | 段 + 扁平键 | 缩进嵌套 map/list |
| 学习成本 | 极低 | 中（缩进敏感） |
| 人工编辑 | 非常适合运维改几行 | 适合复杂结构 |
| 标准程度 | 事实标准、实现有差异 | YAML 1.2 规范 |
| 典型生态 | 传统桌面/嵌入式配置 | CI、K8s、Docker Compose |

详见 [[3. YAML语法|YAML 语法]]。
