# 1. Markdown 语法速查

本 vault 笔记标准格式：无 YAML frontmatter、wikilink 双括号引用、代码块带语言标注。

## 【核心精简提纲】

### 记忆主干

结构靠 `#` 标题与空行；强调靠 `**`/`~~`；列表 `-`/`1.`；代码 `` ` `` 行内 / ` ```lang ` 块；链接 `[text](url)`；图片 `![alt](path)`；表格 `|` + `---`。

### 核心要点清单（vault 用法）

- **标题**：`#` 数量 = 级别（本 vault：`# 1.` / `## 1.1` 强制编号）
- **强调**：`**粗体**` `*斜体*` `~~删除线~~`
- **列表**：无序 `-`；有序 `1.`；嵌套缩进 2～4 空格
- **代码**：行内 `` `code` ``；块 ` ```bash ` 等（禁止无语言缩进块）
- **引用**：`>` 单行或多行
- **链接**：行内 `[text](url)`；wikilink 见下方
- **图片**：`![描述](picture/xxx.png)`（本 vault 统一小写 `.png`）
- **表格**：`| col |` + `| --- |`；`:---:` 居中；**表格内禁止** 带别名的 wikilink（`|` 被当列分隔符）

### 关键流程

```plain
写笔记 → # 1. 标题 → ## 1.1 章节 → 代码块带语言 → wikilink 互链
```

## 【完整拓展详情】

## 1.1 wikilink（vault 特有）

- 目标路径**不含 `.md` 后缀**
- 跨模块使用 vault 相对路径：格式为 双左方括号 + 路径/文件名 + 双右方括号（不含 `.md`）
- 同目录短链可接受但推荐全路径

## 1.2 代码风格

- 代码块必须有语言标注：`` ```c++ ``
- 行内代码用单个反引号

## 1.3 引用与表格

- 引用支持多级 `>` / `>>`
- 表格使用 `|` 分列，列对齐用 `:---` / `:---:` / `---:`
- 表格内需用 HTML `<br>` 换行，而非 \n

## 1.4 图片与链接

- 图片路径从 vault 根出发：`![描述](picture/xxx.png)`
- 链接优先使用 wikilink 而非 URL

## 1.5 分割线

- `---` 或 `***` 表示分割线
- 用于章节间大分隔
