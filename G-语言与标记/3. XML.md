# 1. XML 语法

XML（eXtensible Markup Language）用标签嵌套描述结构化数据，强调 Schema 校验与跨平台交换。与 [[G-语言与标记/7. INI语法|INI]]、[[G-语言与标记/8. YAML语法|YAML]] 互补。

## 【核心精简提纲】

### 记忆主干

**树形标签 + 单一根 + 属性键值**；机器交换与 Schema 校验强，人工编辑弱于 YAML/INI；解析走 DOM（整树）或 SAX（流式）。

### 核心要点清单

- 有且仅有一个根元素；标签必须闭合或自闭合 `<tag/>`
- 属性写在开始标签：`key="value"`；文本内容放子元素而非属性（复杂数据）
- 声明 `<?xml version="1.0" encoding="UTF-8"?>` 推荐写明 encoding
- 特殊字符转义：`&lt;` `&gt;` `&amp;` `&quot;` `&apos;`
- Python：`xml.etree.ElementTree`；C++：pugixml / tinyxml2
- 不用 XML 的场景：人工频繁改配置 → YAML/INI；Web API → JSON；本 vault → Markdown

### 关键结论 / 流程

```
读 XML → parse → find/findall 遍历 → 改 Element → write
写 XML → 建 Element 树 → SubElement → ElementTree.write(xml_declaration=True)
```

| 对比 | XML | INI | YAML |
|------|-----|-----|------|
| 嵌套 | 标签 | 弱 | 缩进 |
| Schema | XSD 成熟 | 无 | 部分 |
| 可读性 | 中 | 高（简单） | 高 |

## 【完整拓展详情】

## 1.1 基本结构与语法规则

```xml
<?xml version="1.0" encoding="UTF-8"?>
<root>
    <device id="cam01" enabled="true">
        <name>main_camera</name>
        <port>8999</port>
    </device>
    <device id="cam02"/>
</root>
```

| 元素 | 说明 |
|------|------|
| 声明 | 版本与编码；缺省 UTF-8 时仍建议显式写出 |
| 标签 | `<tag>...</tag>` 成对；空元素用 `<tag/>` |
| 属性 | 开始标签内 `key="value"`，值必须引号包裹 |
| 注释 | `<!-- 注释，内不可含 --` |
| 根元素 | 文档有且仅有一个顶层元素 |

### 前置依赖

- 理解 **树形 DOM**：节点 = 元素 + 属性 + 文本子节点
- 字符编码：文件编码须与声明一致，否则解析器报 `UnicodeDecodeError` / 乱码

### 适用边界

| 适合 | 不适合 |
|------|--------|
| 企业/移动端配置（Maven、Android Manifest） | 人手高频修改的配置 |
| SOAP、RSS、SVG、JUnit 报告 | 现代 REST JSON API |
| 需要 XSD 强校验的流水线 | 本 vault 笔记（→ Markdown） |

## 1.2 实体、命名空间与 Schema

### 字符实体

| 字符 | 实体 |
|------|------|
| `<` | `&lt;` |
| `>` | `&gt;` |
| `&` | `&amp;` |
| `"` | `&quot;` |

**典型错误**：在属性或文本中直接写 `a < b` 未转义 → 解析失败。

### 命名空间（xmlns）

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <application android:label="App"/>
</manifest>
```

- `xmlns` 避免不同 Schema 下同名标签冲突
- 查询时需带前缀或使用支持 namespace 的 API（`ElementTree` 用 `{uri}tag` 形式）

### XSD / DTD（概念）

- **DTD**：早期文档类型定义，能力有限
- **XSD**：XML Schema，类型系统完整，IDE 与 CI 可校验
- 工程价值：配置错误在构建期暴露，而非运行期

## 1.3 常用场景

| 场景 | 典型用途 |
|------|----------|
| 配置文件 | Maven `pom.xml`、Android `AndroidManifest.xml` |
| 数据交换 | SOAP、RSS/Atom、SVG |
| 文档标记 | DocBook、DITA |
| 构建/工具 | CMake 部分生成物、JUnit 报告、PlantUML 导出 |

## 1.4 Python 解析

```python
import xml.etree.ElementTree as ET

tree = ET.parse("config.xml")
root = tree.getroot()

for dev in root.findall("device"):
    dev_id = dev.get("id")
    name = dev.find("name")
    if name is not None:
        print(dev_id, name.text)

cfg = ET.Element("root")
ET.SubElement(cfg, "device", id="cam01").text = "main"
ET.ElementTree(cfg).write("out.xml", encoding="utf-8", xml_declaration=True)
```

| API | 用途 |
|-----|------|
| `ET.parse` / `fromstring` | 读文件 / 字符串 |
| `find` / `findall` | 直接子元素（不递归全树） |
| `Element.get` / `.text` | 属性 / 文本 |
| `ElementTree.write` | 写回，记得 `xml_declaration=True` |

### 踩坑点

- `find("foo")` 只查**直接**子节点；深层用 `findall(".//foo")` 或遍历
- 未检查 `find` 返回 `None` 就访问 `.text` → `AttributeError`
- 默认不保留空白文本节点；格式化输出需 `indent`（Python 3.9+）或第三方库

## 1.5 C++ 解析

常用 **pugixml**（轻量 DOM）或 **tinyxml2**。

```cpp
#include "pugixml.hpp"
#include <iostream>

int main() {
    pugi::xml_document doc;
    if (!doc.load_file("config.xml")) return 1;

    for (pugi::xml_node dev : doc.child("root").children("device")) {
        std::cout << dev.attribute("id").as_string()
                  << " " << dev.child("name").text().as_string() << '\n';
    }
    return 0;
}
```

- 链接 pugixml 通常只需头文件 + 单 `.cpp` 实现，适合嵌入式工具链
- 大文件考虑 **SAX 流式**（pugixml 的 `xml_reader`）降低内存峰值

## 1.6 XML vs INI vs YAML vs JSON

| 维度 | XML | INI | YAML | JSON |
|------|-----|-----|------|------|
| 嵌套 | 标签 | 弱 | 缩进 | 对象/数组 |
| 注释 | 支持 | 支持 | 支持 | 不支持 |
| 人工可读 | 中 | 高（简单） | 高 | 高 |
| Schema | XSD 成熟 | 无 | 部分 | JSON Schema |
| 典型生态 | Java/Android/企业 | 游戏/传统配置 | DevOps/K8s | Web API |

### 易混概念

- **XML 不是 HTML**：HTML 标签可省略闭合、大小写不敏感；XML 严格良构
- **属性 vs 子元素**：语义等价时常用子元素表示可变长内容，属性表示标识/布尔
