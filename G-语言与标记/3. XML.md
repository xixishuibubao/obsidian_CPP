# 1. XML 语法

XML（eXtensible Markup Language）用 **标签嵌套** 描述结构化数据，强调 Schema 校验与跨平台交换。与 [[7. INI语法|INI]]（扁平配置）、[[8. YAML语法|YAML]]（缩进配置）互补。

## 1.1 基本结构

```xml
<?xml version="1.0" encoding="UTF-8"?>
<root>
    <device id="cam01" enabled="true">
        <name>main_camera</name>
        <port>8999</port>
    </device>
    <device id="cam02"/>
</root>
```

| 元素 | 说明 |
|------|------|
| 声明 | `<?xml version="1.0" encoding="UTF-8"?>` 可选但推荐 |
| 标签 | `<tag>...</tag>` 成对；`<tag/>` 自闭合 |
| 属性 | 写在开始标签内 `key="value"` |
| 注释 | `<!-- 注释 -->` |
| 根元素 | 有且仅有一个顶层元素 |

## 1.2 常用场景

| 场景 | 典型用途 |
|------|----------|
| **配置文件** | Maven `pom.xml`、Android `AndroidManifest.xml`、旧版 .NET |
| **数据交换** | SOAP、RSS/Atom、SVG |
| **文档标记** | DocBook、DITA（技术文档流水线） |
| **构建/工具** | CMake 部分生成物、JUnit 报告、PlantUML 可导出 |

**何时不用 XML**：人工频繁编辑的配置（→ INI/YAML）；Web API（→ JSON）；本 vault 笔记（→ Markdown）。

## 1.3 Python 解析

```python
import xml.etree.ElementTree as ET

tree = ET.parse("config.xml")
root = tree.getroot()

for dev in root.findall("device"):
    dev_id = dev.get("id")
    name = dev.find("name")
    if name is not None:
        print(dev_id, name.text)

# 写入
cfg = ET.Element("root")
ET.SubElement(cfg, "device", id="cam01").text = "main"
ET.ElementTree(cfg).write("out.xml", encoding="utf-8", xml_declaration=True)
```

| API | 用途 |
|-----|------|
| `ET.parse` / `fromstring` | 读文件 / 字符串 |
| `find` / `findall` | 子元素查询 |
| `ElementTree.write` | 写回 |

## 1.4 C++ 解析

常用 **tinyxml2** 或 **pugixml**（轻量 DOM）。

```cpp
#include "pugixml.hpp"
#include <iostream>

int main() {
    pugi::xml_document doc;
    if (!doc.load_file("config.xml")) return 1;

    for (pugi::xml_node dev : doc.child("root").children("device")) {
        std::cout << dev.attribute("id").as_string()
                  << " " << dev.child("name").text().as_string() << '\n';
    }
    return 0;
}
```

## 1.5 XML vs INI vs YAML

| 维度 | XML | INI | YAML |
|------|-----|-----|------|
| 嵌套 | 标签 | 弱 | 缩进 |
| 注释 | 支持 | 支持 | 支持 |
| 人工可读 | 中 | 高（简单时） | 高 |
| Schema | XSD 成熟 | 无 | 部分工具 |
