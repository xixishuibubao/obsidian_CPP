# 1. YAML 语法

## 【核心精简提纲】

### 记忆主干

YAML（YAML Ain't Markup Language）是一种 缩进表示层级 的数据序列化格式，可读性高，广泛用于 DevOps、CI/CD、容器编排与工具配置。与 7. INI语法（扁平段键值）互补。

### 核心要点清单

- **结构**：2 空格缩进 map；`-` 块式列表；`key:` 后必须有空格
- **标量**：`|` 保留换行、`>` 折叠；引号处理特殊字符
- **复用**：`&anchor` / `*ref` / `<<: *merge` 模板去重
- **Python**：`safe_load`（禁对不可信输入用 `load`）
- **C++**：yaml-cpp `YAML::LoadFile`
- **坑**：Tab 缩进、冒号后缺空格、`yes/no` 布尔歧义

### 学习路径

1. 掌握缩进 map + `-` 列表
2. Python `safe_load` / C++ yaml-cpp
3. 写一份 GitHub Actions `steps` 片段
4. 对照 [[G-语言与标记/7. INI语法|INI]] 理解何时不用 YAML

### 关键结论 / 流程

```plain
CI/K8s/Compose → YAML
板端十几键配置 → INI（本项目 server.ini）
```

## 【完整拓展详情】

YAML（YAML Ain't Markup Language）是一种 **缩进表示层级** 的数据序列化格式，可读性高，广泛用于 DevOps、CI/CD、容器编排与工具配置。与 [[G-语言与标记/7. INI语法|INI]]（扁平段键值）互补。

> 本 vault 的 Markdown 笔记 **不使用 YAML frontmatter**（`---` 元数据块），但 CI、K8s、Compose 等工程文件大量使用 YAML。

## 1.1 基本结构

YAML 用 **缩进（通常 2 空格）** 表示嵌套，**不用 Tab**（避免解析歧义）。

```yaml
# 整行注释
app:
  name: telescope-server
  port: 8999
  debug: false

database:
  host: 127.0.0.1
  port: 5432
  credentials:
    user: admin
    password: secret
```

| 元素 | 语法 | 说明 |
|------|------|------|
| 键值对 | `key: value` | 冒号后必须有空格 |
| 嵌套 | 子键多缩进一级 | 同级对齐 |
| 注释 | `#` 至行尾 | |
| 空值 | `key:` 或 `key: null` / `~` | |

## 1.2 标量类型

```yaml
integer: 42
float: 3.14
boolean: true      # 也支持 yes/no, on/off（YAML 1.1 遗留，慎用）
string_plain: hello
string_quoted: "含: 冒号或 # 时建议引号"
multiline: |
  保留换行
  第二行
folded: >
  折行会变成
  单行空格连接
```

| 写法 | 含义 |
|------|------|
| `"双引号"` | 支持 `\n` 等转义 |
| `'单引号'` | 字面量，不转义 |
| `\|` | 保留换行块 |
| `>` | 折叠换行块 |

## 1.3 列表（数组）

```yaml
# 块式列表
ports:
  - 8999
  - 9000

# 行内列表
tags: [api, embedded, linux]

steps:
  - name: build
    run: cmake --build build
  - name: test
    run: ctest
```

## 1.4 锚点与引用（复用）

```yaml
defaults: &defaults
  retries: 3
  timeout: 30

job_a:
  <<: *defaults
  name: deploy

job_b:
  <<: *defaults
  name: rollback
```

`&anchor` 定义锚点，`*anchor` 引用，`**` 合并到当前 map — 常用于 CI 模板去重。

## 1.5 常用场景

| 场景 | 典型文件 | 为何用 YAML |
|------|----------|-------------|
| **CI/CD** | `.github/workflows/*.yml`、`.gitlab-ci.yml` | 步骤列表 + 嵌套 env/matrix |
| **容器编排** | `docker-compose.yml` | 多服务、网络、卷声明 |
| **Kubernetes** | Deployment / Service / ConfigMap | 集群事实标准 |
| **配置管理** | Ansible playbook、Helm `values.yaml` | 变量 + 任务流 |
| **静态分析 / 工具** | `.clang-tidy`、部分 linter 配置 | 规则树状结构 |
| **API / Spec** | OpenAPI（可用 YAML 或 JSON） | 人类可读 Spec |
| **AI / Agent** | 部分 workflow、MCP 配置 | 结构化步骤与参数 |

**不太适合 YAML 的场景**：

- 仅需十几个键的嵌入式配置（[[G-语言与标记/7. INI语法|INI]] 更简单）
- 强 Schema 交换、高性能解析（JSON 更普遍）
- 含大量二进制或极深嵌套且需严格类型（JSON + Schema）

**本项目对照**：

| 配置类型 | 格式 | 示例 |
|----------|------|------|
| 运行时服务/硬件 | INI | `server.ini`、`hardware.ini` |
| 构建/CI（若引入） | YAML | GitHub Actions workflow |

## 1.6 Python 解析

依赖：`pip install pyyaml`（导入名为 `yaml`）。

### 1.6.1 读取：标量与嵌套

示例 `config.yaml` 见 1.1 节。完整读取：

```python
import yaml
from pathlib import Path

path = Path("config.yaml")
with path.open(encoding="utf-8") as f:
    data = yaml.safe_load(f)  # 根节点通常是 dict

app_name = data["app"]["name"]
port = data["app"]["port"]           # 已是 int
db_user = data["database"]["credentials"]["user"]

# 列表（如 CI steps）
for step in data.get("steps", []):
    print(step["name"], step["run"])
```

### 1.6.2 读取：多文档流

一个文件内用 `---` 分隔多段 YAML 时：

```python
with open("multi.yaml", encoding="utf-8") as f:
    for doc in yaml.safe_load_all(f):
        if doc:
            print(doc)
```

### 1.6.3 写入

```python
import yaml

data = {
    "app": {"name": "telescope-server", "port": 8999},
    "tags": ["api", "linux"],
}

with open("out.yaml", "w", encoding="utf-8") as f:
    yaml.safe_dump(data, f, allow_unicode=True, sort_keys=False)
```

| API | 用途 |
|-----|------|
| `yaml.safe_load` | **推荐**；解析单文档 |
| `yaml.safe_load_all` | 连续多文档 |
| `yaml.safe_dump` | 写回 YAML |
| `yaml.load` | **勿用于不可信输入**（可反序列化任意对象） |

## 1.7 C++ 解析

常用 **yaml-cpp**（`sudo apt install libyaml-cpp-dev` 或 vcpkg `yaml-cpp`）。

### 1.7.1 读取

```cpp
#include <iostream>
#include <yaml-cpp/yaml.h>

int main() {
    YAML::Node root = YAML::LoadFile("config.yaml");

    std::string name = root["app"]["name"].as<std::string>();
    int port = root["app"]["port"].as<int>();

    // 列表
    for (const auto& step : root["steps"]) {
        std::cout << step["name"].as<std::string>()
                  << ": " << step["run"].as<std::string>() << '\n';
    }

    // 键不存在时 as<T>() 抛 YAML::Exception；可先 IsDefined()
    if (root["database"]["host"]) {
        std::cout << root["database"]["host"].as<std::string>() << '\n';
    }
    return 0;
}
```

`CMakeLists.txt`：

```cmake
find_package(yaml-cpp REQUIRED)
add_executable(read_yaml main.cpp)
target_link_libraries(read_yaml PRIVATE yaml-cpp)
```

### 1.7.2 写入

```cpp
#include <fstream>
#include <yaml-cpp/yaml.h>

YAML::Node root;
root["app"]["name"] = "telescope-server";
root["app"]["port"] = 8999;
root["tags"].push_back("api");
root["tags"].push_back("linux");

std::ofstream fout("out.yaml");
fout << root;
```

### 1.7.3 库选型

| 库 | 说明 |
|----|------|
| **yaml-cpp** | C++ 最常用，API 直观 |
| **libyaml** | C 库，yaml-cpp 底层依赖之一 |
| **rapidyaml** | 高性能，适合大文件 |

## 1.8 YAML vs INI vs JSON

| 维度 | YAML | INI | JSON |
|------|------|-----|------|
| 可读性 | 高 | 高（简单时） | 中 |
| 嵌套 / 列表 | 原生 | 弱 | 原生 |
| 注释 | 支持 | 支持 | 不支持 |
| 缩进敏感 | 是 | 否 | 否 |
| 典型用途 | DevOps、K8s | 应用 `.ini` | API、配置文件 |

生产环境避免对不可信 YAML 使用 `yaml.load`；CI 配置也应审查第三方 workflow。

## 1.9 常见坑

1. **缩进混用 Tab 与空格** → 解析失败或 silent 错误
2. **冒号后缺空格** — `key:value` 可能被当成字符串而非键值
3. **`yes`/`no`/`on`/`off`** — YAML 1.1 当布尔；1.2 更严格，建议显式 `true`/`false`
4. **特殊字符未引号** — `#`、`:`、`{`、`}` 等需引号包裹
5. **大文件重复** — 用锚点 `&` / `*` 或拆多文件 + `import`（工具相关）
