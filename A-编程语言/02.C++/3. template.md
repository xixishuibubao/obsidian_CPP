# 1. 模板

C++ 模板别名、类/函数特化、可变参模板、元编程与 `enable_shared_from_this` 专题速查。

## 【核心精简提纲】

### 记忆主干

**`using` 可模板别名 · `typedef` 须完全实例化 · 类偏/全特化 · 函数无偏特化靠重载 · 参数包 `typename...` · C++17 折叠表达式 · 模板元编译期求值**

### 核心要点清单

- `using Alias = T` 可配合 `template<typename T> using X = MyClass<T,char>`；`typedef` 只能别名已确定类型
- 模板类/函数定义须在头文件、函数外；成员后缀 `_` 为用户惯例，标准库以下划线开头
- 类偏特化 `template<typename T> class C<T*,int>`；全特化 `template<> class C<double,double>`
- 函数模板无偏特化；用重载或显式全特化 `template<> int f<int,int>(int)`
- 可变参：`template<typename... Args>`；C++17 折叠 `(cout << ... << args)` 须加括号
- `decltype` 编译期推导；`typeof` 为 GCC 扩展非标准
- `MyClass obj()` 是函数声明而非无参构造；转换运算符 `explicit` 防隐式转换
- 模板元：非类型模板参数须编译期常量；`enum { value = ... }` 便于编译期求值

## 1. using和typedef

- C++11 引入的 `using` 可代替 `typedef`，且可为尚未完全实例化的模板类型起别名；`typedef` 只能为类型明确的别名。
- 模板定义必须在函数外。
- 成员类型和变量下划线后置是用户代码的典型做法，标准库则是以下划线开头。

```C++
template<typename T, typename U>
class MyClass {
public:
    T                     t_;
    U                     u_;
    typedef MyClass<T, U> Base_;      /* 这属于完全实例化 */
    typedef T             value_type; /* 这属于完全实例化 */
    typedef value_type*   p_value_type;
};

/* using可以不完全实例化，需要template*/
template<typename T>
using type1 = MyClass<T, char>;
/* typedef 只能完全实例化，不需要template*/
typedef MyClass<int, char> type2;
/* 完全实例化，不需要template*/
using MyType = MyClass<int, char>;
int main() {
    MyType         obj1;
    MyType::Base_  obj2;
    typeof(MyType) obj3; /* 这是一个gcc的拓展，并非C++标准特性 */
    decltype(obj3) obj4;
    return 0;
}
```

## 2. 偏特化，全特化

```C++
/* 模板参数在声明处只是占位符，模板类内部是什么，取决于你的class后面的特化/实例化参数 */
template<typename T,typename U>
class MyClass
{
using _MyType1 = T;
using _MyType2 = U;
};
/* 偏特化指特化部分模板参数，全特化，指明确所有模板参数，譬如int或者自定数据类型 */
/* 偏特化，T& T&& 同理 */
template<typename T>
class MyClass<T*,int>
{
using _MyType1 = T*;
using _MyType2 = int;
};

template<>
class MyClass<double,double>
{
using _MyType1 = double;
using _MyType2 = double;
};

/* 函数的特化同理,但是函数没有偏特化，需要靠重载实现类似偏特化特性*/
template<typename T,typename U>
T func(U);

template<typename T>
T func(std::string);

template<>
int func<int,int>(int);

func<int,double>(3.14); /* 调用通用模板 */
func<std::string>(std::string("test"));/* 调用重载模拟的偏特化 */
func<int,int>(1); /* 调用全特化 */
```

`decltype` 在编译期推导表达式类型。

## 3. 类与结构体

本质都是 UDT（user-defined type），可以互相继承；`struct` 默认 `public`，`class` 默认 `private`。

## 4. enable_shared_from_this<>

C++11 在 `<memory>` 中提供：在对象已被 `shared_ptr` 管理时，安全返回指向自身的 `shared_ptr`。须在堆上创建且先由 `shared_ptr` 持有，否则 `shared_from_this()` 未定义。

> 拼写注意：标准名为 `enable_shared_from_this`（非 `form`）。

## 5. 特殊的成员重载

仿函数：重载 `()` 操作符 `auto operator()(形参列表){ }`

重载转换函数不需要额外声明返回值：`explicit operator int() const { return 1; }`（推荐 `explicit`）。

Most Vexing Parse：无参构造不可写 `MyClass obj();`，应写 `MyClass obj;` 或 `MyClass obj{};`。

## 6. 参数包

C++11 可变参数模板；C++17 折叠表达式须加括号。

```cpp
template <typename... Args>
class MyClass {
public:
    void func(Args... args) {
        (std::cout << ... << args);
    }
};
```

## 7. 模板元编程

模板元编程在编译期求值；非类型模板参数 `<>` 内必须是编译期常量，不能是运行时变量。

```cpp
template <int N>
struct Factorial {
    enum { value = N * Factorial<N - 1>::value };
};

template <>
struct Factorial<0> {
    enum { value = 1 };
};

const int fib = Factorial<10>::value;
```

使用 `enum` 是因为枚举可在编译期确定值；`static const` 亦可但需类外定义，不如 `enum` 简洁：

```cpp
template <int N>
struct Factorial {
    static const int value;
};

template <int N>
const int Factorial<N>::value = N * Factorial<N - 1>::value;

template <>
const int Factorial<0>::value = 1;
```
