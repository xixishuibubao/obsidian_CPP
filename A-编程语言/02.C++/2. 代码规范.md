# 1. 命名规范

团队 C/C++ 代码风格：命名、缩进、声明、类设计、类型转换与 C++11 资源管理约定。

## 【核心精简提纲】

### 记忆主干

**类型大驼峰 · 变量/参数小驼峰 · 宏全大写+括号保护 · 常量/枚举 `k` 前缀 · 四空格缩进 · 常量在比较左侧 · 头文件自包含 · `explicit`/`override`/`=delete` · 优先 `unique_ptr`**

### 核心要点清单

- 命名：类型大驼峰；变量/参数小驼峰；`namespace` 全小写；宏 `UPPER_SNAKE`；常量/枚举 `kCamelCase`
- 格式：四空格；`{` 同行；复合语句必带 `{}`；`switch` 必有 `default`；`case` 内局部变量加 `{}`
- 声明：静态/全局不显式初值 0/NULL；`<stdint.h>` 定长整型；`const` 形参；`sizeof` 代替魔法数
- 比较：常量在左 `if (NULL != ptr)`，防 `=` 误写；`bool` 用 `true`/`false`
- 类：头文件自包含；依赖延迟到 `.cpp`；单参构造 `explicit`；父类析构 `virtual`；三五成对出现或 `= delete`
- 转换：优先枚举参数；`dynamic_cast` 下行、`static_cast` 基本/上行；避免 `reinterpret_cast`
- C++11：`nullptr`；`override`；`using` 别名；`make_unique`/`make_shared`；禁 `move(const T&)`
- 内存：`new`/`delete[]` 配对；RAII；勿长期持有 `c_str()` 指针

## 【完整拓展详情】

## 1.1 命名规则

1. 自定义类型使用大驼峰。
2. 变量和参数使用小驼峰。
3. `namespace` 全小写。
4. 宏使用大写加下划线。
5. 常量和枚举使用 `k` 小驼峰。

## 2. 缩进、对齐、花括号

1. 使用四个空格代替一个 tab 制表符。
2. 在操作符之间加入一个空格，函数与左括号无空格，每个形参和 `,` 后加一个空格：`int res = sum(2, 3);`
3. 左花括号总是跟在语句或函数名同一行：

```cpp
if (NULL != point) {
}

void func() {
}
```

4. 空实现放在函数名同一行：`void func() { }`
5. 在比较运算中，将常量（枚举、宏定义、`NULL`）放在左侧，避免 `==` 误写为 `=`。
6. 每个复合语句必须包含 `{}`，即使只有一条语句或空语句。`else` 与 `if` 的 `}` 在同一行。`do-while`、`for` 同理。
7. `switch-case` 逐级缩进，为每个 `case` 添加缩进。总是包含 `default` 分支。当 `case` 内需要使用局部变量时，使用 `{}` 包裹。
8. 换行时让运算符号或逗号在行尾。
9. 每行只初始化一个变量。

## 3. 声明与定义

1. 不要初始化静态和全局变量为 `0` 或 `NULL`，让编译器做。
2. 在同一行定义和声明所有相同类型的局部变量。
3. 当存在多个类型的变量时，按照先自定义类型（枚举、结构体、联合体），后整型（无符号优先），最后浮点类型的顺序定义：

```cpp
typedef struct {
    int age;
    char* name;
} person_t, *person_pt;

int main(void) {
    person_t person1;
    uint16_t u16num;
    float fnum1;
}
```

4. 总是在块的开头声明局部变量，在第一个可执行语句之前。
5. 避免变量声明时通过函数调用赋值，采用先声明后赋值的方式。
6. 除 `char`、`double`、`float` 外，使用 `<stdint.h>` 的带长度整型类型（如 `uint8_t`）。
7. 不要直接使用函数或表达式结果做条件，应和对应内容比较。对于 `bool` 类型使用 `true`/`false` 或 `!`。
   - 错误：`if(ptr)` `if(a)`
   - 正确：`if(NULL != ptr)` `if(0 != a)`
8. 多使用前增量（减），减少后增量：多用 `++i`，少用 `i++`。
9. 使用 `size_t` 定义长度或大小类变量。
10. 对不应修改的实参使用 `const` 修饰。
11. 对不定类型指针使用 `void*`。
12. 总是使用 `sizeof` 而非魔法数字。
13. 总是使用括号表示优先级，提高可读性。
14. 减少使用变长数组，使用 `malloc` 动态分配。
15. 定义指针时将 `*` 与变量类型对齐：`int* ptr;`。
16. 参数超长时换行，可缩进 4 空格，或让每个参数单独一行。

## 4. 结构体与枚举

1. 未使用 `typedef` 的结构体，类型名不应带 `_t`。
2. 有别名时，原生类型名不应带 `_t`，别名应带 `_t`。
3. 结构体成员初始化使用 C99 风格：`= {.membera = a, .memberb = b};`

## 5. 预编译与注释

1. 宏使用大写字母和下划线组合，使用 `()` 保护输入参数和最终结果。
2. 宏使用多个语句时，使用 `do-while(0)` 形式。
3. 使用 `#if` 和 `#else` 时，不必像 `if-else` 那样缩进，全部顶格。在 `#endif` 处注释对应的 `#if`。
4. 总是对 `__cplusplus` 做检查：

```cpp
#ifdef __cplusplus
extern "C" {
#endif /* __cplusplus */

/* File content here */

#ifdef __cplusplus
}
#endif /* __cplusplus */
```

5. 总是使用 `/* */` 注释。多行注释开头使用空格 `*` 对齐。
6. 与 `#` 相关的预处理命令不缩进，一律放在行首。

## 6. 类的设计规范

1. 类的构造函数类内声明、类外实现时，参数列表可以只在实现处写出。
2. 头文件应当自包含，可独立编译。将依赖在其他 `.cpp` 中包含。
   - 配合前向声明，在 `.h` 声明依赖类型，在 `.cpp` 包含实际依赖的 `.h`。
   - 最好将依赖设计成类内的成员指针，减少编译时间。
3. 标准库可使用延迟包含，不在 `.h` 包含，只在 `.cpp` 包含。
4. `namespace {}` 匿名空间具有内链属性，只在当前文件可见，类似于 `static`。
5. 使用 `explicit` 修饰单参数构造函数，避免意外隐式转化。
6. 对不需要拷贝构造和赋值操作符的类，可 `= delete` 禁用。
7. `constexpr` 修饰函数表示编译期可求值。接收返回值的变量也应用 `constexpr` 修饰。
8. `final` 表示不可继承或不可重写：`class A final { };`，`void func() final override { }`。
9. 虚继承 `virtual` 可解决多继承时的菱形继承问题：`class D : virtual public A { };`。
10. 拷贝构造/赋值、移动构造/赋值，应成对出现或成对禁止。
11. 禁止在构造函数中调用虚函数。构造顺序：先父类后子类；析构相反。
12. 父类析构函数应声明为虚函数，避免内存泄漏。
13. 禁止虚函数使用缺省默认值（默认参数）。
14. 禁止重定义父类非虚函数。

## 7. 类型转化与参数

1. 优先使用枚举作为参数，而非 `#define`。
2. C++ 类型转化：
   - `dynamic_cast<类型>(变量)`：父类指针→子类指针，失败返回 `nullptr`。依赖 RTTI，编译期禁用 RTTI（如 `-fno-rtti`）则不可用。
   - `static_cast<类型>(变量)`：基本类型间转换、子类→父类指针。不进行类型检查。
   - `reinterpret_cast<类型>(变量)`：底层不安全的类型转换，应尽量避免。
   - `const_cast<类型>(变量)`：去除/添加 `const` 或 `volatile` 属性。

## 8. C++11 以后特性

1. 使用 `nullptr` 而非 `NULL`。
2. 使用 `= delete` 删除不需要的方法。
3. 子类重写虚函数使用 `override`。
4. 使用 `using` 而非 `typedef` 定义别名，`using` 可对模板类型别名。
5. 禁止使用 `std::move` 操作 `const` 变量。
6. 优先使用 `unique_ptr` 而非 `shared_ptr`。
7. 使用 `make_unique` 创建 `unique_ptr`，使用 `make_shared` 创建 `shared_ptr`。
8. 自定义 deleter 时不应使用 `make_`，应自定义。
9. Lambda 表达式注意捕获变量的生命周期，避免引用捕获导致垂悬引用。能用函数优先用函数。
10. 所有权与形参：
    - 不涉及所有权时使用 `T*` 或 `T&` 做形参。
    - 涉及所有权时使用 `unique_ptr<T>` 等智能指针类型。

## 9. 内存管理与其他

1. `new` 和 `delete`、`new[]` 和 `delete[]` 应成对出现。
2. 充分利用 RAII，构造获取资源，析构释放资源。
3. 不要保存 `string::c_str()` 返回的指针，其生命周期不确定。
4. `noexcept` 修饰函数可让编译器更大程度优化。
