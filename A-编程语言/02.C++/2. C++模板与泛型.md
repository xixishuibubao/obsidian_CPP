# 2. C++ 模板与泛型

模板别名、类/函数特化、CRTP、可变参模板、折叠表达式与模板元编程专题速查。

## 【核心精简提纲】

### 记忆主干

**`using` 可模板别名 · `typedef` 须完全实例化 · 类偏/全特化 · 函数无偏特化靠重载 · CRTP 静态多态 · 参数包 `typename...` · C++17 折叠表达式 · 模板元编译期求值**

### 核心要点清单

- `using Alias = T` 可配合 `template<typename T> using X = MyClass<T,char>`；`typedef` 只能别名已确定类型
- 模板类/函数定义须在头文件、函数外
- 类偏特化 `template<typename T> class C<T*,int>`；全特化 `template<> class C<double,double>`
- 函数模板无偏特化；用重载或显式全特化 `template<> int f<int,int>(int)`
- CRTP：`class B : public A<B>` 实现静态多态，无虚函数开销
- 可变参：`template<typename... Args>`；C++17 折叠 `(cout << ... << args)` 须加括号
- `decltype` 编译期推导；配合 auto 做后置返回类型
- 模板元：非类型模板参数须编译期常量；`enum { value = ... }` 便于编译期求值
- `enable_shared_from_this`：在对象已被 `shared_ptr` 管理时安全返回 `shared_ptr`
- 仿函数：重载 `()`；转换运算符推荐加 `explicit`

## 1. using 与 typedef

C++11 `using` 可代替 `typedef`，且可为尚未完全实例化的模板类型起别名；`typedef` 只能为类型明确的别名。

```cpp
template<typename T, typename U>
class MyClass {
public:
    T t_;
    U u_;
    typedef MyClass<T, U> Base_;     // 完全实例化
    typedef T value_type;            // 完全实例化
};

// using 可不完全实例化，需要 template
template<typename T>
using type1 = MyClass<T, char>;

// typedef 只能完全实例化
typedef MyClass<int, char> type2;

// using 完全实例化（不需要 template）
using MyType = MyClass<int, char>;
```

## 2. 类模板

### 2.1 泛型实现

```cpp
template<typename T>
class ProxyCast {
private:
    T* p_;
public:
    explicit ProxyCast(void* p) : p_(static_cast<T*>(p)) {}
    T* operator->() const { return p_; }
    T* get() const { return p_; }
    T& operator*() const { return *p_; }
};
```

### 2.2 偏特化与全特化

```cpp
/* 泛型 */
template<typename T1, typename T2>
class Test { };

/* 偏特化：部分模板参数预填充 */
template<typename T>
class Test<int, T> { };

/* 全特化：完全填充模板参数 */
template<>
class Test<int, int> { };

/* 指针/引用特化 */
template<typename T1, typename T2>
class A<T1*, T2*> { };

template<typename T1, typename T2>
class A<T1&, T2&> { };

/* 最优匹配：全特化 > 偏特化 > 泛型 */
```

### 2.3 CRTP 奇异递归模板

CRTP 实现静态多态，无虚函数开销。

```cpp
template<typename T>
class Base {
public:
    void func_a() {
        static_cast<T*>(this)->func();
    }
};

class Derived : public Base<Derived> {
    friend class Base<Derived>;
    void func() { cout << "Derived::func()" << endl; }
};
```

## 3. 函数模板

### 3.1 泛型与重载

函数模板无偏特化，通过重载实现类似效果。

```cpp
template<typename T1, typename T2>
T1 func(T1 t1, T2 t2) { return t1; }    // 泛型

template<typename T>
T func(T t1, int t2) { return t1; }     // 重载（模拟偏特化）

template<>
char func(char t1, int t2) { return t1; }  // 全特化

func<int,double>(3.14);                 // 调用泛型
func<std::string>(string("test"));      // 调用重载
func('a', 1);                           // 调用全特化（T=char 自动推导）
```

### 3.2 最优匹配原则

```
<1> 完全匹配：常规函数优于模板
<2> 提升转换：char/short → int，float → double
<3> 标准转换：int → char，long → double
<4> 用户声明的类型转换

全特化 > 偏特化 > 泛型模板
```

## 4. 可变参模板与折叠表达式

C++11 引入可变参数模板；C++17 折叠表达式须加括号。

```cpp
template <typename... Args>
class MyClass {
public:
    void func(Args... args) {
        (std::cout << ... << args);   // C++17 折叠表达式
    }
};
```

## 5. 模板元编程

### 5.1 编译期求值

非类型模板参数必须是编译期常量。

```cpp
template <int N>
struct Factorial {
    enum { value = N * Factorial<N - 1>::value };
};

template <>
struct Factorial<0> {
    enum { value = 1 };
};

const int fib = Factorial<10>::value;
```

使用 `enum` 是因为枚举可在编译期确定值。`static const` 亦可但需类外定义。

```cpp
template <int N>
struct Factorial {
    static const int value = N * Factorial<N - 1>::value;
};
```

### 5.2 decltype 类型推导

C++11 引入，常配合 auto 做后置返回类型。

```cpp
int a;
decltype(a) b;             // b 的类型为 int

// decltype 内加 () 变为引用
decltype((a)) c = b;       // c 为 int&

// 后置返回类型，适用于多模板场景
template<class F, class... Args>
auto enqueue(F&& f, Args&&... args)
    -> std::future<typename std::result_of<F(Args...)>::type>;
```

## 6. 特殊技巧

### 6.1 enable_shared_from_this

C++11 `<memory>` 提供：在对象已被 `shared_ptr` 管理时安全返回 `shared_ptr`。须在堆上创建且先由 `shared_ptr` 持有，否则未定义。

> 拼写注意：标准名为 `enable_shared_from_this`（非 `form`）。

### 6.2 仿函数

重载 `()` 操作符：

```cpp
struct Functor {
    auto operator()(int x) { return x * 2; }
};
```

### 6.3 转换运算符

```cpp
class A {
    int a_;
public:
    explicit A(int a) : a_(a) {}
    explicit operator int() const { return a_; }  // 推荐 explicit
};
```

### 6.4 Most Vexing Parse

无参构造不可写 `MyClass obj();`（被识别为函数声明），应写 `MyClass obj;` 或 `MyClass obj{};`。
