### 核心要点清单

- OOP：封装隐藏实现、继承复用接口、多态解耦调用方与实现
- `class` 默认 `private`，`struct` 默认 `public`；二者可互继承
- Rule 0：不管理资源则不自定义三五；Rule 3：拷贝构造/拷贝赋值/析构；Rule 5：加移动构造/移动赋值
- 显式定义拷贝或析构会抑制编译器自动生成移动；`= default` / `= delete` 可显式控制
- 多继承注意菱形；`virtual` 继承共享基类子对象；单继承仅一张虚表
- 父类析构须虚函数；构造中禁调虚函数；默认参数不要跨虚函数层传递
- `dynamic_cast` 依赖 RTTI 与虚函数；`static_cast` 编译期；`reinterpret_cast` 慎用

## 2. 类

### 2.1 Rule 0/3/5

```cpp
/*
<1> 0：尽量使用容器，避免手动管理内存。
<2> 3：拷贝构造函数，拷贝赋值函数，析构函数。
<3> 5：在 3 的基础上，增加移动构造函数和移动赋值函数。
     拷贝即形参为 const classname&，
     移动则为 classname&&，移动不带 const（否则移动时无法释放原有资源）
<4> 重载等号操作符一般返回原来对象的引用

注：
--- 拷贝构造，拷贝赋值，移动构造，移动赋值算作一组 A 类方法 ---
// = delete 也算自定义了该方法。
<1> 定义显式带参构造函数后，编译器不再生成默认无参构造。
<2> 定义移动或拷贝函数后，剩余 A 类方法不会自动生成默认实现，
    但仍允许使用 = default 获取默认实现。
<3> 定义析构函数后，不会自动生成移动相关两个方法。
*/
```

### 2.2 继承与访问

```cpp
/* public / protected / private
<1> public：基类、子类、外部都可直接访问。
<2> protected：基类、子类可访问；外部不可直接访问。
<3> private：只能被类自己直接访问。
<4> friend：友元访问权限等同基类自身，不被继承或传递。

继承体系：
- class 默认权限私有，struct 默认权限公有。
- struct 和 class 可相互继承。

单继承：所有函数放在同一虚表，最终只有一个虚表。
多继承：每继承一个就多一个虚表；注意菱形继承。
虚继承：用 virtual 解决菱形继承，祖父类用一次 virtual 即可。
*/
```

### 2.3 构造的细节

构造函数不可作为成员函数调用，但析构在特殊场景可手动调用（如定位 new）。

```cpp
A* val = new(arr) A; 
val->~A();
```

#### 显式构造

子类需显式调用父类构造的场景：父类无默认构造、父类构造非 public、虚继承、父类抽象类、父类构造 explicit。

```cpp
class A { public: explicit A(int a) {}; };
```

#### 初始化列表

- 初始化顺序与成员声明顺序有关，与列表顺序无关
- const 成员和引用成员必须使用初始化列表初始化
- 列表初始化效率比在 {} 内赋值高

```cpp
class A {
    int a_;
    int b_;
public:
    explicit A(int a, int b) : a_(a), b_(b) { }
};
```

### 2.4 析构

自动变量（局部/临时变量）销毁顺序与创建相反（栈的先入后出）。

手动析构通常用于定位 new 构造的场景。

```cpp
A* a = new(arr) A();
a->~A();
```

### 2.5 深浅拷贝

深拷贝会为动态内存重新申请空间，浅拷贝不会。编译器提供的默认构造是浅拷贝。

```cpp
/* 深拷贝需要：在拷贝构造函数中 new 新内存并复制数据 */
```

### 2.6 虚函数与虚析构

```cpp
class A {
public:
    virtual void func() {}
    virtual ~A() = default;  // 父类析构应声明为虚函数
};

class B : public A {
public:
    void func() override {}
};
```

父类指针指向子类时，若无虚析构，子类析构不会被调用，导致内存泄漏。

### 2.7 this 指针

- 处理成员变量与参数同名：`this->member = member`
- 返回自身引用实现链式调用：`return *this`
- 重载 `=` 时需判断 `this == &value`

### 2.8 同名变量

成员函数优先查找类内成员，无同名时才调用外部全局。推荐显式使用 `::a = 5`。

### 2.9 mutable 与 const

```cpp
/* const 后置：标识函数不修改成员变量，const 对象只可调用 const 方法
   const 后置也是签名标识符，可与非 const 版本构成重载 */
class A {
    mutable int c{1};  // const 方法中可修改
    void func() const { c++; }
};
```

### 2.10 静态成员

- 静态成员不与类对象绑定，只能操作同样静态的成员
- C++11 起可用 `inline static` 在类内定义

```cpp
class A { inline static int c = 1; };
class B { static int c; };
int B::c = 1;  // C++11 前需类外定义
```

### 2.11 友元

- 友元只针对当前类/函数，不会被继承
- 类的成员函数是自身类型的友元，可访问形参的私有成员
- 重载 `<<` 推荐使用友元全局函数

```cpp
class A {
    friend void fff(const A*);
    friend class B;
private:
    int val{2};
};

std::ostream& operator<<(std::ostream& os, const A& a) { }
```

### 2.12 final

```cpp
class A final { };        // A 不可被继承
class C : public B {
    void func() override final;  // func 不可再被子类重写
};
```

### 2.13 前向声明

```cpp
class A;    // 可使用 A* 或 A&，但不能访问成员
```

- 保证编译通过，一般用于模板定义场景
- 只适合数据类型，不适合变量

### 2.14 RAII

Resource Acquisition Is Initialization：构造函数获取资源，析构函数释放资源。

### 2.15 RTTI

```cpp
typeid(val).name();  // 运行时获取类型信息（有性能损耗）
```

### 2.16 PIMPL（编译防火墙）

Pointer to Implementation，避免暴露实现细节、减少编译依赖。详见：

### 2.17 类型转化

```cpp
// static_cast：编译期转换，基本类型、无继承关系的指针
// dynamic_cast：运行时转换，父类子类指针，需父类有虚表，失败返回 nullptr
// const_cast：去除 const/volatile
// reinterpret_cast：底层字节级转换，慎用
int a = static_cast<float>(b);
```

### 2.18 初始化

```cpp
// C++11 引入统一初始化 {}，底层依赖 std::initializer_list
int a = 1;
int a(1);
int a{1};              // 统一初始化

class A {
    int a_{1};         // 成员默认值
    int c_ = 3;
};
```
