# 1. 管道

Linux 用户态 IPC 与 fd 操作速查：无名/有名管道、`dup`/`dup2`、`mmap`/`msync`。

## 【核心精简提纲】

### 记忆主干

**`pipe(fd[2])` 半双工仅亲缘进程 · `mkfifo` 路径名 FIFO 任意进程 · `dup`/`dup2` 复制 fd 表项 · `mmap` 映射后 fd 可关 · `prot` ≤ open 权限 · 空文件须先扩长**

### 核心要点清单

- 无名管道：`pipe(fd)`，`fd[0]` 读、`fd[1]` 写；仅父子/兄弟等亲缘进程
- 有名管道：`mkfifo(path, mode)` + `open`；阻塞直到对端打开（常见踩坑）
- `dup(fd)` 复制到最小可用编号；`dup2(old,new)` 指定目标 fd，会关闭原占用
- `mmap(NULL, len, prot, flags, fd, off)`；`MAP_SHARED` 写回文件、`MAP_PRIVATE` 写时复制
- `prot` 读写权限不得超过 `open` 时权限；映射零长文件失败，须 `truncate`/`ftruncate` 或 `lseek`+`write` 扩长
- 映射成功后即可 `close(fd)`，映射区仍有效；共享写后 `msync` 刷盘（`MS_SYNC`/`MS_ASYNC`）

## 【完整拓展详情】


## 1.1 无名管道（只能用于父子或兄弟进程）

半双工、字节流；写端未关闭时读端可能一直阻塞；单进程不宜同时留读写两端（易死锁）。

```c
int pipe_fd[2];  // [0] 读端, [1] 写端
pipe(pipe_fd);
```

## 1.2 有名管道（任意进程通信）

路径名 FIFO；`open` 常阻塞直到对端以互补模式打开；权限由 `umask` 与 `mode` 共同决定。

```c
mkfifo("fifo_name", mode);
int fd = open("fifo_name", O_RDONLY);  // 或 O_WRONLY
```

## 2. 文件描述符操作

`dup2` 若 `new_fd` 已打开会先关闭再复制；重定向标准流常用 `dup2(fd, STDOUT_FILENO)`。

```c
fd2 = dup(fd);                // 重定向文件符号
fd2 = dup2(old_fd, new_fd);   // 重定向文件内容
int new_fd = fcntl(old_fd, F_DUPFD, fd_no);  // 等同于 dup()
```

## 3. 内存映射（mmap）

```c
#include <sys/mman.h>

void* p = mmap(NULL, length, prot, flags, fd, offset);
// prot: PROT_READ | PROT_WRITE
// flags: MAP_SHARED / MAP_PRIVATE
munmap(p, length);
```

- mmap 的读写权限需 <= 打开 fd 时的权限
- 文件必须有内容且长度 > 0，否则会出错（`EINVAL`/`SIGBUS` 视平台而定）
- 打开文件并映射后即可关闭 fd
- `MAP_SHARED` 多进程可见同一映射；`MAP_PRIVATE` 写时复制，不影响原文件
- `offset` 通常须页对齐；映射失败返回 `MAP_FAILED`（不是 NULL）

扩展文件大小：

```c
lseek(fd, offset, SEEK_END);
write(fd, "0", 1);

// 或
truncate("filename", size);
ftruncate(fd, size);
```

对 mmap 共享写入后，用 `msync` 同步文件：

```c
int msync(void *addr, size_t length, int flags);
// flags: MS_ASYNC（异步）, MS_SYNC（同步）, MS_INVALIDATE（使缓存失效）
```
