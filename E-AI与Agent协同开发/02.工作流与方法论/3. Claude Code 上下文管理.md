# 1. Claude Code 上下文管理

约 1M token 窗口；**事前过滤**（`.claudeignore`）+ **事后瘦身**（`/compact`）+ **永久规则**（CLAUDE.md Compact Instructions）。

## 【核心精简提纲】

### 记忆主干

大仓库事前用 `.claudeignore` 过滤；对话变长用 `/compact` 与 CLAUDE.md 压缩规则定向瘦身；彻底重来 `/clear`，旁路问答用 `/btw`。

### 机制速查

| 机制 | 时机 | 作用 |
|------|------|------|
| `.claudeignore` | 事前 | 默认不加载指定路径 |
| `/compact` | 事后 | 重写摘要，定向保留/弱化模块 |
| Compact Instructions | 每次压缩 | CLAUDE.md 中固化保留/丢弃规则 |
| `/clear` | 重置 | 彻底清空会话 |
| `/btw` | 旁路 | 问答不入主上下文 |

### 核心要点清单

- **自动 compact**：按时间裁最早内容，不可定向
- **手动 compact**：`只保留 XX 模块，丢掉 YY` / `忽略 static/`
- **局限**：不能删单条消息；要彻底重来用 `/clear`
- **辅助**：`/context` 看 token 分布
- **切换模块**：compact 忘掉上一模块，保留架构骨架

### 学习路径

1. 配 `.claudeignore` 排除 assets/log
2. 在 CLAUDE.md 写 Compact Instructions
3. 切换功能域时练一次定向 compact
4. 配合 [[E-AI与Agent协同开发/02.工作流与方法论/2. AI如何辅助个体成为超级程序员|模块化 PRD]] 减少单次上下文负载

### 关键结论 / 流程

```
大仓库 → claudeignore 前置过滤
对话变长 → /context 诊断 → /compact 定向瘦身
换模块开发 → compact 聚焦新任务
```

## 【完整拓展详情】

## 1.1 核心概念

### 1.1.1 自动压缩 vs 手动压缩

| 类型 | 触发方式 | 特点 |
|------|----------|------|
| **自动压缩** | 系统阈值自动触发 | 按时间先后裁剪最早内容，无法指定保留/丢弃某部分 |
| **手动压缩** | `/compact` 命令 | 可自由提要求，定向保留或移除某部分上下文 |

### 1.1.2 压缩 vs 过滤

| 机制 | 时机 | 作用 |
|------|------|------|
| **/compact** | 事后瘦身 | 控制聊天记录和已加载的文件内容 |
| **.claudeignore** | 事前阻断 | 项目文件默认不读入上下文，属于前置过滤 |

两者配合使用效果最好。

---

## 1.2 /compact 定向压缩

### 1.2.1 保留指定模块，精简其余

```plaintext
/compact 只保留 auth 用户模块相关讨论，把订单模块、旧调试报错全部大幅压缩精简

/compact 重点保留 src/trade 目录的代码修改记录，忽略之前关于静态资源、前端页面的全部细节
```

### 1.2.2 移除冗余上下文

```plaintext
/compact 彻底丢掉之前反复调试数据库报错的全部过程，只保留最终修复方案和 SQL 结构

/compact 移除所有读取 assets/图片、视频文件产生的上下文，仅保留业务代码变更
```

### 1.2.3 多层目录精准控制

```plaintext
/compact 压缩时忽略所有 static/ 子目录文件内容，只保留 src/service 业务逻辑相关信息
```

### 1.2.4 切换开发模块时聚焦新任务

```plaintext
/compact 忘掉之前商品管理模块的全部细节，仅保留项目基础架构，专注支付模块开发
```

### 1.2.5 工作逻辑

手动 /compact 不会直接删除某一段对话，而是重写一份精简摘要：
- 保留内容：完整保留关键细节、文件路径、代码改动
- 移除内容：只留一两句话概括，大量细节被抹掉，大幅节省 token

### 1.2.6 局限性

- 无法做到精准单条删除某一条消息——只能通过压缩摘要做弱化丢弃
- 需要彻底清空全部历史用 `/clear`（完全重置会话）

## 1.3 .claudeignore 前置过滤

`.claudeignore` 是**事前不加载**的机制，与 `/compact` 事后瘦身互补：

```
# 示例：忽略 assets 目录中的资源文件
assets/*
static/*
*.log
*.png
```

配置后，这些文件默认不读入上下文；如果手动读取过，再用 `/compact` 清理已加载的内容。

---

## 1.4 CLAUDE.md 压缩规则（永久生效）

在项目 `CLAUDE.md` 中写入 **Compact Instructions**，每次自动或手动压缩都会遵循：

```markdown
## Compact Instructions

压缩上下文时固定遵循以下规则：

1. 永久保留：src/api、数据库 schema、当前待修改文件、测试用例
2. 强制精简丢弃：static/ 资源文件、旧调试日志、已修复 bug 的排查过程、node_modules 相关读取内容
3. 仅保留最终结论，删除所有反复试错的中间步骤
```

---

## 1.5 补充工具指令

| 指令 | 用途 |
|------|------|
| `/context` | 查看当前加载了哪些文件、token 占用分布，确认需要压缩的内容 |
| `/clear` | 完全清空所有对话历史，无法找回 |
| `/btw 问题` | 临时查询，问答不会写入主上下文，避免新增冗余内容 |

---

## 1.6 场景方案对比

| 需求 | 解决方案 | 效果 |
|------|----------|------|
| 自动快满 token、自动压缩 | 系统自动 compact | 只能删最早内容，无法定向清理无关模块 |
| 切换功能，想丢掉上一模块细节 | `/compact 只保留 XX 模块，丢掉 YY` | 定向弱化无关上下文，保留项目基础 |
| 不想再看到某文件夹内容 | ① .claudeignore 前置过滤 ② /compact 清理已加载的文件上下文 | 彻底杜绝该目录占用 token |
| 完全清空所有对话 | `/clear` | 所有历史全部删除，无法找回 |
