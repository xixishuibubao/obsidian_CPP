## 1.1 本目录职责

最佳实践层沉淀 **可复用的经验与反模式**：何时让 Agent 写测试、如何拆分上下文、vault 维护节奏。概念与工具链分别见 [[E-AI与Agent协同开发/01.基础概念/1. 编程范式演进：Spec Coding → Agentic Engineering → Harness Engineering|编程范式演进]] 与工具索引。

## 1.2 双分区迭代与 vault 维护

知识条目统一按 **薄→厚（补全详情区）→厚→薄（提炼提纲区）** 迭代，格式见 [[.claude/instructions/02-note-conventions|双分区迭代格式]]。维护时：

1. 先扩 **详情区**：案例、踩坑、wikilink
2. 再收敛 **提纲区**：不引入详情区没有的新事实
3. 本地 commit，推远端前 squash（[[F-版本管理/2. log规范|log 规范]]）
