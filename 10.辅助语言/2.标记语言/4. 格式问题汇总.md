# 1. Markdown 格式问题汇总与解决方案

以下是在修复整个仓库过程中发现的常见手写 MD 问题及对应的解决方案。

## 1. 多级标题层级错误

**问题**：部分文件以 `##` 或 `###` 开头（缺少顶层 `#` 标题），或中文编号（一、二、三）直接作为顶级标题。

**示例**：
```markdown
## 1. 缓存问题
### 一、入门篇
### **一、高性能网络库**
```

**解决方案**：补全 `#` 顶层标题，统一采用 `# 1.`、`## 1.1` 的数字编号体系。

```markdown
# 1. 缓存问题
# 1. 垃圾回收算法
## 1. 入门篇
# 1. 开源项目分析
## 1. 高性能网络库
```

---

## 2. 标题中混入加粗标记

**问题**：标题文字中使用 `**` 包裹，或出现不成对的 `**`。

**示例**：
```markdown
### **一、入门篇**
### **项目定位**
# 3. 学习路径建议（适合不同阶段）**
# 4. ** 避坑提示
```

**解决方案**：移除标题标记中的 `**`，需要强调的内容在正文中使用。

```markdown
## 1. 入门篇
### 项目定位
# 3. 学习路径建议（适合不同阶段）
# 4. 避坑提示
```

---

## 3. 代码块嵌套在列表项中

**问题**：代码块标记 ` ``` ``` 与列表标记 `-` 或 `1.` 在同一行，导致渲染异常。

**示例**：
```markdown
1. 格式如下：
   - ```bash
     if expr1; then
         func1;
     fi
     ```
```

**解决方案**：代码块独立缩进，不与列表标记同行。

```markdown
1. 格式如下：

   ```bash
   if expr1; then
       func1;
   fi
   ```
```

---

## 4. 代码块/注释截断与残留

**问题**：代码块未正确闭合，导致注释内容泄漏到后续段落中。

**示例**（Qt.md 中发现）：
```markdown
```C++
/* 重写事件过滤器 */
    return QObject::eventFilter(obj,event);
```

    也不做则代表使用默认过滤器去处理 直接返回false的 */
```
```

**解决方案**：确保代码块内的注释正确闭合（`*/`），代码块标记 ` ``` ` 独占一行且前后有空行。

---

## 5. 不可见字符（非断行空格）

**问题**：编辑器中混入 `U+00A0`（`&nbsp;`）替代普通空格，肉眼不可见但导致格式异常。

**示例**（Clion.md 中发现，每行开头为 `U+00A0`）：
```markdown
  - `Ctrl + Alt + L` 格式化代码
  - `ALt + Enter` 弹出建议
```

**解决方案**：全局搜索替换 `U+00A0` 为普通空格 `U+0020`，或使用编辑器的"显示空白字符"功能检查。

---

## 6. 内容拼写错误

**问题**：英文术语拼写错误，在修复格式时一并修正。

| 错误 | 正确 | 文件 |
|------|------|------|
| `SINGAL` | `SIGNAL` | Qt.md |
| `ture` | `true` | Qt.md |
| `QObect` | `QObject` | Qt.md |
| `bulid` | `build` | 常用英语.md |
| `blcok` | `block` | 常用英语.md |

---

## 7. 文档合并导致的编号冲突

**问题**：两个独立的文档拼接在一起，各自以 `# 1.` 开头，导致编号重复。

**示例**（CMake.md 中发现）：
```markdown
# 1. 基础语法
...
# cmake-practice
# 1. cmake的是显式假为假...
```

**解决方案**：识别拼接点，为第二部分添加独立编号 `# 2.`，子章节相应调整为 `## 2.1`、`## 2.2` 等。

---

## 8. 内容分散但语义接近

**问题**：多个文件中存在语义接近的分散内容。

**示例**：`代码规范.md` 与 `格式规范.md` 都包含 C++ 编码风格相关内容。

**解决方案**：合并到同一文件，被合并的文件改为交叉引用。

---

## 9. 行尾多余空格和多余空行

**问题**：每行末尾多余的空白字符（` `），以及文件末尾大量无意义的空行。

**示例**（多处文件）：
```
内容内容    
          
	    
```

**解决方案**：统一清理行尾空格，文件末尾保留一个空行。

---

## 10. 中文标点符号混用

**问题**：代码块或命令中使用中文标点，导致命令无效。

**示例**：
```markdown
- `：w` 保存       ← 中文冒号
- `？` 向前搜索    ← 中文问号
```

**解决方案**：确保命令中的符号使用英文标点：`:w`、`?`。

---

## 总结建议

| 预防措施 | 说明 |
|---------|------|
| 使用编辑器插件 | 安装 Markdown lint 工具（如 `markdownlint`）实时检测格式问题 |
| 开启显示空白字符 | 在编辑器中开启"显示空格和制表符"，避免混入不可见字符 |
| 代码块前后留空行 | 代码块 ` ``` ` 前后必须各有一个空行，不与列表标记并列 |
| 标题层级规范 | 顶级用 `# 1.`，子级用 `## 1.1`，避免跳级（如 `#` 后直接 `###`） |
| 规范提交前自查 | 提交前用 `git diff --check` 检查行尾空白和编码问题 |
