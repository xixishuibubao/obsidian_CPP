# 1. 线程与异步

C++11 起提供完整的线程和异步标准库。核心三个类：`std::thread`（底层线程）、`std::packaged_task`（任务包装）、`std::async`（高层异步）。

相关笔记：[[18.并发与多线程/1. atomic与内存序|atomic 与内存序]]

## 1.1 核心区别

| 特性 | `std::thread` | `std::packaged_task` | `std::async` |
|------|-------------|--------------------|------------|
| 层级 | 底层原生线程 | 任务包装器 | 高层异步封装 |
| 自动创建线程 | 必须手动 | 不主动创建，需丢给 `thread` | 自动开线程或延迟执行 |
| 获取返回值 | 无自带机制，需引用/全局传参 | 绑定 `future` 获取 | 直接返回 `future` |
| 等待结束 | 必须 `join()`/`detach()` | `future.get()` 即可 | `future.get()` 即可 |
| 适用场景 | 精细控制线程、常驻任务 | 任务解耦、任务队列调度 | 简单异步调用，不用管线程生命周期 |

> `packaged_task` 本身不是线程，是"装函数 + 存返回值"的容器；`async` 底层依然用到 `thread` + `packaged_task`。

## 1.2 逐个详解

### 1.2.1 `std::thread`

启动即创建系统线程，没有内置返回值获取，必须 `join()` 或 `detach()`：

```cpp
#include <iostream>
#include <thread>
using namespace std;

void work(int a, int b, int &res) {
    res = a + b;
}

int main() {
    int ret = 0;
    thread t(work, 10, 20, ref(ret));
    t.join();
    cout << "结果：" << ret << endl;
    return 0;
}
```

适合：常驻后台线程、循环任务、精细控制线程生命周期。

### 1.2.2 `std::packaged_task` + `std::future`

将函数打包成任务，绑定 `future` 获取返回值，**本身不运行**，需丢给 `thread`：

```cpp
#include <iostream>
#include <thread>
#include <future>
using namespace std;

int add(int a, int b) { return a + b; }

int main() {
    // 1. 打包函数任务
    packaged_task<int(int, int)> task(add);
    // 2. 获取 future，用于接收返回值
    future<int> fu = task.get_future();
    // 3. 将任务丢给线程执行（只能移动，不能拷贝）
    thread t(move(task), 10, 20);
    // 4. 阻塞等待结果
    int res = fu.get();
    cout << "结果：" << res << endl;
    t.join();
    return 0;
}
```

适合：任务队列、线程池、将任务和执行线程解耦。

### 1.2.3 `std::async`

最高层封装，一行实现异步执行。两种策略：

| 策略 | 行为 |
|------|------|
| `launch::async` | 立刻开新线程执行 |
| `launch::deferred` | 延迟执行，直到 `future.get()` 才在当前线程跑 |

默认策略为 `async | deferred`，由系统调度。

```cpp
#include <iostream>
#include <future>
using namespace std;

int add(int a, int b) { return a + b; }

int main() {
    future<int> fu = async(launch::async, add, 10, 20);
    int res = fu.get();  // 阻塞拿结果
    cout << "结果：" << res << endl;
    return 0;
}
```

适合：临时异步调用、只想拿个结果、不想管线程生命周期。

## 1.3 关系串联

```
std::thread          — 裸线程，啥都要自己管
    ↓
std::packaged_task   — 给函数套壳，绑定 future，解决线程返回值问题
    ↓
std::async           — 封装 thread + packaged_task + future，一键异步
```

## 1.4 并发工具总览

### 1.4.1 结果获取类

| 类 | 作用 |
|----|------|
| `std::future` | 获取异步任务结果，`.get()` 阻塞直到返回，只能移动 |
| `std::shared_future` | 允许多个线程同时等待同一个结果，可拷贝 |
| `std::promise` | 手动设置值/异常，让 `future` 接收，用于线程间安全传值 |

```cpp
std::promise<int> p;
std::future<int> f = p.get_future();

// 线程中：p.set_value(100);
// 主线程：int val = f.get();  // 获得 100
```

### 1.4.2 线程同步类

| 类 | 作用 |
|----|------|
| `std::mutex` | 互斥锁，保护共享变量 |
| `std::lock_guard` | RAII 自动加解锁，最推荐 |
| `std::unique_lock` | 更灵活的锁，可延迟加锁 |
| `std::condition_variable` | 条件变量，线程等待/唤醒（生产者-消费者必备） |

```cpp
std::mutex mtx;
std::condition_variable cv;

// 等待
std::unique_lock<std::mutex> lock(mtx);
cv.wait(lock, []{ return ready; });

// 唤醒
cv.notify_one();   // 唤醒一个
cv.notify_all();   // 唤醒全部
```

### 1.4.3 任务包装类

| 类 | 作用 |
|----|------|
| `std::function` | 包装函数/lambda/成员函数为可调用对象，线程池任务队列必用 |
| `std::bind` | 绑定函数参数，生成可调用对象 |

### 1.4.4 其他

| 类/函数 | 作用 |
|---------|------|
| `std::call_once` + `std::once_flag` | 保证函数全局只执行一次，多线程安全 |
| `std::atomic` | 原子类型，无锁线程安全，参见 [[1. atomic与内存序|atomic 与内存序]] |
| `std::this_thread` | 当前线程操作：`sleep_for()`、`get_id()`、`yield()` |

## 1.5 实际开发选型

| 需求 | 使用 |
|------|------|
| 常驻循环线程、嵌入式后台任务 | `std::thread` |
| 线程池、任务队列、先存任务再调度 | `std::packaged_task` + `std::function` |
| 临时异步调用、不想管线程生命周期 | `std::async` |
| 线程安全保护共享数据 | `std::mutex` + `std::lock_guard` |
| 线程等待/唤醒 | `std::condition_variable` |
| 无锁原子操作 | `std::atomic` |

## 1.6 注意事项

1. **避免大量 `std::thread` 频繁创建销毁**：单核 Cortex-A7 开销很大
2. **优先用 `async` 或少量常驻 `thread`**：少开瞬态线程
3. **`future.get()` 是阻塞式**：适合 IO 等待场景，不要在业务循环里频繁死等
4. **嵌入式最常用的 8 个类**：`thread`、`async`、`future`、`mutex`、`lock_guard`、`condition_variable`、`atomic`、`function`
