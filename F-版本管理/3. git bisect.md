# 1. Git Bisect 详解：快速定位 Bug 提交

`git bisect` 用二分法在提交历史中定位「第一个坏 commit」；可手动标记 good/bad，也可用 `bisect run` 自动化。

## 【核心精简提纲】

### 记忆主干

已知「旧版好、新版坏」时，`bisect start` → 标 `bad`（当前）与 `good`（旧 commit）→ 反复测试中间版本并标记 → 输出 first bad commit → **必须** `bisect reset` 回原分支。

### 核心要点清单

- **适用**：Bug 引入时间未知、历史很长、多人协作难排查
- **手动流**：`start` → `bad` → `good <hash>` → 测 → `good`/`bad` 循环
- **自动流**：`git bisect run <测试脚本>`（退出 0=good，非 0=bad）
- **辅助**：`bisect view` 看进度；`skip` 跳过不可测 commit
- **注意**：工作区干净（先 `stash`）；标记必须准确；结束必 `reset`

### 学习路径

1. 手动走一遍 5～10 个 commit 的小仓库
2. 写返回 0/1 的 shell 脚本练 `bisect run`
3. 与 [[F-版本管理/2. log规范|Commit 规范]] 结合：定位后看 message 理解变更

### 关键结论 / 流程

```
start → bad(HEAD) → good(旧hash) → [测试→标记]×N → 定位 → reset
```

## 【完整拓展详情】

它的原理很简单：利用**二分法**，把提交历史不断分成两半，让你标记「正常提交（无 Bug）」和「坏提交（有 Bug）」，自动缩小范围，最终精准找到问题提交。

相关内容：[[1. git|Git 基础]]、[[2. log规范|Commit 规范]]

## 1.1 核心使用场景

当你遇到：

- 代码突然出 Bug，但不知道什么时候引入的
- 提交历史很长，手动一个个排查太麻烦
- 团队多人开发，需要定位谁的代码导致问题

`git bisect` 能把排查时间从**小时级降到分钟级**。

## 1.2 基础使用步骤

### 1.2.1 启动二分查找

```bash
git bisect start
```

### 1.2.2 标记「坏提交」（当前有 Bug 的版本）

直接用当前分支最新提交即可：

```bash
git bisect bad
# 也可以指定提交哈希：
git bisect bad a1b2c3d
```

### 1.2.3 标记「好提交」（确定无 Bug 的版本）

找一个**确定没有这个 Bug** 的旧版本提交哈希：

```bash
git bisect good <提交哈希>
# 示例：
git bisect good 8f9e7d2
```

### 1.2.4 自动切换版本，手动测试

标记完成后，Git 会自动**切到中间的一个提交**，你只需要做一件事：

> 测试当前版本有没有 Bug

- 有 Bug → 标记为坏：`git bisect bad`
- 无 Bug → 标记为好：`git bisect good`

Git 会继续自动缩小范围，重复这个步骤。

### 1.2.5 定位到问题提交

最终 Git 会输出：

```
abc1234 is the first bad commit
```

这就是**引入 Bug 的那次提交**！

### 1.2.6 退出二分查找

排查完成后，必须执行退出命令，回到原来的代码状态：

```bash
git bisect reset
```

## 1.3 完整示例

假设提交历史：`A → B → C → D → E → F`（当前，有 Bug）

1. 启动：`git bisect start`
2. 标记坏：`git bisect bad`
3. 标记好（A 无 Bug）：`git bisect good A`
4. Git 切到 C，测试无 Bug → `git bisect good`
5. Git 切到 D，测试有 Bug → `git bisect bad`
6. Git 直接告诉你：**D 是第一个坏提交**
7. 退出：`git bisect reset`

## 1.4 自动化排查

如果你的项目有**单元测试/脚本**，可以让 Git 自动测试，完全不用手动操作！

### 1.4.1 命令格式

```bash
git bisect run <测试命令/脚本>
```

### 1.4.2 示例

```bash
# 用 npm test 自动测试
git bisect run npm test

# 用自定义脚本测试
git bisect run ./test_bug.sh
```

Git 会自动执行测试、标记好坏提交，**几秒内直接输出问题提交**。

## 1.5 常用辅助命令

查看当前排查状态：

```bash
git bisect view
```

跳过某个提交（不参与排查）：

```bash
git bisect skip <提交哈希>
```

查看帮助：

```bash
git bisect help
```

## 1.6 注意事项

1. **必须退出**：排查完一定要执行 `git bisect reset`，否则会一直停留在二分查找状态。
2. **测试要准确**：标记「好/坏」必须正确，否则会定位错误。
3. **干净工作区**：使用前最好 `git stash` 暂存未提交的修改，避免冲突。

## 1.7 总结

1. `git bisect` = **Git 二分查找定位 Bug 提交**
2. 三步核心：`start` → `bad` + `good` → 测试标记
3. 进阶：`git bisect run` 全自动排查
4. 结束：`git bisect reset` 恢复原分支
