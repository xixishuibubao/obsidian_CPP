# 1. INI 语法

INI（Initialization）是一种 **段（section）+ 键值对** 的配置格式，语法简单、人类可读，长期用于应用配置与 Windows 生态。与 [[8. YAML语法|YAML]]、[[3. XML|XML]] 同属结构化配置/数据标记语言。

> 本 vault 笔记 **不使用 YAML frontmatter**（见 [[1. Markdown|Markdown 笔记]]），但项目运行配置（如 `server.ini`）仍常用 INI。

## 1.1 基本结构

```ini
; 行注释（部分解析器也支持 #）
# 这也是注释

[section_name]
key = value
another_key = 123
flag = true

[database]
host = 127.0.0.1
port = 3306
```

| 元素 | 语法 | 说明 |
|------|------|------|
| Section | `[name]` | 逻辑分组，类似命名空间 |
| 键值对 | `key = value` | `=` 两侧空格可选 |
| 注释 | `;` 或 `#` | 依解析器而定；Python `configparser` 默认 `;` |
| 空行 | 忽略 | 用于可读性 |

**无原生嵌套**：INI 只有「段 + 扁平键」，不能像 YAML 那样缩进嵌套对象。复杂结构常用 **键名带点号** 或 **多段模拟**：

```ini
[paths]
FitsSavePath = /data/fits
RawSavePath = /data/raw

[preview]
width = 1920
height = 1080
```

或约定 `database.host = 127.0.0.1`（需解析器支持或应用层自行拆分）。

## 1.2 值类型与引号

INI **没有标准类型系统**，值一律是字符串，由应用或库做转换：

```ini
[system]
config = TestFake
version = 1.2.3
enabled = true          ; 读出来仍是 "true" 字符串
count = 42
path = /oem/hardware.ini
quoted = "含空格的 value"
```

| 类型 | 处理方式 |
|------|----------|
| 整数 / 布尔 | 应用 `int()`、`lower() == 'true'` 等自行解析 |
| 列表 | 逗号分隔 `hosts = a,b,c`，或重复键（非所有解析器支持） |
| 多行文本 | 不标准；少见，复杂文本更倾向 YAML/JSON |

## 1.3 常用场景

| 场景 | 典型文件 | 为何用 INI |
|------|----------|------------|
| **应用运行时配置** | `server.ini`、`hardware.ini` | 运维可直接改、diff 清晰 |
| **Windows / 传统软件** | `desktop.ini`、游戏设置 | 历史生态 |
| **Git 本地配置** | `~/.gitconfig`（INI 风格） | `[user]` / `[core]` 段 |
| **嵌入式 / 板端** | `/oem/hardware.ini` | 体积小、解析器轻（C/Python 均有） |
| **构建工具部分配置** | 某些 IDE、旧版工具链 | 简单键值足够 |

**本项目**：Telescope1126 用 **ConfigObj** 读 `server.ini` 与 `hardware.ini`，见 [[C-Linux生态/07.Telescope1126实战/2. Server双进程架构|Server 双进程架构]]、[[B-构建与脚本/02.脚本语言/3. Python|Python 笔记]]。

**不太适合 INI 的场景**：

- 深层嵌套结构（Kubernetes 级配置 → 用 YAML）
- 数组/对象列表（CI matrix、多服务编排 → YAML/JSON）
- 跨语言 Schema 校验（OpenAPI、JSON Schema → JSON/YAML）

## 1.4 Python 读取

### 1.4.1 标准库 configparser

```python
import configparser

cfg = configparser.ConfigParser()
cfg.read("server.ini", encoding="utf-8")

fits_path = cfg["path"]["FitsSavePath"]
port = cfg.getint("server", "port", fallback=8999)
```

### 1.4.2 ConfigObj（本项目）

```python
from configobj import ConfigObj

cfg = ConfigObj("server.ini")
fits_path = cfg["path"]["FitsSavePath"]
```

ConfigObj 支持嵌套访问、列表语法等扩展，与纯 INI 略有超出，但文件外观仍是段 + 键值。

## 1.5 INI vs YAML 速查

| 维度 | INI | YAML |
|------|-----|------|
| 结构 | 段 + 扁平键 | 缩进嵌套 map/list |
| 学习成本 | 极低 | 中（缩进敏感） |
| 人工编辑 | 非常适合运维改几行 | 适合复杂结构 |
| 标准程度 | 事实标准、实现有差异 | YAML 1.2 规范 |
| 典型生态 | 传统桌面/嵌入式配置 | CI、K8s、Docker Compose |

详见 [[8. YAML语法|YAML 语法]]。
