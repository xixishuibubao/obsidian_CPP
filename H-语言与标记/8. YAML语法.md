# 1. YAML 语法

YAML（YAML Ain't Markup Language）是一种 **缩进表示层级** 的数据序列化格式，可读性高，广泛用于 DevOps、CI/CD、容器编排与工具配置。与 [[7. INI语法|INI]]（扁平段键值）、[[3. XML|XML]]（标签嵌套）互补。

> 本 vault 的 Markdown 笔记 **不使用 YAML frontmatter**（`---` 元数据块），但 CI、K8s、Compose 等工程文件大量使用 YAML。

## 1.1 基本结构

YAML 用 **缩进（通常 2 空格）** 表示嵌套，**不用 Tab**（避免解析歧义）。

```yaml
# 整行注释
app:
  name: telescope-server
  port: 8999
  debug: false

database:
  host: 127.0.0.1
  port: 5432
  credentials:
    user: admin
    password: secret
```

| 元素 | 语法 | 说明 |
|------|------|------|
| 键值对 | `key: value` | 冒号后必须有空格 |
| 嵌套 | 子键多缩进一级 | 同级对齐 |
| 注释 | `#` 至行尾 | |
| 空值 | `key:` 或 `key: null` / `~` | |

## 1.2 标量类型

```yaml
integer: 42
float: 3.14
boolean: true      # 也支持 yes/no, on/off（YAML 1.1 遗留，慎用）
string_plain: hello
string_quoted: "含: 冒号或 # 时建议引号"
multiline: |
  保留换行
  第二行
folded: >
  折行会变成
  单行空格连接
```

| 写法 | 含义 |
|------|------|
| `"双引号"` | 支持 `\n` 等转义 |
| `'单引号'` | 字面量，不转义 |
| `\|` | 保留换行块 |
| `>` | 折叠换行块 |

## 1.3 列表（数组）

```yaml
# 块式列表
ports:
  - 8999
  - 9000

# 行内列表
tags: [api, embedded, linux]

steps:
  - name: build
    run: cmake --build build
  - name: test
    run: ctest
```

## 1.4 锚点与引用（复用）

```yaml
defaults: &defaults
  retries: 3
  timeout: 30

job_a:
  <<: *defaults
  name: deploy

job_b:
  <<: *defaults
  name: rollback
```

`&anchor` 定义锚点，`*anchor` 引用，`**` 合并到当前 map — 常用于 CI 模板去重。

## 1.5 常用场景

| 场景 | 典型文件 | 为何用 YAML |
|------|----------|-------------|
| **CI/CD** | `.github/workflows/*.yml`、`.gitlab-ci.yml` | 步骤列表 + 嵌套 env/matrix |
| **容器编排** | `docker-compose.yml` | 多服务、网络、卷声明 |
| **Kubernetes** | Deployment / Service / ConfigMap | 集群事实标准 |
| **配置管理** | Ansible playbook、Helm `values.yaml` | 变量 + 任务流 |
| **静态分析 / 工具** | `.clang-tidy`、部分 linter 配置 | 规则树状结构 |
| **API / Spec** | OpenAPI（可用 YAML 或 JSON） | 人类可读 Spec |
| **AI / Agent** | 部分 workflow、MCP 配置 | 结构化步骤与参数 |

**不太适合 YAML 的场景**：

- 仅需十几个键的嵌入式配置（[[7. INI语法|INI]] 更简单）
- 强 Schema 交换、高性能解析（JSON 更普遍）
- 含大量二进制或极深嵌套且需严格类型（JSON + Schema）

**本项目对照**：

| 配置类型 | 格式 | 示例 |
|----------|------|------|
| 运行时服务/硬件 | INI | `server.ini`、`hardware.ini` |
| 构建/CI（若引入） | YAML | GitHub Actions workflow |

## 1.6 Python 读取

```python
import yaml

with open("config.yaml", encoding="utf-8") as f:
    data = yaml.safe_load(f)

port = data["app"]["port"]
```

| API | 用途 |
|-----|------|
| `yaml.safe_load` | **推荐**；不反序列化任意 Python 对象 |
| `yaml.load` | 危险；恶意 YAML 可执行代码（`!!python/object`） |
| `yaml.safe_dump` | 写回文件 |

生产环境避免对不可信 YAML 使用 `yaml.load`；CI 配置也应审查第三方 workflow。

## 1.7 YAML vs INI vs JSON

| 维度 | YAML | INI | JSON |
|------|------|-----|------|
| 可读性 | 高 | 高（简单时） | 中 |
| 嵌套 / 列表 | 原生 | 弱 | 原生 |
| 注释 | 支持 | 支持 | 不支持 |
| 缩进敏感 | 是 | 否 | 否 |
| 典型用途 | DevOps、K8s | 应用 `.ini` | API、配置文件 |

## 1.8 常见坑

1. **缩进混用 Tab 与空格** → 解析失败或 silent 错误
2. **冒号后缺空格** — `key:value` 可能被当成字符串而非键值
3. **`yes`/`no`/`on`/`off`** — YAML 1.1 当布尔；1.2 更严格，建议显式 `true`/`false`
4. **特殊字符未引号** — `#`、`:`、`{`、`}` 等需引号包裹
5. **大文件重复** — 用锚点 `&` / `*` 或拆多文件 + `import`（工具相关）
