# 1. 进程管理

`ps` 静态快照查进程，`top`/`htop` 动态监控与交互管理。

## 【核心精简提纲】

### 记忆主干

**一次性查看**用 `ps`（静态快照）；**持续监控/定位高占用**用 `top`（动态刷新）。常用：`ps aux` 看全进程资源 → `ps -ef` 看 PPID 与完整命令 → `top` 按 P/M 排序 → 线程用 `top -H -p PID` 或 `ps -eLf`。

### 核心要点清单

- `ps aux`：所有用户进程（a/u/x）；`ps -ef`：全格式含 PPID
- `STAT`：R 运行、S 睡眠、Z 僵尸、D 不可中断 I/O 睡眠
- `top` 概览：负载（1/5/15 min）、CPU 细分（us/sy/wa/id）、内存/Swap
- 交互：`P` CPU 排序、`M` 内存、`k` 杀进程、`u` 按用户筛选
- 线程：`ps -T -p PID` / `top -H -p PID` / `pstree -p`
- 选型：`ps` 一次性确认；`top` 实时跟踪异常

### 关键结论 / 流程

```plain
确认进程存活 → ps aux | grep │ 定位高 CPU/内存 → top 排序 │ 查线程 → top -H / ps -eLf
```

## 【完整拓展详情】


`ps`（Process Status）的核心是**静态快照**：执行时一次性捕获当前所有（或指定）进程的状态，不实时更新。

## 1.1 核心作用

- 查看当前系统中运行的进程列表（PID、所有者、资源占用、运行状态等）
- 筛选特定进程（指定用户、PID、进程名等）
- 输出进程详细信息（父进程 ID、内存占用、启动时间、命令路径等）

## 1.2 常用选项

`ps` 的选项分为 Unix 风格（带 `-`）、BSD 风格（不带 `-`）和 GNU 风格（带 `--`），日常使用如下：

| 命令 | 功能说明 |
|------|----------|
| `ps aux` | **最常用**：查看所有用户的所有进程（`a`：所有终端进程；`u`：显示用户信息；`x`：非终端进程） |
| `ps -ef` | 查看所有进程的完整信息（`-e`：所有进程；`-f`：全格式，含 PPID、命令路径） |
| `ps aux \| grep <进程名>` | 筛选指定名称的进程，如 `ps aux \| grep nginx` |
| `ps -u <用户名>` | 查看指定用户的进程，如 `ps -u root` |
| `ps -p <PID>` | 查看指定 PID 的进程详情 |

## 1.3 输出字段含义（`ps aux`）

| 字段 | 含义 |
|------|------|
| `USER` | 进程所有者（如 `root`、`www`） |
| `PID` | 进程唯一 ID，管理进程的核心标识 |
| `%CPU` | 进程占用的 CPU 百分比 |
| `%MEM` | 进程占用的物理内存百分比 |
| `VSZ` | 虚拟内存大小（KB） |
| `RSS` | 物理内存大小（KB，实际占用） |
| `TTY` | 关联终端（`?` 表示后台服务） |
| `STAT` | 进程状态：`R` 运行中、`S` 睡眠、`Z` 僵尸、`D` 不可中断睡眠 |
| `START` | 进程启动时间 |
| `TIME` | 进程累计 CPU 时间 |
| `COMMAND` | 启动命令（`[]` 表示内核进程） |

# 2. `top` — 动态进程监控

`top`（Table Of Processes）的核心是**动态实时监控**：启动后持续刷新（默认每 3 秒），同时显示系统整体资源占用。

## 2.1 核心作用

- 实时监控 CPU、内存、负载、进程总数等系统资源
- 实时跟踪进程资源占用变化（定位异常进程）
- 交互式管理进程（排序、筛选、杀进程、调整优先级）

## 2.2 界面组成

启动 `top` 后界面分为**系统概览区**和**进程列表区**：

```plain
top - 14:30:00 up 2 days, 5:10, 2 users, load average: 0.05, 0.12, 0.15
Tasks: 189 total, 1 running, 188 sleeping, 0 stopped, 0 zombie
%Cpu(s): 0.3 us, 0.1 sy, 0.0 ni, 99.6 id, 0.0 wa, 0.0 hi, 0.0 si, 0.0 st
MiB Mem : 15917.9 total, 12800.3 free, 1800.5 used, 1317.1 buff/cache
MiB Swap: 16384.0 total, 16384.0 free, 0.0 used. 13785.2 avail Mem
```

### 2.2.1 系统概览区（前 5 行）

- **第 1 行**：当前时间、运行时长、登录用户数、**系统负载**（1/5/15 分钟平均，通常不超过 CPU 核心数）
- **第 2 行**：进程总数 / 运行中 / 睡眠 / 停止 / 僵尸数量
- **第 3 行**：CPU 使用率细分：`us` 用户、`sy` 内核、`ni` 优先级调整、`id` 空闲、`wa` I/O 等待（过高可能是磁盘/网络瓶颈）
- **第 4-5 行**：内存 / Swap 的总量、已用、空闲、缓存；`avail Mem` 为可立即分配给新进程的内存

### 2.2.2 进程列表区

默认按 `%CPU` 降序排列，字段与 `ps` 类似，新增 `PR`（优先级）、`NI`（nice 值）等。

## 2.3 交互操作（启动后按键）

| 按键 | 功能 |
|------|------|
| `P` | 按 CPU 使用率降序（默认） |
| `M` | 按内存使用率降序 |
| `N` | 按 PID 升序 |
| `T` | 按运行时间降序 |
| `u` | 输入用户名，筛选该用户进程 |
| `p` | 输入 PID，筛选该进程 |
| `k` | 输入 PID → Enter → `9`（SIGKILL）杀死进程 |
| `1` | 显示所有 CPU 核心使用率 |
| `s` | 调整刷新间隔（如输入 `5` 为 5 秒） |
| `q` | 退出 `top` |

# 3. `ps` 与 `top` 对比

| 维度 | `ps` | `top` |
|------|------|-------|
| 数据特性 | 静态快照（执行时捕获一次） | 动态实时（持续刷新，默认 3 秒） |
| 核心场景 | 查看进程详情、筛选特定进程（一次性） | 监控资源变化、跟踪进程状态（实时性） |
| 交互能力 | 无（执行即结束） | 强（排序、筛选、杀进程等） |
| 资源消耗 | 低（一次执行即结束） | 略高（持续运行） |

# 4. 典型场景

1. **查看 nginx 是否在运行** → `ps aux | grep nginx`（一次性确认）
2. **系统 CPU 过高，定位进程** → `top`（按 CPU 排序，实时跟踪）
3. **查看进程父 ID 和启动命令** → `ps -ef | grep <PID>`（`-ef` 显示 PPID 和完整命令）
4. **监控某用户资源占用** → `top` 后按 `u` 输入用户名筛选

# 5. 查看线程

`ps`、`top`、`pstree` 等工具均可显示进程内的线程信息。

## 5.1 `ps` 查看线程

| 命令 | 功能 |
|------|------|
| `ps -T -p <PID>` | 查看指定进程的所有线程，`SPID` 列为线程 ID（TID） |
| `ps auxT` | 查看所有进程的所有线程（BSD 风格） |
| `ps -eLf` | 长格式显示所有进程及线程，`LWP` 为线程 ID，`NLWP` 为线程总数 |

示例：查看 nginx 的线程：
```bash
ps -eLf | grep nginx
```

## 5.2 `top` 实时监控线程

```bash
top          # 启动 top
# 按 H 键   # 切换为线程视图，PID 列实际为线程 ID（TID）
# 按 p 键   # 输入进程 PID，仅显示该进程的线程
# 按 P/M    # 按 CPU/内存排序，快速定位高消耗线程
```

组合命令直接进入线程模式：
```bash
top -H -p <PID>
```

## 5.3 `pstree` 树形结构

以树形展示进程与线程的关系，线程以 `(...)` 标注：

```bash
pstree -T <PID>    # 显示进程的线程树
pstree -p <PID>    # 显示进程及线程的 PID/TID
```

输出示例：`process_name(1234)─┬─thread1(1235)─┬─...`

## 5.4 `htop` 增强监控（推荐）

`htop` 是 `top` 的增强版，对线程支持更直观（需额外安装：`apt install htop` 或 `yum install htop`）。

操作步骤：
1. 启动 `htop`
2. `F2` 进入设置 → Display options → 勾选「Show threads」，`F10` 保存
3. 线程以缩进形式显示在所属进程下方
4. `F4` 搜索进程名或线程 ID，快速定位

## 5.5 命令对比

| 命令 | 特点 | 适用场景 |
|------|------|----------|
| `ps` | 静态快照，显示线程 ID 和基本信息 | 一次性查看线程列表和数量 |
| `top` | 实时动态，支持排序和筛选 | 跟踪线程资源占用（如 CPU 异常高的线程） |
| `pstree` | 树形结构展示线程与进程的关系 | 理解父子关系或整体结构 |
| `htop` | 可视化强，操作简单 | 日常交互式线程监控（推荐） |

若发现某进程 CPU 占用过高，先用 `top -H -p <PID>` 实时查看该进程内所有线程的 CPU 占用，定位问题线程 TID，再结合 `ps -T -p <PID>` 确认线程信息。

## 1.4 常见踩坑

- **僵尸进程（Z）**：子进程已退出但父进程未 `wait` → 查 PPID，`kill` 父进程或修复 wait 逻辑
- **`ps aux | grep` 误匹配**：grep 自身也会出现 → 用 `grep [n]ginx` 或 `pgrep`
- **load average 误解**：数值可大于核心数（含 I/O 等待）；须结合 `wa` 与 `top -H` 判断
- **容器/PID namespace**：宿主机 `ps` 看到的 PID 与容器内不同 → 进容器 namespace 再查
