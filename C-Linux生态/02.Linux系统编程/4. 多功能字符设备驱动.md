# 1. 多功能字符设备驱动

## 【核心精简提纲】

### 记忆主干

字符设备：`register_chrdev_region` → `cdev_add` → `device_create` → 实现 `file_operations`；`module_exit` 对称释放。内核态小栈、`copy_*_user`、中断上下文禁 `mutex`。

### 核心要点清单

- 注册链：设备号 → cdev → class → `/dev/xxx` 节点
- `ioctl`：`_IO/_IOR/_IOW` 宏 + 公共头文件保证 ABI
- 内存：栈仅 4~8KB，大缓冲 `kmalloc`/`kfree`（中断用 `GFP_ATOMIC`）
- 同步：进程上下文 `mutex`；中断上下文 `spinlock_irqsave`（不可 mutex）
- 中断：handler 极短、不可休眠；模拟可用 `request_irq`
- 与用户态差异：无 `main`、无 stdio、错误码 `-EINVAL` 等、`MODULE_LICENSE("GPL")`

### 关键结论 / 流程

```
insmod → module_init → 注册设备 → 用户 open/ioctl → copy_*_user → rmmod 对称释放
```

### 核心功能

1. 字符设备注册与注销
2. `ioctl` 接口实现（用户态↔内核态双向通信）
3. 内核内存管理（`kmalloc`/`kfree`，栈空间限制）
4. 并发与同步（自旋锁 + 互斥锁，进程/中断上下文）
5. 中断处理（模拟中断，展示中断上下文约束）
6. 内核日志与错误处理（`printk` + 标准错误码 + `goto` 清理）
7. 多文件组织（`core` + `utils`，内核多文件交互）

## 1.2 核心差异总览

| 差异点 | 用户态 | 内核态 |
|--------|--------|--------|
| **入口函数** | `main()`，主动执行，有命令行参数 | `module_init()`，被动加载，无命令行参数 |
| **头文件** | 标准 C 库（`stdio.h`/`stdlib.h`/`unistd.h`） | 内核专属（`linux/*.h`），无标准库 |
| **内存管理** | `malloc()`/`free()`，栈空间大（8MB） | `kmalloc()`/`kfree()`，栈极小（4~8KB），大数组必须用内核内存 |
| **数据拷贝** | 直接指针访问 | `copy_to_user()`/`copy_from_user()`，安全检查 |
| **同步机制** | `pthread` 库（互斥锁/条件变量） | 内核原语（自旋锁用于中断上下文，互斥锁用于进程上下文） |
| **错误处理** | `perror()`，返回值随意 | `printk()`，标准错误码（`-EINVAL`/`-ENOMEM`），`goto` 模式 |
| **编译方式** | `gcc` 直接编译，链接标准库 | Kbuild Makefile，链接内核符号，生成 `.ko` |
| **执行模型** | 独立进程，独立地址空间 | 内核模块，共享内核地址空间，无独立进程 |
| **错误影响** | 仅影响自身 | 可能导致内核崩溃（Kernel Panic）|

## 1.3 头文件（multi_func_drv.h）

```c
#ifndef MULTI_FUNC_DRV_H
#define MULTI_FUNC_DRV_H

#include <linux/module.h>
#include <linux/fs.h>
#include <linux/cdev.h>
#include <linux/spinlock.h>
#include <linux/mutex.h>
#include <linux/interrupt.h>
#include <linux/slab.h>
#include <linux/uaccess.h>
#include <linux/ioctl.h>

/* 差异点：用户态用标准 C 库（stdio.h），内核态用 linux/*.h，无标准库 */

// 设备信息定义
#define DRV_NAME "multi_func_drv"
#define DRV_MAJOR 240
#define DRV_MINOR 0
#define DRV_DEV_COUNT 1

// IOCTL 命令（必须用 _IO/_IOR/_IOW/_IOWR 宏，保证跨平台兼容）
#define DRV_IOCTL_MAGIC 'M'
#define DRV_IOCTL_SET_DATA    _IOW(DRV_IOCTL_MAGIC, 0, struct drv_data)
#define DRV_IOCTL_GET_DATA    _IOR(DRV_IOCTL_MAGIC, 1, struct drv_data)
#define DRV_IOCTL_TRIGGER_IRQ  _IO(DRV_IOCTL_MAGIC, 2)
#define DRV_IOCTL_CLEAR_STATS  _IO(DRV_IOCTL_MAGIC, 3)

// 用户态与内核态共享的数据结构（必须保证二进制一致）
struct drv_data {
    int id;             // 数据 ID
    char buf[64];       // 缓冲区
    unsigned int len;   // 数据长度
};

// 内核态私有统计信息（用户态通过 IOCTL 获取）
struct drv_stats {
    unsigned int read_count;
    unsigned int write_count;
    unsigned int ioctl_count;
    unsigned int irq_count;
};

// 内核态私有数据结构
struct drv_private {
    dev_t devno;
    struct cdev cdev;
    struct class *class;
    struct device *device;
    struct drv_data data;
    struct drv_stats stats;
    spinlock_t spinlock;     // 自旋锁（中断上下文）
    struct mutex mutex;      // 互斥锁（进程上下文）
    void *buffer;            // kmalloc 分配
    int irq;                 // 中断号（模拟）
};

/* 差异点：内核态栈极小（4~8KB），大数组必须用 kmalloc，不可在栈上定义 */

// 工具函数声明
int drv_utils_init(struct drv_private *drv);
void drv_utils_exit(struct drv_private *drv);
void drv_utils_update_stats(struct drv_private *drv, unsigned int type);
int drv_utils_copy_data_to_user(struct drv_data *kernel_data, struct drv_data __user *user_data);
int drv_utils_copy_data_from_user(struct drv_data __user *user_data, struct drv_data *kernel_data);

// 中断处理与字符设备操作声明
irqreturn_t drv_irq_handler(int irq, void *dev_id);
int drv_open(struct inode *inode, struct file *filp);
int drv_release(struct inode *inode, struct file *filp);
ssize_t drv_read(struct file *filp, char __user *buf, size_t count, loff_t *f_pos);
ssize_t drv_write(struct file *filp, const char __user *buf, size_t count, loff_t *f_pos);
long drv_ioctl(struct file *filp, unsigned int cmd, unsigned long arg);

#endif
```

## 1.4 工具函数（multi_func_utils.c）

```c
#include "multi_func_drv.h"

/* 多文件交互：通过头文件声明 + Makefile 同一 obj-y，不加 static 即为编译单元内可见 */

int drv_utils_init(struct drv_private *drv) {
    if (!drv) {
        printk(KERN_ERR "[%s] drv is NULL\n", __func__);
        return -EINVAL;
    }
    spin_lock_init(&drv->spinlock);
    mutex_init(&drv->mutex);
    memset(&drv->stats, 0, sizeof(struct drv_stats));

    // kmalloc（对比用户态 malloc），GFP_KERNEL 表示可休眠
    drv->buffer = kmalloc(4096, GFP_KERNEL);
    if (!drv->buffer) {
        printk(KERN_ERR "[%s] kmalloc failed\n", __func__);
        return -ENOMEM;
    }
    memset(drv->buffer, 0, 4096);
    printk(KERN_INFO "[%s] utils init success\n", __func__);
    return 0;
}

void drv_utils_exit(struct drv_private *drv) {
    if (!drv) return;
    if (drv->buffer) {
        kfree(drv->buffer);   // kfree（对比用户态 free）
        drv->buffer = NULL;
    }
}

/* 同步差异：用户态用 pthread_mutex_t，内核态用 spinlock_t（中断上下文）+ mutex_t（进程上下文） */
void drv_utils_update_stats(struct drv_private *drv, unsigned int type) {
    if (!drv) return;
    unsigned long flags;
    spin_lock_irqsave(&drv->spinlock, flags);  // 自旋锁保护（可被中断上下文访问）
    switch (type) {
    case 0: drv->stats.read_count++;  break;
    case 1: drv->stats.write_count++; break;
    case 2: drv->stats.ioctl_count++; break;
    case 3: drv->stats.irq_count++;   break;
    default: break;
    }
    spin_unlock_irqrestore(&drv->spinlock, flags);
}

/* 数据拷贝差异：用户态可直接指针访问，内核态必须用 copy_to_user/copy_from_user */
int drv_utils_copy_data_to_user(struct drv_data *kernel_data, struct drv_data __user *user_data) {
    if (!kernel_data || !user_data) return -EINVAL;
    if (copy_to_user(user_data, kernel_data, sizeof(struct drv_data))) {
        printk(KERN_ERR "[%s] copy_to_user failed\n", __func__);
        return -EFAULT;
    }
    return 0;
}

int drv_utils_copy_data_from_user(struct drv_data __user *user_data, struct drv_data *kernel_data) {
    if (!user_data || !kernel_data) return -EINVAL;
    if (copy_from_user(kernel_data, user_data, sizeof(struct drv_data))) {
        printk(KERN_ERR "[%s] copy_from_user failed\n", __func__);
        return -EFAULT;
    }
    return 0;
}
```

## 1.5 驱动核心（multi_func_core.c）

```c
#include "multi_func_drv.h"

static struct drv_private g_drv;  // 全局变量，内核地址空间所有进程可见，需同步保护

/* 中断上下文约束：
 * 1. 执行时间极短，不能休眠
 * 2. 只能用 GFP_ATOMIC 分配内存，不能用 GFP_KERNEL
 * 3. 不能用会引起休眠的函数（如 mutex_lock）
 * 4. 必须使用自旋锁同步 */
irqreturn_t drv_irq_handler(int irq, void *dev_id) {
    struct drv_private *drv = (struct drv_private *)dev_id;
    if (!drv) return IRQ_NONE;

    drv->data.id++;
    snprintf((char *)drv->buffer, 4096, "IRQ triggered: id = %d", drv->data.id);
    strncpy(drv->data.buf, (char *)drv->buffer, sizeof(drv->data.buf) - 1);
    drv->data.len = strlen(drv->data.buf);

    drv_utils_update_stats(drv, 3);
    printk(KERN_INFO "[%s] IRQ %d triggered, id = %d\n", __func__, irq, drv->data.id);
    return IRQ_HANDLED;
}

int drv_open(struct inode *inode, struct file *filp) {
    filp->private_data = &g_drv;   // 绑定私有数据到文件结构
    printk(KERN_INFO "[%s] device opened\n", __func__);
    return 0;
}

int drv_release(struct inode *inode, struct file *filp) {
    printk(KERN_INFO "[%s] device released\n", __func__);
    return 0;
}

ssize_t drv_read(struct file *filp, char __user *buf, size_t count, loff_t *f_pos) {
    struct drv_private *drv = filp->private_data;
    if (!drv) return -EINVAL;

    mutex_lock(&drv->mutex);                    // 进程上下文用互斥锁
    int ret = drv_utils_copy_data_to_user(&drv->data, (struct drv_data __user *)buf);
    if (ret < 0) { mutex_unlock(&drv->mutex); return ret; }
    drv_utils_update_stats(drv, 0);
    mutex_unlock(&drv->mutex);

    return sizeof(struct drv_data);
}

ssize_t drv_write(struct file *filp, const char __user *buf, size_t count, loff_t *f_pos) {
    struct drv_private *drv = filp->private_data;
    if (!drv) return -EINVAL;

    mutex_lock(&drv->mutex);
    int ret = drv_utils_copy_data_from_user((struct drv_data __user *)buf, &drv->data);
    if (ret < 0) { mutex_unlock(&drv->mutex); return ret; }
    drv_utils_update_stats(drv, 1);
    mutex_unlock(&drv->mutex);

    return sizeof(struct drv_data);
}

long drv_ioctl(struct file *filp, unsigned int cmd, unsigned long arg) {
    struct drv_private *drv = filp->private_data;
    if (!drv) return -EINVAL;

    // 检查命令有效性
    if (_IOC_TYPE(cmd) != DRV_IOCTL_MAGIC) return -EINVAL;
    if (_IOC_NR(cmd) > 3) return -EINVAL;

    mutex_lock(&drv->mutex);
    drv_utils_update_stats(drv, 2);
    int ret = 0;

    switch (cmd) {
    case DRV_IOCTL_SET_DATA:
        ret = drv_utils_copy_data_from_user((struct drv_data __user *)arg, &drv->data);
        break;
    case DRV_IOCTL_GET_DATA:
        ret = drv_utils_copy_data_to_user(&drv->data, (struct drv_data __user *)arg);
        break;
    case DRV_IOCTL_TRIGGER_IRQ:
        drv_irq_handler(drv->irq, drv);     // 模拟触发中断
        break;
    case DRV_IOCTL_CLEAR_STATS:
        memset(&drv->stats, 0, sizeof(struct drv_stats));
        break;
    default:
        ret = -EINVAL;
        break;
    }

    mutex_unlock(&drv->mutex);
    return ret;
}

static const struct file_operations g_drv_fops = {
    .owner = THIS_MODULE,
    .open = drv_open,
    .release = drv_release,
    .read = drv_read,
    .write = drv_write,
    .unlocked_ioctl = drv_ioctl,
};

/* 入口函数差异：用户态是 main()，内核态是 module_init()，被动加载，无命令行参数
 * 错误处理必须用 goto 保证资源不泄漏 */
static int __init multi_func_drv_init(void) {
    int ret = 0;
    struct drv_private *drv = &g_drv;
    memset(drv, 0, sizeof(struct drv_private));

    // 步骤 1：分配设备号
    drv->devno = MKDEV(DRV_MAJOR, DRV_MINOR);
    if (DRV_MAJOR)
        ret = register_chrdev_region(drv->devno, DRV_DEV_COUNT, DRV_NAME);
    else
        ret = alloc_chrdev_region(&drv->devno, DRV_MINOR, DRV_DEV_COUNT, DRV_NAME);
    if (ret < 0) goto err_out;

    // 步骤 2：初始化字符设备
    cdev_init(&drv->cdev, &g_drv_fops);
    drv->cdev.owner = THIS_MODULE;
    ret = cdev_add(&drv->cdev, drv->devno, DRV_DEV_COUNT);
    if (ret < 0) goto err_unregister_chrdev;

    // 步骤 3：创建设备类与节点
    drv->class = class_create(THIS_MODULE, DRV_NAME);
    if (IS_ERR(drv->class)) { ret = PTR_ERR(drv->class); goto err_cdev_del; }
    drv->device = device_create(drv->class, NULL, drv->devno, NULL, DRV_NAME);
    if (IS_ERR(drv->device)) { ret = PTR_ERR(drv->device); goto err_class_destroy; }

    // 步骤 4：初始化工具模块
    ret = drv_utils_init(drv);
    if (ret < 0) goto err_device_destroy;

    // 步骤 5：注册中断（模拟）
    drv->irq = 10;
    ret = request_irq(drv->irq, drv_irq_handler, IRQF_SHARED, DRV_NAME, drv);
    if (ret < 0) goto err_utils_exit;

    // 步骤 6：初始化共享数据
    drv->data.id = 0;
    strncpy(drv->data.buf, "initial data", sizeof(drv->data.buf) - 1);
    drv->data.len = strlen(drv->data.buf);

    printk(KERN_INFO "[%s] driver init success\n", __func__);
    return 0;

    // goto 错误处理（从后往前释放资源）
err_utils_exit:      drv_utils_exit(drv);
err_device_destroy:  device_destroy(drv->class, drv->devno);
err_class_destroy:   class_destroy(drv->class);
err_cdev_del:        cdev_del(&drv->cdev);
err_unregister_chrdev: unregister_chrdev_region(drv->devno, DRV_DEV_COUNT);
err_out:
    printk(KERN_ERR "[%s] init failed: %d\n", __func__, ret);
    return ret;
}

static void __exit multi_func_drv_exit(void) {
    struct drv_private *drv = &g_drv;
    free_irq(drv->irq, drv);
    drv_utils_exit(drv);
    device_destroy(drv->class, drv->devno);
    class_destroy(drv->class);
    cdev_del(&drv->cdev);
    unregister_chrdev_region(drv->devno, DRV_DEV_COUNT);
    printk(KERN_INFO "[%s] driver exit success\n", __func__);
}

module_init(multi_func_drv_init);
module_exit(multi_func_drv_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Your Name");
MODULE_DESCRIPTION("Multi Function Character Device Driver Demo");
MODULE_VERSION("1.0");
```

## 1.9 用户态 vs 内核态开发核心差异（7 维度）

### 1.9.1 入口函数与程序结构

| 维度 | 用户态 | 内核态 |
|------|--------|--------|
| **入口** | `main()`，主动执行，支持命令行参数 | `module_init()`，被动加载，无参数 |
| **执行模型** | 线性执行：`main()` → `return` | 事件驱动：加载 → 注册 → 等待事件 → 卸载 |
| **出口** | `return` 或 `exit()` | `module_exit()` |

```c
// 用户态：线性、主动
int main(int argc, char *argv[]) {
    printf("Hello\n");
    return 0;
}

// 内核态：事件驱动、被动
static int __init drv_init(void) {
    printk(KERN_INFO "Hello\n");
    register_chrdev(240, "drv", &fops);
    return 0;
}
static void __exit drv_exit(void) {
    unregister_chrdev(240, "drv");
}
module_init(drv_init);
module_exit(drv_exit);
MODULE_LICENSE("GPL");
```

### 1.9.2 API 与库

| 功能 | 用户态 API | 内核态 API |
|------|-----------|-----------|
| 打印日志 | `printf()`/`fprintf()` | `printk()`（有日志级别，写入 `dmesg`） |
| 内存分配 | `malloc()`/`free()` | `kmalloc()`/`kfree()`（物理连续）、`vmalloc()`/`vfree()`（虚拟连续） |
| 延时 | `sleep()`/`usleep()` | `msleep()`/`ssleep()`（休眠）、`udelay()`/`mdelay()`（忙等） |
| 字符串 | 标准 C 库 `string.h` | 内核实现 `linux/string.h`（语法相同，但不是标准库） |

**绝对禁止**在驱动中调用任何标准 C 库函数（`stdio.h`、`stdlib.h` 等）。

### 1.9.3 内存管理

| 维度 | 用户态 | 内核态 |
|------|--------|--------|
| **栈空间** | 约 8MB（可通过 `ulimit` 调整） | **4KB~8KB**（极小！） |
| **堆** | 标准堆，可分配 GB 级内存 | `kmalloc` 最大数百 KB |
| **大数组** | 可直接在栈上定义 | **绝对禁止**，必须用 `kmalloc` |

```c
// 用户态：可以定义大局部数组
void user_stack(void) {
    char buf[1024 * 1024];  // 1MB，没问题
}

// 内核态：绝对禁止！
void kernel_stack(void) {
    char buf[1024 * 1024];  // 导致栈溢出 → 内核崩溃！
    // 正确做法：
    char *buf = kmalloc(1024 * 1024, GFP_KERNEL);
    if (!buf) { /* 处理 ENOMEM */ }
    kfree(buf);
}
```

### 1.9.4 错误处理

| 维度 | 用户态 | 内核态 |
|------|--------|--------|
| **返回值** | 随意（`0` 成功，`-1` 失败） | **标准内核错误码**（`-EINVAL`/`-ENOMEM`/`-EFAULT`/`-EBUSY`） |
| **错误信息** | `perror()` | `printk(KERN_ERR ...)` |
| **资源释放** | 可忽略（进程退出时系统回收） | **必须手动释放**，使用 `goto` 模式 |
| **错误影响** | 仅自身崩溃 | 导致内核崩溃（Kernel Panic） |

```c
int kernel_func(void) {
    void *buf = kmalloc(1024, GFP_KERNEL);
    if (!buf) return -ENOMEM;

    int ret = register_chrdev(240, "drv", &fops);
    if (ret < 0) {
        ret = -EBUSY;
        goto err_free;
    }
    return 0;

err_free:
    kfree(buf);
    return ret;
}
```

### 1.9.5 并发与同步

| 维度 | 用户态 | 内核态 |
|------|--------|--------|
| **同步原语** | `pthread` 互斥锁/条件变量 | `spinlock_t` / `mutex_t` / `semaphore` / `atomic_t` |
| **特殊场景** | 仅进程/线程间同步 | 需处理**进程上下文 + 中断上下文 + 多核并发** |
| **中断上下文** | 不存在 | 不能休眠、不能用 `mutex_lock`、只能用自旋锁 |

```c
static spinlock_t g_lock;
static int g_count;

void process_func(void) {
    unsigned long flags;
    spin_lock_irqsave(&g_lock, flags);
    g_count++;
    spin_unlock_irqrestore(&g_lock, flags);
}

irqreturn_t irq_handler(int irq, void *dev_id) {
    unsigned long flags;
    spin_lock_irqsave(&g_lock, flags);
    g_count++;
    spin_unlock_irqrestore(&g_lock, flags);
    return IRQ_HANDLED;
}
```

### 1.9.6 编译与链接

| 维度 | 用户态 | 内核态 |
|------|--------|--------|
| **编译器** | `gcc` 直接编译 | Kbuild Makefile（依赖内核源码树） |
| **输出** | 可执行文件 | `.ko` 内核模块 |
| **链接** | 链接标准 C 库 | 链接内核符号表，**不能链接标准库** |
| **依赖** | 无 | 必须使用与当前内核一致的内核源码 |

```makefile
# 内核驱动 Makefile
obj-m += hello_drv.o
KERNELDIR ?= /lib/modules/$(shell uname -r)/build
PWD := $(shell pwd)
all:
	$(MAKE) -C $(KERNELDIR) M=$(PWD) modules
```

### 1.9.7 特有约束（用户态没有）

1. **无进程地址空间**：驱动共享内核地址空间，无独立进程保护
2. **中断上下文限制**：执行时间短，不能休眠，不能分配大量内存，不能用可休眠的函数
3. **模块依赖性**：按依赖顺序加载，逆序卸载
4. **许可证要求**：必须声明 `MODULE_LICENSE("GPL")`，否则不能调用 `EXPORT_SYMBOL_GPL` 导出的函数
5. **错误影响全局**：内存越界或空指针直接导致内核崩溃，整个系统死机

## 1.10 常见踩坑

- **栈溢出**：大数组放栈上 → Kernel Panic；改 `kmalloc`
- **中断里 mutex_lock**：可死锁 → 中断上下文只用 spinlock
- **ioctl 未校验 `_IOC_TYPE`**：非法 cmd 导致越界 → switch 前检查 magic 与 nr
- **insmod 版本不匹配**：`Invalid module format` → 用目标内核的 `KERNELDIR` 编译
- **设备节点权限**：`open` EACCES → `udev` 规则或 `chmod 666 /dev/xxx`（开发期）
- **unload 后仍被占用**：`rmmod: module in use` → 先关闭所有 `open` 的进程
