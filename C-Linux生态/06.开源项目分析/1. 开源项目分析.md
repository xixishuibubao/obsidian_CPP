## 1.1 网络与服务端

### 1.1.1 高性能网络库

### Muduo（C++，Reactor 模式）

**项目定位**：Linux 下的 Reactor 模式 C++ 网络库（1.5 万星），作者陈硕。

#### 源码解析资源

1. **《Linux 多线程服务器编程：使用 Muduo C++ 网络库》（书籍）**
   - **核心内容**：
     - Reactor 模式实现（`EventLoop`/`Channel` 机制）
     - 线程池设计（`EventLoopThreadPool`）
     - 定时器队列（最小堆 + 红黑树混合结构）
     - 案例：聊天服务器、HTTP 服务器的架构演进
   - **特色**：作者逐模块解析源码，揭示"如何用 5000 行代码实现高并发"。
   - **学习路径**：配套 GitHub 仓库（[chenshuo/muduo](https://github.com/chenshuo/muduo)），每章对应源码注释。

2. **博客系列：Muduo 源码剖析（陈硕）**
   - **链接**：[https://chenshuo.github.io/](https://chenshuo.github.io/)
   - **重点**：
     - `EPollPoller` 的 LT/ET 模式选择
     - `Buffer` 类的预分配 + 双指针优化
     - 原子操作与 CAS 在 `AtomicInt64` 中的实现

#### 架构核心

- **分层设计**：`muduo::net`（网络层）、`muduo::base`（基础库）
- **性能优化**：无锁队列（`BlockingQueue`）、内存池（`Poco` 风格）
- **设计哲学**："够用即可"的极简主义（对比 ACE 框架的过度设计）

---

### libhv（C/C++ 混合）

**项目定位**：跨平台网络库（C 实现 60% + C++ 封装 30%），支持 HTTP/SSL/WebSocket 等协议。

#### 源码解析资源

1. **CSDN 博客：libhv 源码框架详解**
   - **核心内容**：
     - 事件循环（`hloop`）的跨平台实现（epoll/iocp/kqueue）
     - 内存池（`hbuf_t`）与环形缓存（`HRingBuf`）的零拷贝设计
     - 线程池（`HThreadPool`）的动态扩缩容策略
   - **特色**：对比 Muduo 的 Reactor 模式，解析 libhv 如何通过 C 实现高性能网络库。
   - **学习路径**：GitHub 仓库（[ithewei/libhv](https://github.com/ithewei/libhv)），重点分析 `event/` 和 `evpp/` 目录。

2. **知乎专栏：libhv 的 C++ 封装哲学**
   - **重点**：
     - `HBuf` 对 `hbuf_t` 的面向对象封装
     - `HThread` 的线程状态管理（暂停/恢复）
     - 异步任务（`Defer`）的 RAII 实现
   - **实践案例**：用 libhv 实现简单的 WebSocket 服务器，对比原生 C 与 C++ 封装的代码量差异。

#### 架构核心

- **分层设计**：`base/`（C 基础模块）、`cpputil/`（C++ 工具）、`protocol/`（协议栈）
- **跨平台优化**：统一接口屏蔽 Windows/Linux 差异（如 `hio_t` 对 socket 的抽象）
- **性能指标**：单线程支持 10 万 QPS（HTTP 短连接），内存占用比同类库低 30%。

### Nginx（纯 C）

**项目定位**：高并发 Web 服务器（20 万星），异步非阻塞事件驱动架构。

#### 源码解析资源

   - **核心内容**：

     - HTTP 框架（`ngx_http_core_module`）的请求处理流程
   - **特色**：结合 Nginx 1.9.2 中文注释版（[y123456yz/reading-code-of-nginx-1.9.2](https://github.com/y123456yz/reading-code-of-nginx-1.9.2)）。

2. **知乎专栏：Nginx 与 Apache 的架构对比**
   - **重点**：
     - Nginx 的异步非阻塞 vs Apache 的多进程模型
     - `ngx_http_upstream_module` 的负载均衡算法
     - `ngx_http_proxy_module` 的反向代理实现

#### 架构核心

- **模块化设计**：11 个 HTTP 处理阶段（`NGX_HTTP_POST_READ_PHASE` 等）
- **缓存机制**：`ngx_http_file_cache` 的磁盘缓存与内存索引
- **性能优化**：`sendfile` 零拷贝传输与 `TCP_CORK` 优化

## 1.2 存储与数据库

### 1.2.1 LevelDB（C++，Google）

**项目定位**：嵌入式 KV 存储引擎（17.5 万星），LSM-Tree 实现，Redis/SQLite 均有借鉴。

#### 源码解析资源

1. **《LevelDB 源码剖析》（极客时间，吴咏炜）**
   - **核心内容**：
     - LSM 树实现（`MemTable` → `Immutable MemTable` → SSTable）
     - 压缩算法（Snappy + 块缓存 `BlockCache`）
     - 版本管理（`VersionSet` + 红黑树）
   - **特色**：逐文件解析（如 `db_impl.cc` 事务提交流程），对比 RocksDB 优化点。
   - **学习路径**：配套 GitHub 注释版（[sanxiyn/leveldb](https://github.com/sanxiyn/leveldb)）。

2. **博客系列：LevelDB 设计与实现（美团技术团队）**
   - **链接**：[https://tech.meituan.com/leveldb.html](https://tech.meituan.com/leveldb.html)
   - **重点**：
     - SSTable 格式解析（索引块、数据块、Footer）
     - WAL（预写日志）的刷盘策略
     - 迭代器（`DBIterator`）的惰性加载设计

#### 架构核心

- **LSM 树优化**：写放大 vs 读放大的平衡（`WriteBatch` 批量提交）
- **零拷贝设计**：`ReadOptions` 直接映射 SSTable 到内存
- **错误处理**：`Status` 类的状态码与错误信息分离

#### 架构核心

- **线程模型**：单线程处理 + 后台 I/O 线程（`bio`）

## 1.4 分布式系统与 RPC

## 1.13 附录

### 1.13.1 学习方法：四步拆解法

**第一步：绘制调用链**
调试项目启动流程，记录关键调用栈。示例：
- Muduo：`TCPServer` → `EventLoop::loop()` → `EPollPoller::poll()`
- Redis：`main()` → `initServer()` → `aeMain()`
- SPDK：`spdk_nvme_probe()` → `spdk_nvme_identify()`

**第二步：分析数据结构**
研究核心类的成员设计与位掩码策略。示例：
- Muduo：`Channel` 类的 `revents`/`events` 位掩码
- Redis：`dict` 的哈希表冲突处理（链地址法）与 `sds` 的预分配
- SPDK：`spdk_nvme_qpair` 的环形队列设计

**第三步：对比设计模式**
识别架构中使用的设计模式：
- Reactor 模式（`EventLoop` 为反应器）
- 策略模式（`Poller` 接口）
- 工厂模式（`createObject`）

**第四步：性能测试验证**
用压测工具验证源码设计：
- `ab` 压测 Muduo 服务器，对比单线程/多线程差异
- `redis-benchmark` 对比 O(N) 与 O(1) 命令的耗时
- `fio` 压测对比 SPDK 与内核 NVMe 驱动的 IOPS
