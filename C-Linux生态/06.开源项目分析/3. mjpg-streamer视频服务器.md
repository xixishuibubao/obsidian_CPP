# 1. mjpg-streamer 视频服务器

**mjpg-streamer** 是经典 **UVC/V4L2 → MJPEG/JPEG → HTTP/UDP** 流媒体方案，插件化架构，适合理解 Linux 摄像头推流与输出插件机制。可与 [[C-Linux生态/07.Telescope1126实战/5. V4L2与SC285SL曝光控制|V4L2 与 SC285SL 曝光]]（Telescope1126 MJPEG 管线）对照阅读。

## 1.1 架构概览

```plain
┌─────────────┐    ┌──────────────┐    ┌─────────────────┐
│ input_uvc   │───►│ 核心框架      │───►│ output_http     │──► 浏览器 MJPEG
│ (V4L2 采集)  │    │ (插件链)      │    │ output_udp      │
└─────────────┘    └──────────────┘    │ output_file       │
                                        └─────────────────┘
```

| 组件 | 作用 |
|------|------|
| **input 插件** | 从 `/dev/video*` 读帧（UVC、V4L2） |
| **核心** | 加载插件、传递 `video_buffer` |
| **output 插件** | HTTP multipart、UDP、写文件等 |

## 1.2 与 V4L2 的关系

- 依赖内核 **Video4Linux2** 接口（`VIDIOC_STREAMON`、`mmap` 缓冲队列）。
- 常见格式：`MJPEG` 压缩流直接转发；`YUYV` 等需 **ffmpeg/jpeg** 编码插件。
- Telescope1126 自研驱动走 **MPP + 自有 MJPEG 路径**，未直接基于 mjpg-streamer，但 **预览推流** 问题域相同。

## 1.3 典型启动

```bash
# HTTP 8080 输出 MJPEG
mjpg_streamer -i "input_uvc.so -d /dev/video0 -f 30 -r 1280x720" \
              -o "output_http.so -p 8080 -w /www"
```

| 参数 | 含义 |
|------|------|
| `-i` | input 插件 `.so` 及参数 |
| `-o` | output 插件 `.so` 及参数 |
| `-d` | 设备节点 |

## 1.4 插件机制（与 dlopen 同构）

- 各插件编译为独立 `.so`，运行时 **dlopen** 加载。
- 统一 C 接口：init / run / stop；与 [[C-Linux生态/01.Linux环境/2. 动态库与静态库|动态库]]、[[D-系统与架构/02.软件架构设计/5. 插件架构设计|插件架构设计]] 一致。
