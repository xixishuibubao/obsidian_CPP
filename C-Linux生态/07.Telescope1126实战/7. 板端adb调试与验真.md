# 1. 板端 adb 调试与验真

## 【核心精简提纲】

### 记忆主干

相机与 IPC 逻辑必须在真实 RV1126B 板验证；HardwareDriver 内建 adb shell 脚本构成从寄存器到 pybind 再到 Server HTTP 的分层验真体系。日志与 socket 路径是排障第一入口。

### 核心要点清单

- adb 工作流：devices → shell → push/pull；板端 Server 在 `/oem/usr/share/Telescope1126Server/`
- 期望端口 :8999（Tornado）、:8080（preview_server）、:9001（theodolite），用 `ss`/`ps` 确认
- `run_adb.sh` 系列验证 V4L2 控件与曝光时序；`board_test_pybind.sh` 验证 `.so` + notify
- `sheet2` PASS 只代表寄存器层，不代表 App 层 SingleExposure 一定 PASS
- push `.so` 后须重启 Python 进程，已 import 模块不会热替换
- 日志地图：Executor 离线看 http.log；曝光卡住看 notify sock 与 executor.log
- gdb 可 attach Python 进程对 C++ 符号下断点（进阶）

### 关键结论 / 流程

底层 `run_adb.sh` → pybind 冒烟 → Server HTTP/WS 集成 → OTA 后 `verify_preview_api.sh` 回归。

## 1.1 adb 基础工作流

```bash
# 确认设备
adb devices

# 进 shell
adb shell

# 推文件
adb push local.so /oem/usr/share/Telescope1126Server/DriverSO/

# 拉日志
adb pull /userdata/Telescope1126Server/log/http.log .
```

板端关键路径：

| 路径 | 内容 |
|------|------|
| `/oem/usr/share/Telescope1126Server/` | Server 安装目录 |
| `/oem/hardware.ini` | 设备 SN、config |
| `/userdata/Telescope1126Server/log/` | http.log、Driver_API.log、executor.log |
| `/tmp/executor_ipc.sock` | IPC socket |
| `/tmp/preview_server.log` | 预览 C 进程日志（若启用） |

## 1.5 日志排障地图

| 现象 | 先看 |
|------|------|
| Executor 离线 | `http.log` 心跳 timeout；`ss -x` 看 ipc sock |
| 曝光不完成 | `executor.log`；`/tmp/executor_notify.sock` 是否存在 |
| 驱动 connect 失败 | `Driver_API.log`；`import CMOS285CAMERAMain` 在板端 Python 测试 |
| 预览无流 | `/tmp/preview_server.log` 或 streaming 相关 Server 日志 |
| 寄存器不对 | `run_adb.sh sheet2` 输出 vs `test_exposure_calc` 主机对照 |

```bash
adb shell "grep -i error /userdata/Telescope1126Server/log/Driver_API.log | tail -20"
adb shell "tail -30 /userdata/Telescope1126Server/log/http.log"
```

## 1.6 gdb 调试 .so（进阶）

Python 进程加载 `.so` 后，可在 C 层下断点（板端需装 gdb）：

```bash
adb shell
gdb -p $(pidof python3)   # 或 attach 到 executor 进程
# (gdb) set sysroot ...
# (gdb) break CMOS285CAMERAMain::start_exposure
```

详见 [[C-Linux生态/04.调试与优化/1. gdb|gdb 笔记]]。pybind 符号可能被 mangling，用 `info functions exposure` 搜索。

## 1.9 典型错误与边界

| 现象 | 先看 |
|------|------|
| pybind 通过但 Server 任务失败 | `hardware.ini` config 是否连 `hv285_main`；DriverSO 版本是否一致 |
| push 新 `.so` 行为未变 | 未重启 python/executor 进程 |
| 寄存器 PASS 但 FITS 失败 | 问题在 Executor 任务链或后处理，非 V4L2 层 |

**反例**：仅 `board_test_pybind.sh` 通过就关闭曝光时序回归 — App 层还有 IPC、FITS、路径配置。

**正例**：分层验证 — `sheet2` → `board_test_pybind` → `TryStartSimpleTask` HTTP → 查三份日志。
