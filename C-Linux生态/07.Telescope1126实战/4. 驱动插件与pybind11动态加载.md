# 1. 驱动插件与 pybind11 动态加载

Telescope1126 的驱动体系 = **插件架构的 Python 变体**：C/C++ 编译为 `.so`，Python 用 **importlib** 加载，上层 **DriverAPI** 做统一封装。

## 1.1 四层结构

```plain
┌─────────────────────────────────────────┐
│  APIServer/HVDevices/                   │  connect_device('main_camera', 'hv285_main')
├─────────────────────────────────────────┤
│  DriverAPI/CameraDrivers/CMOS285Main.py │  class CMOS285Main(BaseCameraDriverTemplate)
├─────────────────────────────────────────┤
│  DriverSO/CMOS285CAMERAMain.*.so        │  pybind11 模块（C++ 实现）
├─────────────────────────────────────────┤
│  /dev/video12  /dev/v4l-subdev7         │  V4L2 内核接口
└─────────────────────────────────────────┘
```

**映射表**在 `APIServer/HVDevices/AllSupportedDriver.py`：

```python
'hv285_main': ('DriverAPI.CameraDrivers.CMOS285Main', 'CMOS285Main'),
```

`HVDevices.connect_device()` → `importlib.import_module` 加载 Python 类 → Python 类再 `import CMOS285CAMERAMain` 加载 `.so`。

## 1.2 Python 侧如何加载 .so

```python
# DriverAPI/CameraDrivers/CMOS285Main.py（简化）
_driver_so_path = Path(__file__).resolve().parents[2] / "DriverSO"
sys.path.insert(0, str(_driver_so_path))

mod = importlib.import_module("CMOS285CAMERAMain")   # 不含 .so 后缀
cam = mod.CMOS285CAMERAMain(dev_path)
cam.connect()
cam.start_exposure(ms, raw_path)
```

主服务启动时同样追加路径：

```python
# HoriVisionDriverServer.py
sys.path.append('DriverSO')
```

### 1.2.1 与 dlopen 的对比

| 方式 | API | 本项目 |
|------|-----|--------|
| C/C++ 插件 | `dlopen` / `dlsym` | 未在主程序直接使用 |
| Python 扩展 | `import` / `importlib` | **采用** |
| 底层实现 | 仍是 `dlopen` 加载 `.so` | Python 解释器完成 |

## 1.3 pybind11 模块暴露什么

HardwareDriver 编译产物示例：

| .so 模块名 | C++ 绑定文件 | Python 类 | 设备角色 |
|------------|--------------|-----------|----------|
| `CMOS285CAMERAMain` | `CMOS285_main_binding.cpp` | `CMOS285CAMERAMain` | main_camera |
| `CMOS285CAMERAGuide` | `CMOS285_guide_binding.cpp` | `CMOS285CAMERAGuide` | guide_camera |

典型方法（主相机）：

| 方法 | 用途 |
|------|------|
| `connect` / `disconnect` | 打开 V4L2 / 初始化 MPP |
| `start_exposure(ms, raw_path)` | 科学曝光 + 写 RAW |
| `start_streaming` / `stop_streaming` | MJPEG 预览 |
| `pause_streaming_for_science` | 曝光前释放预览占用 |
| `get_jpeg_buffer` | 取最新 JPEG 帧 |
| `set_gain` / `set_offset` | 传感器参数 |

绑定宏示例：

```cpp
PYBIND11_MODULE(CMOS285CAMERAMain, m) {
    py::class_<CMOS285CAMERAMain>(m, "CMOS285CAMERAMain")
        .def("connect", &CMOS285CAMERAMain::connect)
        .def("start_exposure", &CMOS285CAMERAMain::start_exposure,
             py::arg("exposure_ms"), py::arg("raw_path"));
}
```

## 1.4 BaseCameraDriverTemplate 模板层

`DriverAPI/CameraDrivers/BaseCameraDriverTemplate.py` 定义：

- WebSocket 可调用命令列表（`WSBaseDevice` 装饰器体系）
- 曝光后处理子进程（RAW→FITS→JPG）
- 与 `SupportLib` 的 `ProcessWSCommunicator` 协作

**Fake 驱动**（`FakeCamera.py`）与 **真机驱动**（`CMOS285Main.py`）继承同一模板，便于无板开发。

## 1.5 DirectBaseDriver 与动态 import

`APIServer/HVDevices/BaseDevice.py` 中 `DirectBaseDriver.load_driver_package()`：

```python
driver_package = SupportedDevices.get_driver_package(device_type, driver_name)
# 返回 ('DriverAPI.CameraDrivers.CMOS285Main', 'CMOS285Main')
driver_module = importlib.import_module(driver_package[0])
self.driver_object = getattr(driver_module, driver_package[1])()
```

这是 **运行时插件注册表** 模式：新增驱动 = 在 `AllSupportedDriver.py` 加一行 + 实现 DriverAPI 类 + 提供 .so。

## 1.6 DriverSO 目录还放什么

见 `Telescope1126Server/DriverSO/ReadMe.md`：

| 文件 | 来源 |

## 1.7 新增驱动检查清单

1. HardwareDriver 或 BuildDrivers 编译出 pybind `.so` → 复制到 `DriverSO/`
2. 编写 `DriverAPI/.../YourDriver.py` 封装 .so
3. 在 `AllSupportedDriver.py` 注册驱动名
4. 在 `StartUpConfig.py` / `hardware.ini` 的 config 中引用
5. 板端 `board_test_pybind.sh` 冒烟
