# 1. V4L2 与 SC285SL 曝光控制

## 【核心精简提纲】

### 记忆主干

HardwareDriver 把 App 请求的曝光秒数经 `CameraExposure.c` 换算为 V4L2 控件值，在 `/dev/video12` 抓 BG12 RAW，完成后经 datagram notify Executor。预览与科学曝光共用传感器，由 coordinator 互斥调度。

### 核心要点清单

- `/dev/v4l-subdev7` 写传感器控件（VB、exposure、增益）；`/dev/video12` 采集 RAW 流
- 曝光换算依据 Excel「V2修正hts」，`get_exposure_time_linear()` 输出寄存器值
- 长曝光（>~62s）会扩展 h_blanking（HTS），见 `EXPOSURE_SECONDS_MAPPING.md`
- 科学曝光前 `pause_streaming_for_science`，结束后 `resume`，避免与 MJPEG 争用
- 环境变量可覆盖默认设备路径与 notify sock
- 主机可用 `test_exposure_calc` 对照 CSV 验算，无需板子
- 曝光完成走 datagram 而非 Python 阻塞 poll，与 [[3. Unix域套接字IPC实战]] 一致

### 关键结论 / 流程

曝光秒数 → `CameraExposure.c` 换算 → subdev 写控件 → video 抓 RAW → notify → Executor 后处理 FITS。

HardwareDriver 核心任务：把 App 请求的 **曝光秒数** 转成 SC285SL 传感器的 **V4L2 控件**，再抓取 **BG12 RAW** 帧。

## 1.1 V4L2 在本项目中的角色

**Video4Linux2（V4L2）** 是 Linux 采集相机/传感器的标准用户态 API。

| 设备节点 | 典型路径 | 作用 |
|----------|----------|------|
| **video 节点** | `/dev/video12` | 采集 BG12 RAW 视频流 |
| **subdev 节点** | `/dev/v4l-subdev7` | 传感器控件：曝光、增益、blanking |

```plain
App 曝光 3.0s
    → CMOS285Main.py
    → CMOS285CAMERAMain.start_exposure(ms, path)
    → get_exposure_time_linear(seconds)     # CameraExposure.c
    → v4l2 写 vertical_blanking + exposure
    → 抓一帧 RAW 写入 raw_path
    → notify /tmp/executor_notify.sock
```

## 1.2 曝光换算：CameraExposure.c

规格来自 Server 仓库 Excel **「V2修正hts」** 工作表；C 实现为 `get_exposure_time_linear()`：

```c
// 输入：曝光秒数 exposure_seconds
// 输出：写入 V4L2 的 v_blanking、exposure、h_blanking
ExposureTimeRegValue get_exposure_time_linear(float exposure_seconds);
```

关键常量（`CameraExposure.c`）：

| 宏 | 值 | 含义 |
|----|-----|------|
| `PCLK` | 216 MHz | 像素时钟 |
| `HTS_MIN` | 3200 | 行长度相关 |
| `ACTIVE_ROWS` | 1080 | 有效行高 |
| `MIN_EXPOSURE_TIME` | ~14.8 µs | 单行时间 |

关系（中等曝光区）：

```plain
frame_seconds = exposure_seconds + 8 × MIN_EXPOSURE_TIME
VTS_hw = v_blanking + 1080
exp_hw = v_blanking + 1080 - 8
```

长曝光（> ~62s）会扩展 **h_blanking（HTS）**，详见 HardwareDriver 文档 `CMOS/docs/EXPOSURE_SECONDS_MAPPING.md`。

## 1.4 预览与科学曝光互斥

主相机 **同一传感器** 不能同时满速 MJPEG 预览与长曝光 RAW：

```plain
pause_streaming_for_science()
    → start_exposure()
    → resume_streaming_after_science()
```

`cmos285_coordinator.c` 协调 ISP/MJPEG 管线与 V4L2 控件切换。预览侧依赖 **Rockchip MPP + rkaiq**（见 [[6. 交叉编译与DriverSO部署]]）。

## 1.5 环境变量覆盖

C 层默认值（`cmos285_exposure_common.h`）：

```c
#define CMOS285_DEFAULT_SENSOR_DEV   "/dev/v4l-subdev7"
#define CMOS285_DEFAULT_RAW_VIDEO    "/dev/video12"
#define CMOS285_DEFAULT_NOTIFY_SOCK  "/tmp/executor_notify.sock"
```

可通过环境变量 `CMOS285_SENSOR_DEV`、`CMOS285_VIDEO_DEV` 等覆盖（`cmos285_v4l2_backend.cpp` 读取）。

## 1.6 内核侧：sc285sl.c

`CMOS/sysdrv/source/kernel/drivers/media/i2c/sc285sl.c` 为 **I2C 传感器驱动**，与 user space V4L2 栈配合。修改传感器底层时涉及：

- 内核模块编译 / 设备树
- 与 LuoBoCameraSDK `sysdrv` 集成

日常应用层开发多数只碰 **v4l2-ctl / 用户态 backend**，不必改内核。

## 1.9 典型错误与边界

| 误区 | 后果 |
|------|------|
| 混淆预览 FPS 与曝光帧周期 | 误判长曝光时序回归失败 |
| 曝光未 pause 预览 | subdev 控件冲突，RAW 异常或预览挂死 |
| 只改应用层却期望改传感器底层 | 需走内核 `sc285sl.c` + SDK sysdrv，非日常路径 |

**反例**：用 `vertical_blanking` 直接当曝光秒数 — VB 是行数寄存器，须经 `get_exposure_time_linear` 换算。

**正例**：`run_adb.sh sheet2` 写读回对照 Excel + 主机 `test_exposure_calc` 双重验算。
