# 1. LuoBoCameraSDK 打包与 OTA

**Telescope1126Server** 与 **HardwareDriver** 的 `.so` 最终通过 **LuoBoCameraSDK** 打进板端 **oem 分区**，经 **Recovery OTA** 升级。

前置：[[1. 1126B项目总览与学习路线]] · [[6. 交叉编译与DriverSO部署]] · 关联：[[C-Linux生态/03.Linux开发/5. linux构建-buildroot|Buildroot]]

## 1.1 三仓库在开发机上的关系

```plain
code/
├── LuoBoCameraSDK/              固件 SDK（镜像、OTA、自启）
├── Telescope1126Server/         Python 业务 → 打包进 oem
├── Telescope1126HardwareDriver/   C++ 驱动 → 编译到 Server/DriverSO
└── preview_server_30fps/        可选，独立 :8080 预览 C 程序
```

Makefile 引用（缺目录会导致 `./build.sh app` 失败）：

```makefile
# LuoBoCameraSDK/project/app/telescope1126_server/Makefile
TELESCOPE_SRC_DIR := $(SDK_ROOT)/../Telescope1126Server
```

## 1.2 SDK 顶层目录

| 路径 | 用途 |
|------|------|
| `project/build.sh` | 主构建：`app` / `firmware` / `ota` |
| `project/cfg/BoardConfig_Aura/` | Aura 板配置 |
| `project/app/` | 用户态应用（telescope1126_server、preview、theodolite…） |
| `project/scripts/` | OTA manifest、Recovery 脚本 |
| `sysdrv/` | U-Boot、内核、Buildroot rootfs、ko |
| `media/` | Rockchip MPP / ISP 库 |
| `output/image/` | `boot.img`、`oem.img`、`update_ota.tar` |

## 1.3 板端安装位置

| 开发机 | 板端 |
|--------|------|
| `Telescope1126Server/` | `/oem/usr/share/Telescope1126Server/` |
| `DriverSO/*.so` | `/oem/usr/share/Telescope1126Server/DriverSO/` |
| `preview_server` 二进制 | `/oem/usr/bin/preview_server` |
| `hardware.ini`（Deploy 脚本生成） | `/oem/hardware.ini` |
| 运行日志 | `/userdata/Telescope1126Server/log/` |

## 1.4 自启链（上电顺序）

```plain
/etc/init.d/S20linkmount          挂载分区
    → S21appinit
        → RkLunch-RKIPC_RV1126B.sh
            → insmod_ko.sh
            → preview_server &              :8080
            → telescope1126_server.sh &     :8999
            → theodolite_app &              :9001
```

关键脚本：

- `project/app/rkipc/.../RkLunch-RKIPC_RV1126B.sh`
- `project/app/telescope1126_server/scripts/telescope1126_server.sh`
- `project/app/telescope1126_server/config/server_board.ini` → 合并进运行配置

## 1.5 构建命令

```bash
cd LuoBoCameraSDK
source ./build_env.sh          # 可选，修正 Buildroot host 库

./build.sh app                 # 改 Server/应用后
./build.sh firmware            # 完整 boot/rootfs/oem 镜像
./build.sh ota                 # 仅打 update_ota.tar
```

改 **Python 业务** → Server 仓库 + `./build.sh app`  
改 **C++ 驱动** → HardwareDriver `build_all.sh` + `./build.sh app`  
改 **内核/ko/分区** → `./build.sh firmware`

## 1.6 OTA 流程

```bash
# 1. 编辑版本
# project/scripts/ota_manifest.json

# 2. 打包
./build.sh ota
# 产物: output/image/update_ota.tar

# 3. 推板
adb push output/image/update_ota.tar /userdata/
adb shell sh /usr/bin/RK_OTA_trigger_recovery.sh

# 4. 验收
adb shell cat /userdata/ota_version.json
```

Recovery 细节见 SDK 内 `docs/OTA_AURA.md`。

## 1.7 hardware.ini 与 Deploy

Server 仓库 `Deploy/deploy_to_device.py`（**不要打进镜像**，仅开发机用）：

```bash
cd Telescope1126Server/Deploy
python deploy_to_device.py
# → 读 MAC → 生成 Magic → adb push /oem/hardware.ini
```

`hardware.ini` 的 `config` 字段决定 `BeforeStartScript` 连哪套驱动（见 [[1. 1126B项目总览与学习路线#1.6 工程内辅助文档]]）。

OTA 包加密需含 `config` 字段 — 见 Server `doc/更新包加密方式.md`。

## 1.8 RK_APP_TYPE 与模块开关

Aura 板配置示例：

```plain
RK_APP_TYPE="RKIPC_RV1126B RK_APP_RKAI UVC_TINY THEODOLITE_APP TELESCOPE1126_SERVER"
RK_BUILD_APP_TO_OEM_PARTITION=y
```

| 开关 | 产物 |
|------|------|
| `TELESCOPE1126_SERVER` | 打包 Telescope1126Server |
| `THEODOLITE_APP` | 经纬仪 :9001 |
| preview 源码存在即编 | `preview_server` |

## 1.9 开发分工原则（ONBOARDING 摘要）

| 改什么 | 改哪个仓库 |
|--------|------------|
| Python API / 任务 / WS | Telescope1126Server |
| MJPEG/抓帧/电机 HTTP | preview_server 或 HardwareDriver preview |
| 自启、分区、OTA、ko | LuoBoCameraSDK |
| pybind 曝光/V4L2 | Telescope1126HardwareDriver |

## 1.10 板端快速验收

```bash
adb shell "ss -lntp | grep -E '8080|8999|9001'"
adb shell "wget -qO- http://127.0.0.1:8999/preview/status"   # 若含 PreviewAPI
adb shell "cat /userdata/ota_version.json"
```

Server 测试脚本可由 SDK Makefile 安装到 `/userdata/scripts/`（如 `verify_preview_api.sh`）。
