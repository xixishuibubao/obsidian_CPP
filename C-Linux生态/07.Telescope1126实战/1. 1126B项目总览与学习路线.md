# 1. 1126B 项目总览与学习路线

## 【核心精简提纲】

### 记忆主干

Telescope1126 由 **Server**（Python 编排）与 **HardwareDriver**（C++/pybind 硬件）两仓组成，经 `DriverSO/*.so` 衔接；板端 RV1126B 上多进程分端口对外服务。本专题把大工程拆成可复习模块，并映射到 vault 已有 Linux、异步与插件笔记。

### 核心要点清单

- Server 管 HTTP/WebSocket :8999、任务编排与 Executor 监管；HardwareDriver 管 V4L2 曝光与 RAW/MJPEG
- 物理桥梁是交叉编译产物放入 `Telescope1126Server/DriverSO/`，Python 用 importlib 加载
- 完整板端栈还含 preview_server :8080、theodolite :9001 与 LuoBoCameraSDK 固件打包
- 推荐四周路径：Server 架构 → IPC/任务 → 驱动/V4L2 → 编译上板与 FITS/OTA
- 工程内 HTML 与 `ONBOARDING_1126_LUOBO_STACK.md` 是入口地图，优先对照结构图阅读
- P0 先补 Python import 与 HTTP/WS；P1 补 fd/socket、JSON；P2+ 再深入 asyncio、V4L2、pybind
- 运行环境是嵌入式 Linux 板，非 Windows 桌面；IPC 与 V4L2 须在板端或 Linux 开发机验证

### 关键结论 / 流程

Server 编排 → Executor 子进程 → DriverAPI → `.so` → V4L2/MPP；固件经 SDK 打入 oem 分区 OTA 上板。

## 1.1 项目是什么

| 项目                              | 路径（开发机）                             | 语言               | 职责                                           |
| ------------------------------- | ----------------------------------- | ---------------- | -------------------------------------------- |
| **Telescope1126Server**         | `code/Telescope1126Server/`         | Python + Tornado | HTTP/WebSocket API、任务编排、设备管理                 |
| **Telescope1126HardwareDriver** | `code/Telescope1126HardwareDriver/` | C/C++ + pybind11 | SC285SL 相机 V4L2 曝光、RAW 采集、MJPEG 预览           |
| **LuoBoCameraSDK**（外部）          | 与上两者同级或独立 clone                     | Buildroot/镜像     | 固件打包、OTA、板端自启                                |
| **preview_server**（外部 C 工程）     | 见 ONBOARDING 文档                     | C                | 早期 :8080 MJPEG 独立进程（与 HardwareDriver 预览库有渊源） |

运行环境：**RV1126B** 嵌入式 Linux 板（Aura），不是 Windows 桌面程序。

## 1.2 两个仓库如何衔接

```plain
HardwareDriver  CMOS/build_all.sh 编译
        │
        ▼
Telescope1126Server/DriverSO/*.so    ← 物理桥梁
        │
        ▼
DriverAPI/CameraDrivers/CMOS285Main.py   importlib 加载 .so
        │
        ▼
APIServer / executor.py  调用驱动完成曝光任务
```

核心概念：**Python 是壳，`.so` 是硬件能力**。这与 [[D-系统与架构/02.软件架构设计/5. 插件架构设计|插件架构设计]] 中的「主程序 + 动态模块」一致，只是这里用 **pybind11 + importlib** 而不是 `dlopen`。

## 1.3 板端进程与端口（完整栈）

| 端口 | 进程 | 职责 |
|------|------|------|
| **8999** | `HoriVisionDriverServer.py` | 对外 API、WebSocket、Executor 监管 |
| **8080** | `preview_server`（C） | MJPEG 预览、部分电机 HTTP（视固件版本） |
| **9001** | `theodolite_app` | 经纬仪 MCU |

本地 Server 仓库侧重 **8999 + Executor + DriverSO**；预览编排（PreviewOrchestrator）在部分分支/固件中更完整，阅读 `code/ONBOARDING_1126_LUOBO_STACK.md` 可补全图景。

## 1.8 适用边界与典型误区

| 边界 | 说明 |
|------|------|
| 开发机 Windows | 可读 Python/C 源码，**不能**跑 Unix Socket IPC 与板端 V4L2 |
| 本地只 clone Server | 无 HardwareDriver 则缺 `.so` 与曝光验算工具 |
| 只看 :8999 | 忽略 :8080 preview_server 会误判 MJPEG 架构 |

**反例**：在 Windows 上直接 `python HoriVisionDriverServer.py` 期望连 Executor — UDS 不存在，IPC 必失败。

**正例**：Linux 开发机先 `build_all.sh` 填 DriverSO，再读 `FakeCamera.py` 无板跑通驱动加载链路。
