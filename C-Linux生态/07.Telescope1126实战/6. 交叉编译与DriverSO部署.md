# 1. 交叉编译与 DriverSO 部署

## 【核心精简提纲】

### 记忆主干

HardwareDriver 在 x86 Linux 开发机交叉编译 aarch64 pybind `.so`，产物安装到 `Telescope1126Server/DriverSO/`，再随 Server 打包上板。工具链与 Python ABI 必须与板端 LuoBoCameraSDK 一致。

### 核心要点清单

- 入口 `CMOS/build_all.sh`：定位 SDK 工具链 → CMake 编 MPP/rkaiq → 复制到 DriverSO
- `build_bindings.sh` 简化版不链 preview/MPP，适合快速验证 binding
- Main 模块链 rockchip_mpp + rkaiq；Guide 模块通常只链 pthread
- 产物名含 `cpython-312-aarch64-linux-gnu` 后缀，板端 Python 版本须匹配
- libcfitsio 等用 ctypes `CDLL` 加载，与 pybind `import` 方式不同
- Windows 无法直接跑编译脚本；缺 SDK 时脚本 WARN 但可能链接失败
- 改 C++ 后须 push `.so` 并重启 Python 进程（已 import 模块不热替换）

### 关键结论 / 流程

改源码 → `build_all.sh` → DriverSO → adb push/rsync → 重启服务 → `board_test_pybind.sh` 冒烟。

## 【完整拓展详情】


**Telescope1126HardwareDriver** 在 x86 Linux 开发机上 **交叉编译** aarch64 的 pybind `.so`，安装到 **Telescope1126Server/DriverSO/**，再随 Server 打包上板。

关联：[[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|交叉编译优化选项]] · [[C-Linux生态/03.Linux开发/4. linux构建-buildroot|Buildroot]] · [[4. 驱动插件与pybind11动态加载]]

## 1.1 编译脚本入口

```bash
cd Telescope1126HardwareDriver/CMOS
./build_all.sh
```

脚本逻辑（`build_all.sh` 摘要）：

| 步骤 | 内容 |
|------|------|
| 1 | 定位 `LuoBoCameraSDK`（`../../LuoBoCameraSDK`） |
| 2 | 使用 `aarch64-rockchip1240-linux-gnu-gcc/g++` |
| 3 | CMake 配置 MPP/rkaiq 头文件与库路径 |
| 4 | 链接 Python 3.12 sysroot |
| 5 | 产物复制到 `../Telescope1126Server/DriverSO/` |

简化版：`build_bindings.sh`（不链 preview/MPP，仅 binding + backend，适合快速验证）。

## 1.2 关键路径变量

```bash
SDK_ROOT="${SDK_ROOT:-../../LuoBoCameraSDK}"
TOOLCHAIN="${SDK_ROOT}/tools/linux/toolchain/aarch64-rockchip1240-linux-gnu"
MPP_INCLUDE="${SDK_ROOT}/output/out/media_out/include/rockchip"
MPP_LIB="${SDK_ROOT}/output/out/media_out/lib"
SYSROOT="${SDK_ROOT}/sysdrv/.../arm64-buildroot-linux-gnu/sysroot"
DRIVER_SO="${TELESCOPE_SERVER_ROOT}/DriverSO"
```

缺 SDK 时会 WARN；**Windows 上无法直接跑此脚本**。

## 1.3 CMake 产物

`CMOS/src/CMakeLists.txt` 生成两个模块：

| 目标 | 源文件 | 链接库 |
|------|--------|--------|
| `CMOS285CAMERAMain` | main_binding + preview/* + backend | rockchip_mpp, rockit, rkaiq, pthread |
| `CMOS285CAMERAGuide` | guide_binding + backend | pthread |

安装名（Python ABI 后缀固定）：

```plain
CMOS285CAMERAMain.cpython-312-aarch64-linux-gnu.so
CMOS285CAMERAGuide.cpython-312-aarch64-linux-gnu.so
```

**Python 版本与 ABI 必须匹配板端**（3.12 aarch64）。

## 1.4 CMake 与优化

```cmake
# CMakeLists.txt 默认 Release（避免 -O0 导致预览 FPS 瓶颈）
if(NOT CMAKE_BUILD_TYPE)
    set(CMAKE_BUILD_TYPE Release)
endif()
target_compile_definitions(CMOS285CAMERAMain PRIVATE ENABLE_RK_MPP ENABLE_ISP_MJPEG)
```

RV1126B 与 [[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|RK3588 交叉编译笔记]] 同属 ARM64，但 **工具链前缀不同**（rockchip1240 vs 3588 工具链），勿混用。

## 1.5 DriverSO 其它依赖库

`DriverSO/ReadMe.md` 列出需手动交叉编译的库：

| 库 | 用途 | 构建提示 |
|----|------|----------|
| `libcfitsio.so` | FITS 读写 | cmake + 指定 sysroot |
| `libastroc.so` | 天文坐标 | horivision/ASTROC |
| raw2fits .so | RAW 转 FITS | SupportLibReplace v2 |

Python 侧 `SupportLib/fits.py`：

```python
cfitsio = CDLL('DriverSO/libcfitsio.so')
```

## 1.7 常见失败原因

| 现象 | 可能原因 |
|------|----------|
| `ImportError: CMOS285CAMERAMain` | .so 未复制到 DriverSO 或 ABI 后缀不对 |
| `wrong ELF class` | 用 x86 .so 上了 aarch64 板 |
| undefined symbol MPP | MPP_LIB 路径错或未编 media |
| Python version mismatch | 板子 3.11 却编了 3.12 后缀 |

排查：

```bash
file DriverSO/CMOS285CAMERAMain*.so    # 应显示 ARM aarch64
ldd DriverSO/CMOS285CAMERAMain*.so     # 在板端查依赖
python3 -c "import CMOS285CAMERAMain"  # 在板端 PYTHONPATH=DriverSO
```

## 1.8 开发工作流（改 C++ 后）

```plain
1. 改 CMOS/src 或 preview/src
2. ./build_all.sh
3. adb push DriverSO/*.so 到板子对应目录（或 rsync 整包）
4. 重启 telescope1126_server.sh
5. board_test_pybind.sh / run_adb.sh sheet2
```

## 1.9 典型错误与边界

| 现象 | 根因 |
|------|------|
| `wrong ELF class` | x86 `.so` 上了 aarch64 板 |
| undefined symbol MPP | MPP_LIB 路径错或未编 media |
| ImportError 模块名 | ABI 后缀与板端 Python 版本不一致 |

**反例**：用 RK3588 工具链编 1126B 驱动 — 前缀不同（rockchip1240 vs 3588），勿混用。

**正例**：板端 `file` + `ldd` 确认 ARM aarch64，`python3 -c "import CMOS285CAMERAMain"` 在 DriverSO 路径下通过。
