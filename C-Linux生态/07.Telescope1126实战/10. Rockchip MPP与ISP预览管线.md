# 1. Rockchip MPP 与 ISP 预览管线

**Telescope1126HardwareDriver** 主相机预览：V4L2 采集 BG12 → **Rockchip MPP + rkaiq ISP** 硬件 JPEG → 内存 `latest_jpeg` → Python `get_jpeg_buffer()`。

前置：[[5. V4L2与SC285SL曝光控制]] · [[6. 交叉编译与DriverSO部署]] · 关联：[[C-Linux生态/03.Linux开发/1. 讯为RK3588|RK 平台笔记]]

## 1.1 为什么需要 MPP/ISP

| 环节 | 纯 CPU | 本项目 |
|------|--------|--------|
| BG12 RAW → 可视 JPEG | 慢，占 CPU | **MPP VENC JPEG** 硬件编码 |
| 3A（AE/AWB/AF） | 需自研 | **rkaiq** 加载 IQ 文件 |
| 1080p 30fps 预览 | 难达标 | `cmos285_isp_mjpeg.c` 目标 ~30fps |

RK1126B 自带 **ISP 硬件块**；用户态通过 Rockchip **`rk_mpi_*`** API 访问，不是写 `/dev/video` 就自动有 JPEG。

## 1.2 源码文件分工

```plain
CMOS/preview/src/
├── cmos285_preview_capture.c   V4L2 打开 video 节点、采集线程
├── cmos285_isp_mjpeg.c         rkaiq + VI bind VENC(JPEG)
├── cmos285_coordinator.c       预览 ↔ 科学曝光 互斥
├── cmos285_science_raw.c       科学 BG12 单帧
└── cmos285_exposure_common.c   公共 notify / 默认设备路径
```

绑定层：`CMOS285_main_binding.cpp` 暴露 `start_streaming`、`get_jpeg_buffer` 等。

## 1.3 管线概念（VI → VENC）

```plain
Sensor SC285SL
    → /dev/video12 (BG12 RAW)
    → VI (Video Input, rk_mpi_vi)
    → ISP / rkaiq (iq 目录 + .json)
    → VENC JPEG (rk_mpi_venc)
    → g_latest_jpeg 缓冲
    → Python get_jpeg_buffer()
```

`cmos285_isp_mjpeg.c` 头文件引用：

```c
#include <rk_aiq_user_api2_sysctl.h>
#include "rk_mpi_vi.h"
#include "rk_mpi_venc.h"
```

常量示例：

```c
#define VI_REAL_PIPE_ID    0
#define VENC_CHN           0
static uint8_t *g_latest_jpeg;   // 最新一帧 JPEG
static pthread_mutex_t g_frame_lock;
```

**不做 RTSP** — 帧留在进程内给 Server 中继，而非独立 `preview_server` HTTP（早期架构有独立 C 进程，见 [[11. LuoBoCameraSDK打包与OTA]]）。

## 1.4 preview API（C 层）

见 `cmos285_preview_capture.h`：

| 函数 | 作用 |
|------|------|
| `cmos285_preview_init` | 一次性 V4L2 + MPP 初始化 |
| `cmos285_preview_start` | 后台采集编码线程 |
| `cmos285_preview_copy_jpeg` | 拷贝最新 JPEG 到调用者 buffer |
| `cmos285_preview_pause_for_science` | 科学曝光前停预览 |
| `cmos285_preview_resume_after_science` | 曝光后恢复 |
| `cmos285_preview_set_controls` | subdev 曝光/增益 |

配置结构体字段：`video_dev`、`subdev`、`iq_dir`、`width/height`、`jpeg_quality`、`preview_fps`。

## 1.5 与科学曝光互斥

同一 sensor **不能** 同时满速 MJPEG 与长曝光 RAW：

```plain
pause_streaming_for_science()
    → coordinator 停 VENC/VI 或让出 subdev
start_exposure() → V4L2 写 VB/exp → 抓 RAW
resume_streaming_after_science()
```

预览 FPS 与曝光帧周期是**两套指标**（HardwareDriver 文档强调勿混淆）。

## 1.6 编译依赖（CMake）

```cmake
target_compile_definitions(CMOS285CAMERAMain PRIVATE ENABLE_RK_MPP ENABLE_ISP_MJPEG)
target_link_libraries(CMOS285CAMERAMain PRIVATE rockchip_mpp rockit rkaiq pthread)
```

头文件/库来自 **LuoBoCameraSDK**：

```bash
MPP_INCLUDE=${SDK}/output/out/media_out/include/rockchip
MPP_LIB=${SDK}/output/out/media_out/lib
```

缺 MPP 时链接失败或预览 FPS 为 0。

## 1.7 CMake Release 的重要性

```cmake
# CMOS/src/CMakeLists.txt
# -O0 会使 BG12→灰度等 CPU 路径成为 FPS 瓶颈
set(CMAKE_BUILD_TYPE Release)
```

嵌入式图像管线：**Debug 构建会误导性能结论**。

## 1.8 Server 侧如何消费 JPEG

两种架构（视固件版本）：

| 方案 | 说明 |
|------|------|
| **进程内** | `CMOS285CAMERAMain.get_jpeg_buffer()` → Server WS/HTTP 推送 |
| **独立 preview_server :8080** | C 进程 HTTP `/stream`；Server `StreamRelay` 中继（ONBOARDING 1.0.52+） |

读代码前先确认板端跑哪条路径（`ps | grep preview`）。

## 1.9 常见问题

| 现象 | 排查 |
|------|------|
| 预览黑屏 | `iq_dir` / rkaiq JSON 是否部署 |
| ~12fps 而非 30fps | 是否每帧写 `/tmp/latest_frame.raw`（旧 preview_server bug） |
| 曝光后预览挂死 | 是否忘记 `resume_streaming_after_science` |
| 紫边/AWB 异常 | `g_awb_purple_fixed` 等 ISP 调参逻辑 |

```bash
adb shell "tail -30 /tmp/preview_server.log"
# 或 Driver_API.log 中 streaming 相关行
```
