# 1. Rockchip 分区与 parameter.txt

RK 系列芯片（RK3568、RK3588、RV1126 等）通过 **parameter.txt** 中的 **CMDLINE** 行定义 eMMC/SD/NAND 分区表；末分区可用 **`-` 尺寸 + `:grow` 标志** 自动占满剩余存储，实现自适应扩容。

## 【核心精简提纲】

### 记忆主干

**`parameter.txt` → `CMDLINE: mtdparts=...` → 打包 `update.img` 时生效**；末分区写 `-@起始地址(分区名:grow)` 即可吃掉磁盘剩余空间，换不同容量存储无需手算末分区大小。

### 核心要点清单

- 分区格式：`大小@起始扇区(名称[:标志])`，逗号分隔；扇区单位 **512 字节**
- **`-`** 作大小表示「延伸至介质末尾」；须配合 **`:grow`** 写在分区名后
- **仅最后一个分区** 可使用 grow；要让别的分区扩容，须把它挪到表末
- 改中间分区大小时，须连锁更新后续分区的 **起始扇区**（上一起始 + 上一大小）
- 配置文件多在 `device/rockchip/<soc>/parameter-*.txt` 或板级 `BoardConfig` 目录
- 与 [[C-Linux生态/03.Linux开发/4. linux构建-buildroot|Buildroot]] 的 genimage 分区表是不同体系；RK SDK 固件以 parameter.txt 为准

### 关键结论 / 流程

```plain
定位 parameter.txt → 确认末分区 → 改 size 为 - 并加 :grow → 重打 firmware/update.img → 烧录验收
```

## 1.1 parameter.txt 与 CMDLINE

Rockchip Linux SDK 在打包固件（`update.img`）时读取 **parameter.txt**。其中 **CMDLINE** 行的 `mtdparts=` 字段描述整盘分区布局，U-Boot、内核与烧录工具（`rkdeveloptool` 等）共用这一描述。

典型文件头部字段：

```plain
FIRMWARE_VER: 1.0
MACHINE_MODEL: RK3568
TYPE: GPT
CMDLINE: mtdparts=rk29xxnand:0x00002000@0x00004000(uboot),0x00002000@0x00006000(misc),0x00010000@0x00008000(boot),0x00010000@0x00018000(recovery),0x00010000@0x00028000(backup),0x00c00000@0x00038000(rootfs),0x00040000@0x00c38000(oem),-@0x00c78000(userdata:grow)
```

| 字段 | 含义 |
|------|------|
| `mtdparts=` 前缀 | 分区表载体标识（如 `rk29xxnand`、`rk3566mmc`），与介质类型对应 |
| `0xSIZE@0xSTART(name)` | 单个分区：大小、起始扇区、名称 |
| `name:flag` | 可选标志，见下节 |

> 修改 parameter.txt 后须 **重新打包固件**（如 `./build.sh firmware`），单独 OTA 应用分区通常不会改写整盘分区表。

## 1.2 分区条目语法

每条分区遵循：

```plain
[大小]@[起始扇区]([名称][:可选标志])
```

### 大小与地址

- **扇区 = 512 字节**（Rockchip 官方约定）
- 换算示例：`0x00002000` 扇区 = 8192 × 512 B = 4 MiB

```plain
0x00002000 * 512 byte = 4 MiB
```

- **起始扇区** = 上一分区起始 + 上一分区大小（十六进制加法）
- 调整任一中间分区大小时，必须从该分区起 **依次重算** 后续所有起始地址

### 可选标志

| 标志 | 作用 |
|------|------|
| `bootable` | 设置 legacy_boot，U-Boot 在该分区查找 `extlinux.conf` |
| `grow` | 分区延伸至存储介质末尾（自适应剩余容量） |

`:grow` 必须与 **`-` 尺寸** 配合使用在 **分区表最后一项**：

```plain
-@0x00c78000(userdata:grow)
```

含义：

- **`-`**：大小不写死，由工具在打包/烧录时按介质总容量计算
- **`:grow`**：声明该分区为 grow 分区，占满起点至盘末的全部空间

## 1.3 自适应扩容（grow）机制

### 基本用法

将 **最后一个分区** 的固定大小改为 `-`，并在括号内名称后加 `:grow`：

```plain
# 固定 2G userdata（换大容量 eMMC 会浪费或未用尽）
0x00400000@0x00c78000(userdata)

# 自适应：任意容量 eMMC 均把剩余空间给 userdata
-@0x00c78000(userdata:grow)
```

换用 8 GiB、32 GiB、128 GiB eMMC 时，**前面分区布局不变**，userdata 自动变大，无需为每种容量维护多份 parameter 文件。

### 让非末分区承担剩余空间

grow **只能作用于表末一项**。若希望 **rootfs** 而非 userdata 吃剩余空间，须 **调整分区顺序**，把目标分区移到末尾：

```plain
# 原序：rootfs → oem → userdata(grow)
0x00c00000@0x00078000(rootfs),0x00040000@0x00c78000(oem),-@0x00cb8000(userdata:grow)

# 改序：oem → userdata(固定) → rootfs(grow)
0x00040000@0x00078000(oem),0x00200000@0x000ba000(userdata),-@0x002ba000(rootfs:grow)
```

### 新增分区时的注意点

在 grow 分区 **之前** 插入新分区时，须：

1. 为新分区指定 **固定** `大小@起始`
2. 重算 grow 分区的起始地址
3. grow 项仍为表末，保持 `-` 与 `:grow`

```plain
# 在 amp 与 userdata 之间插入 test（1 MiB 量级示例）
...,0x00100000@0x00cba000(test),-@0x00dba000(userdata:grow)
```

## 1.4 与挂载、文件系统的关系

parameter.txt 只定义 **块设备分区边界**；分区内的 ext4/squashfs 镜像大小在 Buildroot/SDK 构建阶段另配。常见组合：

| 场景 | 做法 |
|------|------|
| userdata 仅存日志/配置 | grow 到整盘剩余，首次挂载由 init 或 `resize2fs` 扩文件系统 |
| rootfs 只读 squashfs + overlay | rootfs 分区固定，userdata grow 承载可写层 |
| OTA 仅更新 oem | 不动 parameter.txt；改分区表须 full firmware 重烧 |

板端挂载脚本（如 Aura 的 `S20linkmount`）依赖分区名与偏移；改表后须同步检查 **fstab / linkmount** 与 **package-file** 中的分区列表。

## 1.5 修改检查清单

```plain
1. 备份原 parameter.txt
2. 用 parted/sgdisk 或官方文档核对 eMMC 总扇区数
3. 从前往后验证：start[i+1] == start[i] + size[i]
4. 仅最后一项使用 - 与 :grow
5. 若删分区：同步删 package-file、rootfs_overlay 挂载项、BoardConfig 引用
6. ./build.sh firmware → 烧录 → lsblk/parted 验收末分区容量
```

## 1.6 踩坑

| 现象 | 原因 / 处理 |
|------|-------------|
| 烧录后末分区仍很小 | 未用 `-` 或未加 `:grow`；或 grow 不在最后一项 |
| 打包失败 / 分区重叠 | 中间分区改大小后未更新后续起始地址 |
| 板端无法挂载 userdata | 分区名与挂载脚本不一致；或镜像大于新分区边界 |
| 换盘容量后 rootfs 不够 | rootfs 为固定大小，应把需扩容的分区挪到表末并 grow |
| OTA 后分区没变 | OTA 包通常不含 parameter 重分区；需 full firmware |

## 1.7 与其他构建系统的对应

Yocto **meta-rockchip** 可通过 `RK_PARTITION_GROW=1` 在生成 parameter 时自动把末项改为 `-@...:grow` 形式，本质与手写 parameter.txt 相同。

RK SDK 路径示例：

```plain
device/rockchip/rk3588/parameter-ubuntu-fit.txt
device/forlinx/.chips/ok3562/parameter-buildroot-*.txt
project/cfg/BoardConfig_Aura/   # 板级可能软链或覆盖 parameter
```
