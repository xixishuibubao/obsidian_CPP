# 1. 交叉编译优化选项（RK3588）

RK3588（Cortex-A76/A55, AArch64/ARMv8.2-A）交叉编译时 GCC 的 `-march` 与优化选项。

## 【核心精简提纲】

### 记忆主干

**交叉编译禁用 `-march=native`**；RK3588 原生 `armv8.2-a` + `-mtune=cortex-a76`；图像/算法需 SIMD（ARMv8.2 默认带）；用 `objdump` 查 NEON 指令验证。

### 核心要点清单

- 兼容性优先：`armv8-a`；板端性能：`armv8.2-a -mtune=cortex-a76`
- `-march` 决定**可用指令**；`-mtune` 只调**调度**，不改变 ISA
- 常用组合：`-O3 -march=armv8.2-a -mtune=cortex-a76 -ffast-math -fopenmp`
- 嵌入式减体积：`-fno-rtti -fno-exceptions`
- 验证：`aarch64-linux-gnu-objdump -d prog | grep -E "vadd|vmul|ld1"`

### 快速选择表

| 目标 | `-march` |
|------|----------|
| 最大兼容性 | `armv8-a` |
| 图像/算法 + SIMD | `armv8.2-a` |
| 极致性能 | `armv8.2-a -mtune=cortex-a76` |

## 【完整拓展详情】

## 1.1 RK3588 `-march` 配置

### 推荐写法

```bash
# 基础通用（兼容性最好，可跑在多数 AArch64 板）
-march=armv8-a

# 显式 SIMD 扩展写法（老文档常见）
-march=armv8-a+simd

# 调度优化（不改变指令集）
-march=armv8-a+simd -mtune=cortex-a76

# RK3588 原生 ARMv8.2-A（推荐生产默认）
-march=armv8.2-a -mtune=cortex-a76
```

> 通用 `-march` 规则（交叉编译禁用 `native`、ARMv8 SIMD 默认行为、`-mtune` vs `-march`）详见 [[B-构建与脚本/01.构建工具/1. g++|g++: -march 核心规则]]。

### 前置依赖

- 安装匹配 triple 的工具链：`aarch64-linux-gnu-gcc`（或 Buildroot/Yocto 生成的 SDK）
- 理解目标 SoC 的 **ISA 级别**（RK3588 = ARMv8.2-A），见 [[D-系统与架构/01.架构体系/1. ARM与X86|ARM 与 X86]]

### 适用边界

| 场景 | 建议 |
|------|------|
| 仅 RK3588 / 同代 A76 板 | `armv8.2-a -mtune=cortex-a76` |
| 二进制需跑多种 AArch64 板 | 保守 `armv8-a`，牺牲新指令 |
| 内核模块 / 驱动 KO | 与 **板端运行内核** 的 vermagic、工具链一致，见 [[C-Linux生态/03.Linux开发/1. 讯为RK3588|RK3588]] |

### 易混概念

- **`-march=native` 在交叉编译无意义**：会在 build 机上探测，产物可能在 ARM 板上非法指令
- **`+simd` 在 armv8.2-a 上常冗余**：v8-A 基线已含 NEON；写 `armv8.2-a` 即可表达 RK3588

## 1.2 完整优化组合

```bash
aarch64-linux-gnu-g++ -O3 -march=armv8.2-a -mtune=cortex-a76 \
    -ffast-math -fopenmp -fno-rtti -fno-exceptions \
    -o app main.cpp
```

| 选项 | 作用 | 注意 |
|------|------|------|
| `-O3` | 激进优化、自动向量化 | 编译慢；调试困难 |
| `-march=armv8.2-a` | 允许 v8.2 指令 | 旧板可能不能跑 |
| `-mtune=cortex-a76` | 针对 A76 流水线调度 | 不改变最低 CPU 要求 |
| `-ffast-math` | 放宽 IEEE，加速浮点 | 影响精度与 NaN 语义 |
| `-fopenmp` | OpenMP 并行 | 需链接 `-lgomp` |
| `-fno-rtti -fno-exceptions` | 减小体积 | 禁用 C++ 异常/RTTI |

### 踩坑点

- `-ffast-math` 与严格数值库（部分 BLAS）结果不一致
- OpenMP 线程数默认占满核，板端需设 `OMP_NUM_THREADS`
- 混用不同 GCC 版本编 KO 与用户态 → 偶发 ABI/C++ ABI 问题

## 1.3 验证 SIMD 是否开启

```bash
aarch64-linux-gnu-objdump -d your_program | grep -E "vadd|vmul|ld1"
```

能搜到 `ld1` / `vadd` 等向量指令说明自动向量化或 intrinsics 已生效。

### 正反示例

- **正**：热点循环在 `-O3 -march=armv8.2-a` 下出现 NEON 指令
- **反**：仅 `-O0` 或未指定 march → 纯标量 `fadd`，图像卷积慢一个数量级
