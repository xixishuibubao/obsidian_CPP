# 1. 讯为 RK3588 驱动编译

讯为 RK3588 开发板（`ARCH=arm64`）上 Linux 内核模块的编译、配置与板端加载流程。

## 【核心精简提纲】

### 记忆主干

**模块编译依赖已编译过的同版本内核树**；`Makefile` 用 `make -C $(KDIR) M=$(PWD) modules` 挂接 Kbuild；交叉编译设 `ARCH=arm64` + `CROSS_COMPILE`；板端 `insmod` 需 vermagic 与运行内核一致。

### 核心要点清单

- 最小模块：`module_init` / `module_exit` + `MODULE_LICENSE("GPL")`，源文件须 C99
- 外置模块：`obj-m += xxx.o`，在内核树外独立目录编译
- 内置进内核：`Kconfig` + `obj-$(CONFIG_xxx) +=`，父目录 `source` 引入
- `menuconfig`：`[ ]` 二态、`<>`` 三态（M/*/空）、`()` 填 hex/字符串
- 部署：`scp` 传 `.ko` → `insmod` / `rmmod` / `modinfo` 验真

### 关键流程

```plain
准备内核树(KDIR) → 写 .c + Makefile → make -j → scp 到板 → insmod → dmesg/lsmod
```

## 【完整拓展详情】

## 1.1 要点与流程

- **编译模块的环境必须曾经编译过内核**；模块源必须是符合 C99 的 `.c` 文件（不能用 C++ 直接写 `.ko`）。

### 1.1.1 最小内核模块

```c
#include <linux/module.h>
#include <linux/init.h>

static int myinit(void) {
    printk("myinit\n");
    return 0;
}

static void myexit(void) {
}

module_init(myinit);
module_exit(myexit);
MODULE_LICENSE("GPL");
/* MODULE_AUTHOR / MODULE_VERSION 可选，会出现在 modinfo 中 */
```

### 1.1.2 Makefile 模板（外置模块）

```makefile
export ARCH=arm64
export CROSS_COMPILE=/path/to/aarch64-none-linux-gnu-

KDIR=/path/to/kernel
PWD?=$(shell pwd)

obj-m += my.o

all:
    make -C $(KDIR) M=$(PWD) modules
    # -C 切到内核树；M= 指向模块源码目录

clean:
    rm -f *.o *.mod *.mod.c *.order *.ko
```

> 若不在 Makefile 里 `export`，须在命令行显式传入：`make -C $(KDIR) M=$(PWD) ARCH=arm64 CROSS_COMPILE=... modules`

### 1.1.3 编译与部署

1. 在模块目录执行 `make -j`。
2. 交叉编译完成后 `scp` 到开发板：`scp user@ip:/abs/path/xxx.ko ./`
3. 板端加载与卸载：

```bash
sudo insmod xxx.ko
lsmod
sudo rmmod xxx
modinfo xxx.ko    # 作者、license、vermagic、架构
```

### 踩坑

| 现象 | 原因 / 处理 |
|------|-------------|
| `Invalid module format` / vermagic 不符 | 模块用 A 版本内核编，板端跑 B 版本；换匹配内核树或板端内核 |
| `disagrees about version of symbol` | 内核配置或补丁不一致，需与板端运行内核同源 |
| `Unknown symbol` | 依赖未导出符号，检查 `EXPORT_SYMBOL` 或内核 config |
| 编译通过但功能异常 | `KDIR` 指向错误架构或错误 defconfig |

## 1.2 make menuconfig 与 Kconfig

1. `make menuconfig`：空格切换选项。
   - `[ ]`：`[*]` 内置 / `[ ]` 不编
   - `< >`：`[M]` 模块 / `[*]` 内置 / `[ ]` 不编
   - `( )`：填十六进制或字符串
2. 使用前 `export ARCH=arm64`（RK3588 为 arm64）。
3. `make xxx_defconfig` 从 `arch/arm64/configs/` 生成 `.config`。

### Kconfig 示例（内置驱动）

```kconfig
config helloworld
    bool "helloworld support"
    default y
    help
        my helloworld
```

对应目录 `Makefile`：

```makefile
obj-$(CONFIG_helloworld) += helloworld.o
```

在父级 `Kconfig` 用 `source "drivers/char/helloworld/Kconfig"` 引入（类似 `#include`）。
