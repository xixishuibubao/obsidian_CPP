# 1. 交叉编译优化选项（RK3588）

RK3588（Cortex-A76/A55, AArch64/ARMv8.2-A）交叉编译时 GCC 的 `-march` 与优化选项。

相关笔记：[[1. 讯为RK3588|RK3588 驱动编译]]、[[B-构建与脚本/01.构建工具/1. g++|g++ 编译命令]]（通用交叉编译参数详见 g++ 笔记）

## 1.1 RK3588 `-march` 配置

### 推荐写法

```bash
# 基础通用（兼容性最好）
-march=armv8-a

# 开启 NEON/SIMD（图像/算法必加）
-march=armv8-a+simd

# 针对 Cortex-A76/A55 调度优化（不改变指令集）
-march=armv8-a+simd -mtune=cortex-a76

# ARMv8.2-A（RK3588 原生支持，默认带 SIMD）
-march=armv8.2-a -mtune=cortex-a76
```

> 通用 `-march` 规则（含交叉编译禁用 `native`、ARMv8 SIMD 默认行为、`-mtune` vs `-march` 区别）详见 [[B-构建与脚本/01.构建工具/1. g++#2.3.2 `-march` 核心规则|g++: -march 核心规则]]。

### 快速选择

| 目标 | 推荐 `-march` |
|------|---------------|
| 最大兼容性 | `armv8-a` |
| 图像/算法 + SIMD | `armv8.2-a`（默认带 SIMD） |
| 极致性能 | `armv8.2-a -mtune=cortex-a76` |

## 1.2 完整优化组合

```bash
-O3 -march=armv8.2-a -mtune=cortex-a76 -ffast-math -fopenmp -fno-rtti -fno-exceptions
```

| 选项 | 作用 |
|------|------|
| `-O3` | 最高级别优化 |
| `-march=armv8.2-a` | RK3588 指令集（默认带 SIMD） |
| `-mtune=cortex-a76` | 流水线调度优化 |
| `-ffast-math` | 浮点加速（图像/算法场景） |
| `-fopenmp` | OpenMP 并行 |
| `-fno-rtti -fno-exceptions` | 减小体积（嵌入式项目推荐） |

## 1.3 验证 SIMD 是否开启

```bash
# 检查生成的机器码是否包含 NEON 指令
aarch64-linux-gnu-objdump -d your_program | grep -E "vadd|vmul|ld1"
```

能搜到向量指令说明 SIMD 已正常开启。
