# 1. 交叉编译优化选项（RK3588）

RK3588（Cortex-A76/A55, AArch64/ARMv8.2-A）交叉编译时 GCC 的 `-march` 与优化选项。

相关笔记：[[1. 讯为RK3588|RK3588 驱动编译]]、[[B-构建与脚本/01.构建工具/1. g++|g++ 编译命令]]

## 1.1 `-march` 核心规则

### 1.1.1 常见错误写法

| 写法 | 结果 |
|------|------|
| `-march=active` | **非法参数**，GCC 无此值，编译报错 |
| `-march=native` | **交叉编译禁用**，识别的是编译主机（x86）而非目标板（RK3588） |
| `armv8.2+` | **不规范**，标准写法必须是 `armv8.2-a` |

### 1.1.2 RK3588 正确写法

```bash
# 基础通用（兼容性最好）
-march=armv8-a

# 开启 NEON/SIMD（图像/算法必加）
-march=armv8-a+simd

# 针对 Cortex-A76/A55 调度优化（不改变指令集）
-march=armv8-a+simd -mtune=cortex-a76

# ARMv8.2-A（RK3588 原生支持，默认带 SIMD）
-march=armv8.2-a -mtune=cortex-a76
```

### 1.1.3 ARMv8 各版本 SIMD 开关

`armv8-a` 及以上版本**默认开启 SIMD（NEON/ASIMD）**，无需手动添加 `+simd`：

| `-march` | SIMD/NEON |
|----------|-----------|
| `armv8-a` | ✅ 默认自带 `+fp +simd` |
| `armv8.1-a` | ✅ 继承，默认保留 |
| `armv8.2-a` | ✅ 继承，默认保留 |

以下写法**效果完全相同**，均开启 SIMD：

```bash
-march=armv8-a
-march=armv8.1-a
-march=armv8.2-a
```

只有显式写 `+nosimd` 才会关闭：
```bash
-march=armv8.2-a+nosimd   # ❌ 禁用 NEON
```

> `armv8.2-a` 新增的是 FP16、点积指令等，**不改变原有 SIMD/NEON 开关**。

## 1.2 完整优化组合

```bash
-O3 -march=armv8.2-a -mtune=cortex-a76 -ffast-math -fopenmp -fno-rtti -fno-exceptions
```

| 选项 | 作用 |
|------|------|
| `-O3` | 最高级别优化 |
| `-march=armv8.2-a` | RK3588 指令集（默认带 SIMD） |
| `-mtune=cortex-a76` | 流水线调度优化 |
| `-ffast-math` | 浮点加速（图像/算法） |
| `-fopenmp` | OpenMP 并行 |
| `-fno-rtti -fno-exceptions` | 减小体积（嵌入式项目推荐） |

## 1.3 验证 SIMD 是否开启

编译后用 `objdump` 检查是否生成 NEON 指令：

```bash
aarch64-linux-gnu-objdump -d your_program | grep -E "vadd|vmul|ld1"
```

能搜到向量指令说明 SIMD 已正常开启。
