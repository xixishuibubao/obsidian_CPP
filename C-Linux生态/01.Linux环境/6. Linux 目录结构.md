# 1. Linux 目录结构

Linux 根目录遵循 FHS，按「配置 / 程序 / 库 / 运行时 / 可变数据」分类定位资源。

## 【核心精简提纲】

### 记忆主干

FHS 把根目录分为四层记忆：**可执行**（`/bin`、`/sbin`、`/usr/bin`）→ **配置**（`/etc`）→ **库与头文件**（`/lib`、`/usr/lib`、`/usr/include`）→ **运行时/可变**（`/proc`、`/sys`、`/dev`、`/run`、`/var`）。用户数据在 `/home`，临时在 `/tmp`，第三方可选在 `/opt`。

### 核心要点清单

- `/etc`：纯文本配置，改前先备份；不含二进制
- `/usr`：主要软件资源（bin/lib/include/share）；现代发行版 usrmerge 合并 `/bin`→`/usr/bin`
- `/lib(64)`：共享库与动态链接器；内核模块在 `/lib/modules/$(uname -r)/`
- `/dev`：设备节点（块/字符/伪设备）；`/dev/null` 丢弃输出
- `/proc`、`/sys`：内核虚拟 FS，读 CPU/内存/进程/设备拓扑
- `/var`：日志（`/var/log`）、缓存、队列；生产常独立分区防撑满根分区

### 关键结论 / 流程

```plain
找命令 → which/whereis (/usr/bin) │ 找配置 → /etc │ 找库 → ldd/readelf + /lib │ 查状态 → /proc /sys │ 查日志 → /var/log
```

## 【完整拓展详情】


Linux 根目录结构遵循 **FHS（Filesystem Hierarchy Standard）**，定义了各个目录的用途和存放内容的类型。以下是完整目录树概览：

```plain
/
├── bin/          # 用户必备命令（二进制）
├── boot/         # 引导加载程序文件
├── dev/          # 设备文件
├── etc/          # 系统配置文件
├── home/         # 普通用户家目录
├── lib/          # 系统库
├── lib64/        # 64 位系统库
├── media/        # 可移动介质挂载点
├── mnt/          # 临时挂载点
├── opt/          # 可选第三方软件
├── proc/         # 进程与内核虚拟文件系统
├── root/         # root 用户家目录
├── run/          # 运行时变量数据
├── sbin/         # 系统管理命令
├── srv/          # 服务数据
├── sys/          # 内核与设备虚拟文件系统
├── tmp/          # 临时文件
├── usr/          # 系统资源（第二层级）
└── var/          # 可变数据
```

# 2. 目录详述

## 2.1 `/bin` — 用户必备命令

**Essential user command binaries**，包含所有用户都可执行的基本命令（单用户模式也需可用）。

```bash
ls /bin/
# 输出示例：bash cat chmod cp date echo grep kill ln ls mkdir mv ps pwd rm rmdir sed sh sleep sort stty su sync tar touch uname 等
```

- 在较新发行版（Fedora 35+、Ubuntu 24.04+）中，`/bin` 通常是 `/usr/bin` 的**符号链接**（即合并到 `/usr/bin`）
- 包含 `bash`、`ls`、`cp`、`mv`、`cat` 等最基本的命令

## 2.2 `/boot` — 引导文件

包含系统启动所需的**内核映像**和**引导加载程序**文件。

```bash
ls /boot/
# 输出示例：vmlinuz-6.8.0-45-generic  initrd.img-6.8.0-45-generic  grub/  config-6.8.0-45-generic  System.map-6.8.0-45-generic
```

| 文件 | 用途 |
|------|------|
| `vmlinuz-*` | 压缩后的 Linux 内核映像 |
| `initrd.img-*` / `initramfs-*` | 初始内存磁盘（加载早期驱动用） |
| `grub/` | GRUB 引导加载程序配置与模块 |
| `config-*` | 内核编译配置 |
| `System.map-*` | 内核符号表 |

> 空间敏感目录——内核升级后旧版本文件不会自动删除，需手动清理或通过包管理器 `autoremove`。

## 2.3 `/dev` — 设备文件

Linux 中**一切皆文件**，硬件设备在 `/dev` 下以**设备文件**（device node）呈现。应用程序通过操作这些文件与硬件交互。

```bash
ls /dev/
# 输出示例：sda  sda1  sda2  sdb  tty0  tty1  null  zero  random  urandom  loop0  mapper/  disk/  ...
```

### 设备类型

| 类型 | 特征 | 示例 |
|------|------|------|
| **块设备（block）** | 支持随机访问、有缓冲区 | `sda`（硬盘）、`sda1`（分区）、`loop0`（回环设备） |
| **字符设备（char）** | 顺序流式访问、无缓冲区 | `tty`（终端）、`null`、`random`、`zero` |
| **伪设备（pseudo）** | 虚拟设备，不实际对应硬件 | `/dev/null`、`/dev/random`、`/dev/zero` |

### 常用虚拟设备

| 设备 | 用途 |
|------|------|
| `/dev/null` | **空设备**——写入全丢弃，读取返回 EOF |
| `/dev/zero` | 读取无限返回 `\0` 字节 |
| `/dev/random` | 从环境噪声生成随机数（阻塞） |
| `/dev/urandom` | 伪随机数生成器（非阻塞，常用） |
| `/dev/tty` | 当前终端 |

查看设备主次编号：

```bash
ls -l /dev/sda
# brw-rw---- 1 root disk 8, 0 5月 21 10:30 /dev/sda
# 8 = 主编号（驱动识别），0 = 次编号（设备实例）
```

### 设备文件管理

现代 Linux 使用 **udev** 在用户空间动态管理 `/dev`：
- 设备插入时自动创建对应文件
- 设备移除时自动删除
- 可编写 udev 规则自定义设备命名

> 旧系统使用 `devfs` 或 `devtmpfs`，现在基本统一为 `udev` + `devtmpfs`。

## 2.4 `/etc` — 系统配置文件

**Editable Text Configuration**，存放系统的纯文本配置文件。**不包含二进制文件**（二进制库在 `/usr/lib`，二进制可执行在 `/usr/bin`）。

```bash
ls /etc/
# 输出示例：passwd  shadow  group  hosts  hostname  resolv.conf  fstab  ssh/  nginx/  cron.d/  systemd/  ...
```

### 核心配置文件

| 文件/目录 | 用途 |
|-----------|------|
| `/etc/passwd` | 用户账号信息 |
| `/etc/shadow` | 用户密码哈希 |
| `/etc/group` | 用户组信息 |
| `/etc/hostname` | 主机名 |
| `/etc/hosts` | 静态主机名 ↔ IP 映射 |
| `/etc/resolv.conf` | DNS 解析器配置 |
| `/etc/fstab` | 文件系统挂载表 |
| `/etc/ssh/sshd_config` | SSH 服务配置 |
| `/etc/cron.d/` | 定时任务配置 |
| `/etc/systemd/` | systemd 服务配置 |
| `/etc/nginx/` / `/etc/apache2/` | 各服务自身配置 |

> **⚠️ 备份优先**：编辑 `/etc/` 下的配置前先备份——一个错误的配置可能导致服务无法启动。养成 `cp file file.bak` 的习惯。

## 2.5 `/home` — 用户家目录

每个普通用户的家目录位于 `/home/<用户名>/`，存放用户的个人文件、配置和应用程序数据。

```plain
/home/
├── alice/         # 用户 alice 的家目录
│   ├── Documents/
│   ├── Downloads/
│   ├── .bashrc    # 用户级 shell 配置
│   ├── .config/   # 用户级应用配置
│   └── .local/    # 用户级数据
└── bob/
    └── ...
```

- **新建用户**时系统自动创建对应家目录（默认从 `/etc/skel/` 复制初始文件）
- root 用户的家目录在 `/root`（独立于 `/home`，即使 `/home` 无法挂载也能登录维护）

## 2.6 `/root` — root 用户家目录

超级用户（root）的家目录。**不**在 `/home/root` 下的原因：

> 如果 `/home` 挂载在独立分区且分区损坏，root 在单用户模式下仍需要一个家目录来完成恢复操作。因此 `/root` 始终位于根文件系统上。

```bash
ls -la /root/
# 输出示例：.bashrc  .ssh/  .bash_history  ...
```

## 2.7 `/lib` 与 `/lib64` — 系统库

存放系统启动和 `/bin`、`/sbin` 中命令所需的**共享库**（shared libraries）。

```bash
ls /lib/x86_64-linux-gnu/ | head -5
# 输出示例：libc-2.31.so  libc.so.6  libm.so.6  libpthread.so.0  libdl.so.2

ls /lib64/
# 通常是一个目录或符号链接，包含 ld-linux-x86-64.so.2（动态链接器）
```

| 目录 | 用途 |
|------|------|
| `/lib` | 基础共享库（32 位或架构无关） |
| `/lib64` | 64 位共享库（x86_64 架构） |
| `/lib/modules/` | 内核模块（`ls /lib/modules/$(uname -r)/`） |
| `/lib/firmware/` | 固件文件 |

- 在合并式发行版（如 Fedora 35+、Ubuntu 24.04+）中，`/lib` 和 `/lib64` 是 `/usr/lib` 和 `/usr/lib64` 的**符号链接**
- 核心 C 库 `libc.so.6` 是 glibc 的加载库，几乎所有动态链接程序都依赖它

## 2.8 `/media` — 可移动介质挂载点

**可移动介质**（removable media）的自动挂载点。系统自动检测并挂载 U 盘、SD 卡、光盘等设备。

```bash
# 插入 U 盘后
/media/
└── alice/
    └── USB_DRIVE/         # 自动挂载到这里
```

- 由桌面环境或 `udisks` 自动管理，不手动操作
- `/mnt` 用于手动挂载，`/media` 用于自动挂载

## 2.9 `/mnt` — 临时挂载点

系统管理员**手动**挂载临时文件系统的位置。

```bash
# 典型用法
sudo mount /dev/sdb1 /mnt              # 临时挂载分区
sudo mount -t nfs server:/share /mnt   # 临时挂载 NFS，持久与同路径见 [[8. NFS 网络文件系统|NFS 笔记]]
```

> 建议只用于临时操作。长期挂载点应在 `/mnt` 下创建子目录，或定义到其他位置（如 `/data`）。

## 2.10 `/opt` — 可选软件包

**Optional add-on software packages**，存放第三方商业软件或非系统包管理器统一管理的软件。

```bash
/opt/
├── google/chrome/         # Google Chrome
├── VBoxGuestAdditions/    # VirtualBox 增强功能
└── jetbrains/             # JetBrains IDE（部分版本）
```

- 每个软件独立一个子目录，便于清理
- 二进制文件通常放在 `/opt/<package>/bin/`，可能需要手动加入 `PATH`
- 约定：遵循 FHS，可选包不应在 `/opt` 外散布文件

## 2.11 `/proc` — 进程虚拟文件系统

**Process information virtual filesystem**，一个**伪文件系统**（pseudo filesystem）——不占磁盘空间，由内核动态生成并映射为文件结构。

```bash
ls /proc/
# 输出示例：1/  2/  cpuinfo  meminfo  mounts  uptime  version  net/  ...
```

| 条目 | 用途 |
|------|------|
| `/proc/1/` | PID 为 1 的进程（init/systemd）信息目录 |
| `/proc/cpuinfo` | CPU 详细信息（型号、核心数、频率等） |
| `/proc/meminfo` | 内存使用详情（总量、可用、缓存等） |
| `/proc/uptime` | 系统运行时间 |
| `/proc/version` | Linux 内核版本 |
| `/proc/loadavg` | 系统负载平均值 |
| `/proc/mounts` | 当前挂载的文件系统 |
| `/proc/net/` | 网络统计信息 |
| `/proc/sys/` | 内核运行时参数（可通过 `sysctl` 调整） |

```bash
# 查看 CPU 信息
cat /proc/cpuinfo | grep "model name" | head -1
# model name : Intel(R) Core(TM) i7-10750H CPU @ 2.60GHz

# 查看内存总量
grep MemTotal /proc/meminfo
# MemTotal:       16289512 kB

# 通过 /proc/sys 动态调整内核参数
echo 1 > /proc/sys/net/ipv4/ip_forward    # 临时启用 IP 转发
# 等价于：sysctl -w net.ipv4.ip_forward=1
```

> **`/proc` 内容随时变化**——每次读取都由内核即时生成。目录中的数字子目录对应运行中的进程 PID。

## 2.12 `/run` — 运行时变量数据

存放**系统启动以来**的运行时数据。在 FHS 3.0 中引入，取代传统的 `/var/run`。

```bash
ls /run/
# 输出示例：lock/  log/  user/  systemd/  mount/  ...
```

- 使用 **tmpfs**（内存文件系统），重启后清空
- 存放 PID 文件、Unix 域套接字、锁文件等
- `/var/run` 通常是 `/run` 的符号链接

## 2.13 `/sbin` — 系统管理命令

**System binaries**，存放系统管理员使用的管理命令（普通用户未必在 `PATH` 中）。

```bash
ls /sbin/
# 输出示例：fdisk  mkfs  mount  fsck  iptables  sysctl  ifconfig  poweroff  reboot  shutdown  ...
```

| 命令 | 用途 |
|------|------|
| `fdisk` / `parted` | 磁盘分区 |
| `mkfs.*` | 文件系统格式化 |
| `mount` / `umount` | 挂载/卸载文件系统 |
| `fsck` | 文件系统检查与修复 |
| `iptables` / `nft` | 防火墙规则管理 |
| `shutdown` / `reboot` / `poweroff` | 系统关机/重启 |
| `sysctl` | 内核参数运行时调整 |

- 现代发行版中 `/sbin` 通常是 `/usr/sbin` 的符号链接
- 普通用户使用某些 `/sbin` 命令时需 `sudo`

## 2.14 `/srv` — 服务数据

**Service data**，存放系统提供的**服务数据**（如 Web 服务器、FTP 服务器的数据文件）。

```bash
/srv/
├── www/              # Web 服务器文件（部分发行版）
├── ftp/              # FTP 共享文件
└── git/              # Git 仓库（部分部署）
```

> 实践中 `/srv` 使用率不高——多数发行版将服务数据放在各自配置目录下（如 `/var/www/`）。FHS 推荐用 `/srv`，但非强制。

## 2.15 `/sys` — 内核与设备虚拟文件系统

**Sysfs**，`/proc` 的补充——将内核的设备模型（device model）以文件形式暴露。

```bash
ls /sys/
# 输出示例：block/  bus/  class/  devices/  firmware/  power/  kernel/  ...
```

| 条目 | 用途 |
|------|------|
| `/sys/class/` | 按设备类别分类（如 `net/` 网络设备、`block/` 块设备） |
| `/sys/devices/` | 设备树，完整硬件拓扑 |
| `/sys/block/` | 块设备信息 |
| `/sys/bus/` | 总线（PCI、USB 等） |
| `/sys/power/` | 电源管理状态 |

```bash
# 查看网络设备状态
cat /sys/class/net/eth0/operstate
# up

# 查看 CPU 信息
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor
# powersave
```

> **对比 `/proc` 与 `/sys`**：
> - `/proc`：面向进程和内核通用信息（`/proc/<pid>/`）
> - `/sys`：面向设备、驱动和硬件拓扑（分类更清晰）

## 2.16 `/tmp` — 临时文件

存放临时文件，所有用户均可读写。

```bash
# 任何人都可在此创建文件
echo "temp data" > /tmp/test.txt
```

- 通常挂载为 tmpfs（内存+交换），或存放在磁盘上
- **重启后清空**（FHS 要求系统启动时清理 `/tmp`）
- 文件可能被系统自动清理（`systemd-tmpfiles`、`tmpwatch`）
- 不适合存放重要数据——**没有持久性保证**

> **`/tmp` vs `~/tmp`（用户自己的 tmp）**：`/tmp` 全局可见且有清理风险。持久性临时文件可放在家目录下的自定义 tmp 目录。

## 2.17 `/usr` — 系统资源（第二层级）

**Unix System Resources**，系统最主要的软件资源存放位置。最初是"用户目录"（user），实际是核心系统文件的次级层级。

```plain
/usr/
├── bin/           # 大多数用户命令（最常用！）
├── lib/           # 共享库
├── lib64/         # 64 位共享库
├── sbin/          # 系统管理命令
├── include/       # C/C++ 头文件
├── share/         # 架构无关的共享数据
├── local/         # 本地安装的软件（类似 /usr 的次级层级）
├── src/           # 内核源码（可选）
└── ...
```

### `/usr/bin`

**最重要**的二进制目录——绝大多用户日常使用的命令都在这里：

```bash
ls /usr/bin/ | wc -l     # 通常数千个程序
```

在该目录的同级目录中有 `python3`、`gcc`、`git`、`vim`、`curl` 等几乎所有用户空间程序。

### `/usr/local`

本地管理员手动编译安装软件的位置（不通过系统包管理器）：

```plain
/usr/local/
├── bin/           # 本地安装的可执行文件
├── lib/           # 本地安装的库
├── include/       # 本地安装的头文件
└── share/         # 本地安装的共享数据
```

- 默认在 `PATH` 末尾，优先级低于 `/usr/bin`
- 手动 `./configure && make && make install` 的默认安装前缀
- 包管理器安装的软件不应污染 `/usr/local`

### `/usr/share`

架构无关（architecture-independent）的共享数据文件：

```bash
/usr/share/
├── man/           # man 手册页
├── doc/           # 文档
├── icons/         # 图标主题
├── fonts/         # 字体文件
├── locale/        # 区域与语言翻译
├── zoneinfo/      # 时区数据
└── applications/  # .desktop 文件
```

### `/usr/include`

C/C++ 标准库头文件：

```bash
ls /usr/include/
# 输出示例：stdio.h  stdlib.h  string.h  unistd.h  linux/  sys/  ...
```

- 系统头文件默认搜索路径
- 内核头文件在 `/usr/include/linux/`、`/usr/include/sys/` 等

### 合并趋势

现代发行版（Fedora 35+、Ubuntu 24.04+、Arch Linux）将 `/{bin,sbin,lib,lib64}` 统一为 `/usr/{bin,sbin,lib,lib64}` 的符号链接：

```plain
/bin    → /usr/bin
/sbin   → /usr/sbin
/lib    → /usr/lib
/lib64  → /usr/lib64
```

这称为 **usrmerge**：一个 `/bin` 目录，所有程序都在 `/usr/bin`，根目录不再分拆 `/{bin,sbin,lib}`。

## 2.18 `/var` — 可变数据

**Variable data**，存放运行时不断变化的数据——日志、缓存、队列、数据库等。

```plain
/var/
├── log/           # 系统日志（syslog、auth.log、kern.log 等）
├── cache/         # 应用程序缓存（apt、pacman、pip 等）
├── lib/           # 状态数据（数据库、包管理器数据库）
├── spool/         # 任务队列（邮件、打印作业）
├── tmp/           # 保留临时文件（重启不清空）
├── opt/           # /opt 软件的可变数据
└── mail/          # 用户邮件（部分发行版）
```

### `/var/log` — 日志文件

最重要的日志目录：

```bash
/var/log/
├── syslog          # 系统日志（Ubuntu/Debian）
├── messages        # 系统日志（RHEL/CentOS）
├── auth.log        # 认证日志（SSH 登录等）
├── kern.log        # 内核日志
├── dmesg           # 内核环缓冲区日志
├── bootstrap.log   # 系统启动日志
├── apt/            # APT 日志
└── nginx/          # Nginx 访问和错误日志（如果安装）
```

查看日志的命令便利：

```bash
tail -f /var/log/syslog        # 实时跟踪新日志
grep "ERROR" /var/log/syslog   # 过滤错误
journalctl -xe                 # systemd 统一日志（大部分发行版现在默认）
```

> **`/var` 独立分区的价值**：将 `/var` 放在独立分区可防止日志爆满填满根分区。这是生产服务器的常见实践。

# 3. 目录 FHS 合规摘要

| 目录 | 存储内容 | 持久性 | 访问权限 |
|------|----------|--------|----------|
| `/bin` | 用户命令 | 持久 | 所有用户 |
| `/boot` | 内核/引导 | 持久 | root 只读 |
| `/dev` | 设备文件 | 运行时（udev） | 按设备 |
| `/etc` | 配置文件 | 持久 | root |
| `/home` | 用户数据 | 持久 | 按用户 |
| `/lib(64)` | 共享库 | 持久 | 所有用户 |
| `/media` | 可移动设备 | 运行时 | 当前用户 |
| `/mnt` | 手动挂载 | 挂载期间 | 管理员 |
| `/opt` | 第三方软件 | 持久 | root |
| `/proc` | 进程/内核信息 | 运行时（虚拟） | 只读 |
| `/root` | root 家目录 | 持久 | root |
| `/run` | 运行时数据 | 运行时（tmpfs） | 按需 |
| `/sbin` | 管理命令 | 持久 | root |
| `/srv` | 服务数据 | 持久 | 按服务 |
| `/sys` | 设备/驱动信息 | 运行时（虚拟） | 只读 |
| `/tmp` | 临时文件 | 重启清空 | 所有用户 |
| `/usr` | 系统资源 | 持久 | 所有用户 |
| `/var` | 可变数据 | 持久 | 按目录 |

# 4. 快速定位命令

```bash
# 查看根目录
ls -la /

# 了解程序安装位置
which ls                  # → /usr/bin/ls
type -a ls                # 所有匹配路径 + 别名
whereis ls                # 二进制 + 源码 + man 页

# 查看文件属于哪个包（Debian/Ubuntu）
dpkg -S /bin/ls           # → coreutils: /bin/ls

# 查看文件属于哪个包（RHEL/Fedora）
rpm -qf /bin/ls           # → coreutils-8.32-34.fc35.x86_64

# 目录空间占用
du -sh /var/log

# 查看挂载点类型
df -Th /tmp               # 显示 tmpfs 等
mount | grep "^/"         # 根文件系统挂载情况
```
