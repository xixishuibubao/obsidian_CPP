# 2. 查找
## 2.1 查找文件
### 2.1.1 `find`
1. `find . -type d -exec chmod 755 {} \;`
	1. `find . -type d`, 在当前目录查找子目录，并获取名字。
	2. `-exec`，执行后续指令。
	3. `{}` 占位符，自动替换前面`find`的结果到`{}`。
	4. `\;`,对分号的转义，避免语法错误。
2. `find mydir -type l ! -exec test -e {} \; -delete`, 删除失效的链接。
3. `find mydir -type l ! -exec test -e {} \; -print`,打印失效的链接。
### 2.1.2 grep
1. 常规用法`ls | grep str`，配合管道`|`来快速查找当前目录的某些文件。
2. `grep -nr str ./`，以当前目录为起点，递归查找所有文件，并标识包含`str`字符的行号和文件。
3. 详细选项与组合用法见 [[5. grep 文本搜索|grep 文本搜索]]。

# 4. SSH 免密登录

## 4.1 背景

每次通过 `ssh user@host` 登录远程服务器都需要输入密码，频繁操作时很麻烦。
`ssh-copy-id` 可以将本机的 SSH **公钥** 追加到远程服务器的 `~/.ssh/authorized_keys` 中，
此后 SSH 登录将自动使用密钥对完成身份验证，不再需要手动输入密码。

## 4.2 前置条件

- 本机已生成 SSH 密钥对（公钥 + 私钥）
  - 检查是否已有：`ls ~/.ssh/`，看是否存在 `id_rsa`（或 `id_ed25519`）和 `id_rsa.pub`（对应 `.pub` 文件）
  - 如果没有，先生成：`ssh-keygen -t ed25519 -C "your_email@example.com"`
    - `-t ed25519`：指定使用 Ed25519 算法（更安全、更高效）；也可用 `-t rsa -b 4096`
    - `-C`：注释标签，通常填邮箱便于识别
- 远程服务器已开启 SSH 服务，且当前知道密码（首次仍需密码来安装公钥）

## 4.3 基本用法

```bash
ssh-copy-id user@remote_host
```

- 首次执行会提示输入远程用户的密码
- 之后 `ssh user@remote_host` 即免密登录

### 指定端口

```bash
ssh-copy-id -p 2222 user@remote_host
```

### 指定使用的公钥文件

```bash
ssh-copy-id -i ~/.ssh/mykey.pub user@remote_host
```

- 不指定 `-i` 时，`ssh-copy-id` 会自动尝试 `~/.ssh/` 下默认的密钥

## 4.4 手动等价操作

`ssh-copy-id` 本质就是做了以下操作：

```bash
cat ~/.ssh/id_ed25519.pub | ssh user@remote_host "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

所以即使目标机器没有 `ssh-copy-id` 命令，也可以用这行管道命令手动完成。

## 4.5 原理简述

```
┌──────────────┐          ┌─────────────────┐
│   本机       │          │   远程服务器     │
│              │   SSH    │                 │
│  id_ed25519  │◄────────►│  authorized_keys │
│  (私钥)      │          │  (存放公钥)      │
│              │          │                 │
│  id_ed25519  │          │  匹配则免密登录  │
│  .pub (公钥) │          │                 │
└──────────────┘          └─────────────────┘
```

- 私钥（`id_ed25519`）**留在本机，绝不外传**
- 公钥（`id_ed25519.pub`）可以安全地复制到任意服务器
- SSH 通过**挑战-响应机制（challenge-response）**验证：服务器用公钥加密一个随机数，客户端用私钥解密并返回，证明持有私钥

## 4.6 常见问题

### 4.6.1 免密登录仍提示输入密码

检查文件和目录权限（SSH 对权限要求严格）：

```bash
# 远程服务器上执行
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
chmod 700 ~          # 家目录也不能对其他用户可写
```

### 4.6.2 多个密钥的管理

编辑 `~/.ssh/config` 为不同主机指定不同密钥：

```
Host myserver
    HostName 192.168.1.100
    User root
    Port 2222
    IdentityFile ~/.ssh/mykey
```

之后直接 `ssh myserver` 即可。

### 4.6.3 `ssh-copy-id` 未安装

macOS / 多数 Linux 发行版自带。若没有：
- **Ubuntu/Debian**: `apt install openssh-client`
- **CentOS/RHEL**: `yum install openssh-clients`

也可直接使用第 4.4 节的手动管道命令。

## 4.7 安全注意事项

- **私钥必须保护好**：设置文件权限 `chmod 600 ~/.ssh/id_ed25519`
- 可以考虑为私钥设置密码（passphrase）：`ssh-keygen -p`
  - 搭配 `ssh-agent` 可避免每次使用都输入 passphrase：`eval "$(ssh-agent -s)" && ssh-add ~/.ssh/id_ed25519`
- 离职或密钥泄露后，及时从远程服务器的 `authorized_keys` 中移除对应公钥
- 如遇 SSH 安全加固（如禁止密码登录），务必在确认密钥登录可用后再关闭密码认证

# 5. Vim 操作参考

## 5.1 操作基础

### 5.1.1 命令模式

- vim 使用 ESC 进入命令模式，且打开文件默认是命令模式
- 在命令模式下按键将会被识别为操作符。

### 5.1.2 短命令

- `i` 在光标之前开始编辑
- `a` 在光标之后开始编辑
- `I` 光标行首开始编辑
- `A` 光标行尾开始编辑
- `o` 在光标下一行创建空行
- `O` 在光标上一行创建空行
- `^` 光标切到行首（第一个非空字符）
- `$` 光标切到行尾
- `v` 进入字符选模式
- `V` 进入行选模式
- `Ctrl+v` 进入列选（块选）模式
- `0` 跳转到行首（这是数字0）
- `w` 跳转到下一个单词
- `b` 跳转到上一个单词
- `hjkl` 分别对应 左移，下移，上移，右移
- `d` 删除当前
- `d + 数字 + d` 删除多少行
- `c` 删除并进入插入模式
- `u` 撤销到上一步操作
- `Ctrl+r` 回退一步撤销
- `数字+` 向后跳转数字行
- `数字-` 向前跳转数字行
- `gg` 文件头
- `G` 文件尾
- `/` 向后搜索
- `?` 向前搜索（搜索模式下按 enter 确定，然后按 n 查找下一个，N 上一个）

### 5.1.3 长命令

- `:w` 保存
- `:q` 退出
- `:q!` 强制退出
- `:wq` 保存并退出
- `:set number` 显示行号
- `:set nonumber` 隐藏行号
- `:noh` 取消搜索高亮（`nohlsearch` 的简写）

## 5.2 文本对象（Text Objects）

VIM 的文本对象是「动词 + 范围 + 对象」的组合操作，是效率的核心。

### 5.2.1 常用文本对象

| 对象 | 含义 |
|------|------|
| `w` | word（单词） |
| `s` | sentence（句子） |
| `p` | paragraph（段落） |
| `(` / `)` | 一对圆括号 ( ) |
| `{` / `}` | 一对花括号 { } |
| `[` / `]` | 一对方括号 [ ] |
| `'` | 一对单引号 ' ' |
| `"` | 一对双引号 " " |
| `<` / `>` | 一对尖括号 < > |
| `t` | XML/HTML tag 标签 |

### 5.2.2 操作语法

`<动词><范围><对象>` — 动词决定做什么，范围决定内外，对象决定选区。

| 范围 | 含义 |
|------|------|
| `i` | inside（内部，不包含定界符） |
| `a` | around / a（包含定界符，含空格） |

常用组合示例：

```vim
ciw        " 删除当前单词并进入插入模式（change inner word）
ci(        " 删除括号内内容并进入插入模式
ca(        " 删除括号及内容并进入插入模式
di"        " 删除双引号内内容
da"        " 删除双引号及内容
yi}        " 复制花括号内内容（yank inner block）
vi(        " 选中括号内内容（visual inner）
ci"        " 修改引号内容（极高频操作）
cit        " 修改 HTML 标签内内容（change inner tag）
```

### 5.2.3 VIM 哲学：复合命令

VIM 的核心是**可组合的操作符**：

```vim
d2w         " 删除两个单词（delete 2 words）
c3j         " 删除当前行及下 3 行并进入插入模式
y2i(        " 复制两层括号内内容
gUiw        " 将当前单词转为大写（global uppercase inner word）
guw         " 将当前单词转为小写
```

操作符、次数、范围、对象的组合几乎可以表达任何编辑意图，无需鼠标。

## 5.4 可视模式进阶

### 5.4.1 三种可视模式

| 命令 | 模式 | 用途 |

| `V` | 行选择模式 | 按整行选中 |

### 5.4.2 可视模式下的常用操作

选中后配合以下命令：

```vim

y           " 复制选中内容
>           " 向右缩进
<           " 向左缩进

u           " 转为小写
```

### 5.4.3 列编辑经典场景

```vim
1. Ctrl+v 进入块选模式

```

常用于：批量注释/取消注释、对齐、批量添加分隔符。

## 5.5 查找与替换

### 5.5.1 基础替换

```vim
:s/old/new            " 替换当前行第一个匹配
:s/old/new/g          " 替换当前行所有匹配
:%s/old/new/g         " 替换全文所有匹配
:%s/old/new/gc        " 替换全文，每次确认（c = confirm）
:5,10s/old/new/g      " 替换 5-10 行范围
:.,+5s/old/new/g      " 替换当前行到后 5 行
```

### 5.5.2 确认替换模式

配合 `c` 标志位（confirm）时可选：

| 按键 | 行为 |
|------|------|
| `y` | 确认替换 |
| `n` | 跳过 |
| `a` | 全部替换（all） |
| `q` | 退出替换 |
| `l` | 替换本次并退出（last） |

### 5.5.3 高级替换

```vim
:%s/\<word\>/newword/g      " 精确匹配整个单词（\< \> 是单词边界）
:%s/old/&/g                 " 引用整个匹配（& 代表整个匹配文本）
:%s/\(old\)/\1_new/g        " 分组引用（\1 引用第一个 \(\) 分组）
:%s/old/newword/gi          " 忽略大小写（i = ignorecase）
:%s/old/newword/gI          " 不忽略大小写
```

> VIM 替换中 `\1`~`\9` 引用正则分组，`&` 或 `\0` 引用整个匹配。

### 5.5.4 跨文件查找

```vim
:vimgrep /pattern/ **/*.c   " 在整个项目 .c 文件中搜索 pattern
:copen                       " 打开 quickfix 列表查看所有匹配
:cnext / :cprev             " 跳转上/下一个匹配
```

也可在 VIM 内使用 `:!grep ...` 调用外部 grep。

## 5.6 多文件编辑

### 5.6.1 缓冲区（Buffer）

VIM 打开的文件会载入缓冲区，可同时操作多个文件而不关闭当前文件。

```vim
:e file.txt          " 打开另一个文件到当前缓冲区（edit）
:ls                  " 列出所有已载入的缓冲区
:b n                 " 跳转到编号为 n 的缓冲区
:bn                  " 下一个缓冲区（buffer next）

:bd                  " 关闭当前缓冲区（buffer delete），不退出 VIM
:bw                  " 彻底擦除缓冲区（buffer wipeout）
:bufdo %s/old/new/ge " 对所有缓冲区执行替换（e 标志忽略无此模式的错误）
```

### 5.6.2 窗口分屏

```vim
:split file.txt      " 水平分屏打开文件（缩写 :sp）
:vsplit file.txt     " 垂直分屏打开文件（缩写 :vsp）
:new                 " 水平分屏打开空缓冲区
:vnew                " 垂直分屏打开空缓冲区
Ctr+w s              " 水平分割当前窗口（等于 :split）
Ctr+w v              " 垂直分割当前窗口（等于 :vsplit）
```

窗口间导航：

```vim
Ctr+w h              " 焦点移到左侧窗口
Ctr+w j              " 焦点移到下方窗口
Ctr+w k              " 焦点移到上方窗口
Ctr+w l              " 焦点移到右侧窗口
Ctr+w w              " 在窗口间循环切换
Ctr+w =              " 所有窗口等宽等高
Ctr+w _              " 当前窗口最大化高度
Ctr+w |              " 当前窗口最大化宽度
Ctr+w o              " 关闭除当前窗口外的所有窗口（只保留一个）
```

### 5.6.3 标签页（Tab）

```vim
:tabnew file.txt     " 新建标签页打开文件
:tabedit file.txt    " 同上（缩写 :tabe）
:tabclose            " 关闭当前标签页
:tabnext / :tabn     " 下一个标签页
```

## 5.7 寄存器（Registers）

VIM 的寄存器是**多个剪贴板**，可独立存取文本。

### 5.7.1 常用寄存器

| 寄存器 | 名称 | 用途 |
|--------|------|------|
| `""` | 无名寄存器 | 默认寄存器，`d`/`y`/`p` 默认操作这个 |
| `"0` | 复制寄存器 | 只保存最近一次 `y` 操作，不受 `d` 覆盖 |
| `"1` ~ `"9` | 历史寄存器 | `"1` 是最近一次删除，`"2` 是上上次。。。 |
| `"a` ~ `"z` | 命名寄存器 | 用户自定义，可手动指定写入和读取 |
| `"+` | 系统剪贴板 | 与系统剪贴板互通（跨应用粘贴） |
| `"*` | X11 选择剪贴板 | Linux 下鼠标中键粘贴 |
| `"_` | 黑洞寄存器 | 删除内容不存入任何寄存器（等于"扔掉"） |
| `"=` | 表达式寄存器 | 执行简单计算并插入结果 |

### 5.7.2 基本用法

```vim
"ayy             " 复制当前行到寄存器 a
"ap              " 粘贴寄存器 a 的内容
"+yy             " 复制当前行到系统剪贴板
"+p              " 从系统剪贴板粘贴
"_dd             " 删除当前行且不存入任何寄存器
"=3+5<CR>p       " 计算 3+5 并将结果粘贴进来
```

### 5.7.3 系统剪贴板

需 VIM 支持 `+clipboard` 特性，查看方式：

```vim
:version          " 查看 clipboard 前是否有 + 号
```

若不支持，安装 `vim-gtk` 或 `gvim` 版获取系统剪贴板支持。

## 5.8 宏（Macros）

宏可以录制一系列按键操作并重放，用于对重复模式执行相同操作。

### 5.8.1 录制与回放

```vim
qa               " 开始录制宏到寄存器 a（q + 寄存器名）
（执行一系列操作）
q                " 停止录制
@a               " 回放寄存器 a 中的宏
@@               " 回放最近一次执行的宏
5@a              " 连续执行宏 a 5 次
```

### 5.8.2 经典用例：批量加引号

对每行末尾的数值加括号：

```vim
qa               " 开始录制
A                " 到行尾进入插入模式
, 100)           " 添加内容
Esc              " 回到命令模式
j                " 下一行
q                " 停止录制
10@a             " 对后续 10 行执行宏
```

### 5.8.3 宏常见技巧

- 宏中尽量使用**相对移动**（`w`, `f{char}`, `t{char}`）+ 文本对象，避免绝对位置
- 宏执行到文件尾会报错终止（正常现象）
- 宏写入命名寄存器后可被多个文件共享（换个缓冲区 `@a` 仍然可用）
- 用 `:reg a` 查看寄存器 `a` 中记录的宏内容

## 5.9 标记（Marks）

标记（Marks）可以在文件中打点，快速跳转。单个 VIM 会话可随时创建和跳转标记。

### 5.9.1 基本用法

```vim
ma               " 在光标位置创建标记 a（小写字母，仅当前文件）
mA               " 创建全局标记 A（大写字母，跨文件）
`a               " 跳转到标记 a 的精确位置（行 + 列）
'a               " 跳转到标记 a 的行首
``               " 跳回上一个位置（相当于反引号反引号）
''               " 跳回上一行的行首
```

### 5.9.2 特殊标记

| 标记 | 含义 |
|------|------|
| `` `. `` | 上次编辑的位置 |
| `"` | 上次退出文件时的位置 |
| `[` | 上次修改/粘贴的起始位置 |
| `]` | 上次修改/粘贴的结束位置 |
| `<` | 上次可视模式选区的起始 |
| `>` | 上次可视模式选区的结束 |

### 5.9.3 管理标记

```vim
:marks           " 列出所有标记
:marks a         " 查看标记 a 的具体位置
:delmarks a      " 删除标记 a
:delmarks!       " 删除所有小写标记
```

## 5.10 重复与快捷操作

| 命令 | 作用 |
|------|------|
| `.` | **重复上一次修改**（VIM 中最强大的键之一！） |
| `;` | 重复上一次 `f`/`t`/`F`/`T` 查找 |
| `,` | 反向重复上一次 `f`/`t`/`F`/`T` 查找 |
| `~` | 切换光标处字符大小写 |
| `J` | 将下一行拼接到当前行尾 |
| `gJ` | 拼接下一行但不插入空格 |
| `>>` | 向右缩进（增加缩进） |
| `<<` | 向左缩进（减少缩进） |
| `==` | 自动缩进当前行 |
| `Ctrl+a` | 光标处的数字 +1 |
| `Ctrl+x` | 光标处的数字 -1 |
| `ga` | 显示光标处字符的 ASCII / Unicode 值 |
| `gf` | 打开光标处的文件路径（go to file） |

### Dot 命令（`.`）的精髓

`.` 会重复「上一次完整修改」，包括进入插入模式 → 输入内容 → 回到命令模式的整个过程。

```vim
示例：将三行末尾加上分号

原始：
int a
int b
int c

操作：
1. 光标停在第一行，按 A 到行尾，输入 ;，按 ESC
2. 光标移到第二行开头，按 ...VIM 自动重复 A;ESC
3. 第三行同样按 .  即可
```

## 5.11 折叠（Folding）

### 5.11.1 常用折叠命令

```vim
zc               " 折叠（close fold）
zo               " 展开折叠（open fold）
zM               " 关闭所有折叠
zR               " 展开所有折叠
zf{motion}       " 创建折叠（fold） — 如 `zfip` 折叠段落
zd               " 删除当前折叠
zj               " 跳到下一个折叠起始
zk               " 跳到上一个折叠起始
```

### 5.11.2 选择折叠方式

```vim
:set foldmethod=indent    " 按缩进折叠（适合代码）
:set foldmethod=syntax    " 按语法折叠
:set foldmethod=marker    " 按标记折叠（默认 `{{{`, `}}}`）
:set foldmethod=manual    " 手动创建折叠（`zf`）
```

## 5.12 实用配置（vimrc）

常用 `.vimrc` 配置项，放入 `~/.vimrc`（Linux/macOS）或 `~/vimfiles/vimrc`（Windows）生效。

```vim
" 基础
set nocompatible              " 不与 vi 兼容
syntax on                     " 语法高亮
filetype plugin indent on     " 文件类型检测
set number                    " 显示行号
set relativenumber            " 相对行号（配合 number 更高效）
set cursorline                " 高亮当前行

" 缩进
set tabstop=4                 " Tab 宽度 4
set shiftwidth=4              " 缩进宽度 4
set softtabstop=4             " 按退格键时一次退 4 格
set expandtab                 " Tab 转为空格
set autoindent                " 自动缩进
set smartindent               " 智能缩进（C 语言等）

" 搜索
set hlsearch                  " 搜索高亮
set incsearch                 " 增量搜索（输入同时匹配）
set ignorecase                " 搜索忽略大小写
set smartcase                 " 智能大小写（搜索含大写时变为区分大小写）

" 编辑体验
set showmode                  " 显示当前模式
set showcmd                   " 显示正在输入的命令
set wildmenu                  " 命令行补全菜单
set nowrap                    " 不自动换行
set scrolloff=5               " 上下滚动时保持 5 行上下文
set sidescrolloff=5           " 左右滚动时保持 5 列上下文

" 备份（可选）
set nobackup                  " 不生成 ~ 备份文件
set noswapfile                " 不生成 .swp 交换文件（仅用于个人编辑）
" set undofile                " 保留持久化撤销历史（想开启时取消注释）
" set undodir=~/.vim/undodir  " 撤销文件存放目录

" 键位映射示例
" 设置 <leader> 键为空格
let mapleader = " "
" 按 <leader>w 快速保存
nnoremap <leader>w :w<CR>
" 按 <leader>q 快速退出
nnoremap <leader>q :q<CR>
```

## 5.13 VIM 模式速查

| 模式 | 进入方式 | 作用 |
|------|---------|------|
| **Normal**（命令模式） | `Esc` / `Ctrl+[` | 默认模式，按键被解释为操作符 |
| **Insert**（插入模式） | `i` / `a` / `o` 等 | 输入文本 |
| **Visual**（可视模式） | `v` / `V` / `Ctrl+v` | 选择文本块 |
| **Command-line**（命令行模式） | `:` / `/` / `?` | 执行 Ex 命令 / 搜索 |
| **Replace**（替换模式） | `R` | 逐个替换字符 |

> `Ctrl+[` 等同于 `Esc`，小指不用移到左上角，触感更舒适。
