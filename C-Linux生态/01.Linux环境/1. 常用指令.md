# 2. 查找
## 2.1 查找文件
### 2.1.1 `find`
1. `find . -type d -exec chmod 755 {} \;`
	1. `find . -type d`, 在当前目录查找子目录，并获取名字。
	2. `-exec`，执行后续指令。
	3. `{}` 占位符，自动替换前面`find`的结果到`{}`。
	4. `\;`,对分号的转义，避免语法错误。
2. `find mydir -type l ! -exec test -e {} \; -delete`, 删除失效的链接。
3. `find mydir -type l ! -exec test -e {} \; -print`,打印失效的链接。
### 2.1.2 `xargs`

#### 2.1.2.1 生成式 vs 逐次式

`xargs` 是一个参数生成器（arguments builder），它从标准输入（`stdin`）读取数据，拼成**一条命令**执行：

```bash
find . -name "*.log" | xargs rm
```

与 `find -exec` 的区别：

| 特性 | `-exec` | `xargs` |
|------|---------|---------|
| 执行方式 | 每个结果执行一次（`\;`）或一批一次（`+`） | 默认**批量拼接到一条命令** |
| 数据来源 | `find` 输出 | 任意管道传来的 stdin |
| 效率 | 逐条调用时慢，但安全 | 一次调用多个参数，效率高 |
| 特殊字符 | `find` 自带处理 | 默认空格分隔，需 `-print0` + `-0` |
| 独立性 | 不依赖外部命令 | 依赖管道 |

#### 2.1.2.2 常用选项

```bash
# 限制每批参数个数（避免命令行过长）
find . -type f | xargs -n 10 rm    # 每 10 个文件调一次 rm

# 处理文件名含空格（必须与 find -print0 配对！）
find . -name "*.log" -print0 | xargs -0 rm

# 占位符指定位置（-I：替换标记）
find . -name "*.txt" | xargs -I {} cp {} /backup/

# 并行执行（-P：一次跑 N 个进程）
find . -name "*.gz" | xargs -P 4 -n 1 gunzip
```

> **安全原则**：文件名可能含空格时，永远用 `find ... -print0 \| xargs -0` 配对，否则空格会被当作分隔符。

#### 2.1.2.3 `-exec` 补充：加号模式

`find -exec` 末尾用 `+` 而非 `\;` 也能**批量**执行：

```bash
find . -type f -name "*.c" -exec grep "main" {} +   # 所有结果拼成一条命令
```

行为类似 `xargs`（一次传多个文件），效率远高于逐条 `\;`，且**没有空格分裂问题**。

#### 2.1.2.4 场景对比

```bash
# 场景一：简单批量删除 → 优先 -exec
find . -name "*.tmp" -delete                        # 👍 find 内置，最简
find . -name "*.tmp" -exec rm {} +                  # 👍 安全，无空格问题

# 场景二：后续还要挂管道 → 用 xargs
find . -name "*.log" -print0 | xargs -0 tar czf logs.tar.gz

# 场景三：需要并发加速 → xargs -P 完胜
find . -name "*.jpg" -print0 | xargs -0 -P 8 -n 1 convert -resize 50%

# 场景四：少量文件、逐条确认 → -exec \; 直观
find . -type d -empty -exec rmdir {} \;
```

**选型建议**：
- **有空格风险 + 无需管道后续** → `-exec {} +`
- **需要并行/管道后续/处理非 find 来源** → `xargs`
- **需要逐条确认/精确控制每次调用** → `-exec {} \;`
- **文件含空格时 xargs 必须配 `-print0` + `-0`**

### 2.1.3 grep
1. 常规用法`ls | grep str`，配合管道`|`来快速查找当前目录的某些文件。
2. `grep -nr str ./`，以当前目录为起点，递归查找所有文件，并标识包含`str`字符的行号和文件。
3. 详细选项与组合用法见 [[5. grep 文本搜索|grep 文本搜索]]。

# 3. 压缩与归档
## 3.1 tar

### 3.1.1 简介

tar 是一款打包和压缩解压缩软件，默认的压缩算法是 gz。

### 3.1.2 命令字

1. `tar -控制参数(c,r,x,z,v,f) 包名 包内容1 包内容2`
2. 一般使用 `czvf`、`xzvf name`，name 是包内容也可以是一个目录。
   - `c` 创建
   - `r` 追加
   - `x` 解压缩
   - `z` 使用压缩成 .gz 文件
   - `v` 详细信息
   - `f` 指定包名称
   - `tf` 不解压列出包内容（`t` 是 list）

# 4. SSH 免密登录

## 4.1 背景

每次通过 `ssh user@host` 登录远程服务器都需要输入密码，频繁操作时很麻烦。
`ssh-copy-id` 可以将本机的 SSH **公钥** 追加到远程服务器的 `~/.ssh/authorized_keys` 中，
此后 SSH 登录将自动使用密钥对完成身份验证，不再需要手动输入密码。

## 4.2 前置条件

- 本机已生成 SSH 密钥对（公钥 + 私钥）
  - 检查是否已有：`ls ~/.ssh/`，看是否存在 `id_rsa`（或 `id_ed25519`）和 `id_rsa.pub`（对应 `.pub` 文件）
  - 如果没有，先生成：`ssh-keygen -t ed25519 -C "your_email@example.com"`
    - `-t ed25519`：指定使用 Ed25519 算法（更安全、更高效）；也可用 `-t rsa -b 4096`
    - `-C`：注释标签，通常填邮箱便于识别
- 远程服务器已开启 SSH 服务，且当前知道密码（首次仍需密码来安装公钥）

## 4.3 基本用法

```bash
ssh-copy-id user@remote_host
```

- 首次执行会提示输入远程用户的密码
- 之后 `ssh user@remote_host` 即免密登录

### 指定端口

```bash
ssh-copy-id -p 2222 user@remote_host
```

### 指定使用的公钥文件

```bash
ssh-copy-id -i ~/.ssh/mykey.pub user@remote_host
```

- 不指定 `-i` 时，`ssh-copy-id` 会自动尝试 `~/.ssh/` 下默认的密钥

## 4.4 手动等价操作

`ssh-copy-id` 本质就是做了以下操作：

```bash
cat ~/.ssh/id_ed25519.pub | ssh user@remote_host "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

所以即使目标机器没有 `ssh-copy-id` 命令，也可以用这行管道命令手动完成。

## 4.5 原理简述

```
┌──────────────┐          ┌─────────────────┐
│   本机       │          │   远程服务器     │
│              │   SSH    │                 │
│  id_ed25519  │◄────────►│  authorized_keys │
│  (私钥)      │          │  (存放公钥)      │
│              │          │                 │
│  id_ed25519  │          │  匹配则免密登录  │
│  .pub (公钥) │          │                 │
└──────────────┘          └─────────────────┘
```

- 私钥（`id_ed25519`）**留在本机，绝不外传**
- 公钥（`id_ed25519.pub`）可以安全地复制到任意服务器
- SSH 通过**挑战-响应机制（challenge-response）**验证：服务器用公钥加密一个随机数，客户端用私钥解密并返回，证明持有私钥

## 4.6 常见问题

### 4.6.1 免密登录仍提示输入密码

检查文件和目录权限（SSH 对权限要求严格）：

```bash
# 远程服务器上执行
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
chmod 700 ~          # 家目录也不能对其他用户可写
```

### 4.6.2 多个密钥的管理

编辑 `~/.ssh/config` 为不同主机指定不同密钥：

```
Host myserver
    HostName 192.168.1.100
    User root
    Port 2222
    IdentityFile ~/.ssh/mykey
```

之后直接 `ssh myserver` 即可。

### 4.6.3 `ssh-copy-id` 未安装

macOS / 多数 Linux 发行版自带。若没有：
- **Ubuntu/Debian**: `apt install openssh-client`
- **CentOS/RHEL**: `yum install openssh-clients`

也可直接使用第 4.4 节的手动管道命令。

## 4.7 安全注意事项

- **私钥必须保护好**：设置文件权限 `chmod 600 ~/.ssh/id_ed25519`
- 可以考虑为私钥设置密码（passphrase）：`ssh-keygen -p`
  - 搭配 `ssh-agent` 可避免每次使用都输入 passphrase：`eval "$(ssh-agent -s)" && ssh-add ~/.ssh/id_ed25519`
- 离职或密钥泄露后，及时从远程服务器的 `authorized_keys` 中移除对应公钥
- 如遇 SSH 安全加固（如禁止密码登录），务必在确认密钥登录可用后再关闭密码认证

# 5. Vim 操作参考

## 5.1 操作基础

### 5.1.1 模式概览

| 模式 | 进入 | 作用 |
|------|------|------|
| **Normal**（命令模式） | `Esc` / `Ctrl+[` | 默认模式，按键为操作符 |
| **Insert**（插入模式） | `i` / `a` / `o` 等 | 输入文本 |
| **Visual**（可视模式） | `v` / `V` / `Ctrl+v` | 选择文本块 |
| **Command-line**（命令行模式） | `:` / `/` / `?` | 执行 Ex 命令 / 搜索 |

> `Ctrl+[` 等同于 `Esc`，小指不用移到左上角。

### 5.1.2 常用命令

**移动**
- `h`/`j`/`k`/`l` — 左/下/上/右
- `w`/`b` — 下/上一个单词；`0`/`^`/`$` — 行首/首非空/行尾
- `gg`/`G` — 文件头/文件尾；`{`/`}` — 段首/段尾
- `%` — 括号匹配跳转；`f{char}` — 行内正向查找字符
- `Ctrl+d`/`Ctrl+u` — 向下/上翻半页

**编辑**
- `i`/`a` — 光标前/后插入；`I`/`A` — 行首/行尾插入
- `o`/`O` — 下/上一行新建空行并插入
- `x` — 删除光标处字符；`dd` — 删除当前行
- `yy` — 复制当前行；`p`/`P` — 后/前粘贴
- `u` — 撤销；`Ctrl+r` — 重做；`.` — 重复上一次修改
- `>>`/`<<` — 向右/左缩进

**搜索**
- `/` — 向后搜索；`?` — 向前搜索
- `n`/`N` — 下/上一个匹配；`:noh` — 取消搜索高亮

**文件操作**
- `:w` — 保存；`:q` — 退出；`:q!` — 强制退出；`:wq` — 保存并退出
- `:set number` / `:set nonumber` — 显示/隐藏行号

## 5.2 文本对象（Text Objects）

语法：`<动词><范围><对象>`，动词 = 操作，范围 = `i`(内部) / `a`(含定界符)，对象 = 选区。

| 对象 | 含义 |
|------|------|
| `w` | 单词 |
| `p` | 段落 |
| `(` / `{` / `[` / `"` / `'` | 成对标点/括号内 |
| `t` | HTML/XML tag 内 |

高频组合：

```vim
ciw    " 改当前单词（change inner word）
ci(    " 改括号内内容
ci"    " 改引号内内容
cit    " 改 tag 内内容
di(    " 删除括号内内容
yi}    " 复制花括号内内容
vi(    " 选中括号内内容
```

## 5.3 剪切、复制与粘贴

| 命令 | 作用 |
|------|------|
| `dd` | 删除（剪切）当前行 |
| `D` | 删除光标到行尾 |
| `dw` | 删除光标到单词尾 |
| `x` | 删除光标处字符 |
| ─ | ─ |
| `yy` | 复制当前行 |
| `yw` / `y$` | 复制到单词尾 / 行尾 |
| ─ | ─ |
| `p` / `P` | 后 / 前粘贴 |
| `"+y` / `"+p` | **复制/粘贴到系统剪贴板**（需 VIM 支持 `+clipboard`） |

> `d` 删除的内容自动保存，可直接 `p` 粘贴，等效于剪切。`c` 也是删除，区别是删除后进入插入模式。

## 5.4 可视模式

| 模式 | 用途 |
|------|------|
| `v` | 字符选择（逐字） |
| `V` | 行选择（逐行） |
| `Ctrl+v` | 块选择（矩形块，列编辑） |

选中后：`d`删除 / `y`复制 / `>`缩进 / `<`反缩进 / `J`合并 / `~`大小写切换。

**列编辑经典**：`Ctrl+v` 选中多行 → `I`（行首插入）或 `A`（行尾插入）→ 输入内容 → `Esc` 批量生效。常用于批量加注释/分隔符。

## 5.5 查找与替换

```vim
:s/old/new           " 替换当前行第一个
:%s/old/new/g        " 替换全文所有
:%s/old/new/gc       " 替换全文，逐一确认（y/n/a/q）
:5,10s/old/new/g     " 替换 5-10 行
:.,+5s/old/new/g     " 替换当前行到后 5 行
```

`:%s/old/new/g` 是最常用的替换模式，其中：
- `%` = 全文范围，`g` = 行内全部匹配
- `c` = confirm（确认模式：y确认/n跳过/a全部/q退出）

## 5.6 多文件编辑

### 缓冲区

```vim
:e file.txt     " 打开文件到当前缓冲区
:ls             " 列出所有已载入的缓冲区
:bn / :bp       " 下/上一个缓冲区
:bd             " 关闭当前缓冲区（不退出 VIM）
```

### 窗口分屏

```vim
:sp file.txt     " 水平分屏
:vsp file.txt    " 垂直分屏
Ctrl+w h/j/k/l  " 焦点移到左/下/上/右窗口
Ctrl+w =        " 所有窗口等分
Ctrl+w o        " 关闭其他窗口，只剩当前
```

## 5.7 实用配置（vimrc）

放入 `~/.vimrc`（Linux/macOS）或 `~/vimfiles/vimrc`（Windows）生效：

```vim
set nocompatible              " 不与 vi 兼容
syntax on                     " 语法高亮
filetype plugin indent on     " 文件类型检测

set number                    " 行号
set tabstop=4                 " Tab 宽度 4
set shiftwidth=4              " 缩进宽度 4
set expandtab                 " Tab 转空格
set autoindent                " 自动缩进

set hlsearch                  " 搜索高亮
set incsearch                 " 增量搜索
set ignorecase                " 忽略大小写
set smartcase                 " 含大写时区分大小写

set scrolloff=5               " 保持 5 行上下文

" 键位映射：<leader> 设为空格
let mapleader = " "
nnoremap <leader>w :w<CR>     " <空格>w 快速保存
nnoremap <leader>q :q<CR>     " <空格>q 快速退出
```
