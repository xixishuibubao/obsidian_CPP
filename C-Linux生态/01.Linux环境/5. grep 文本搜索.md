# 1. grep 文本搜索

## 【核心精简提纲】

### 记忆主干

`grep` 在文本流或目录中按模式匹配行。选型：**基础**（`-i/-v/-c/-n`）→ **上下文**（`-A/-B/-C` 查日志）→ **范围**（`-r/-R` 递归）→ **精度**（`-w/-F/-o` 整词/字面/提取）→ **文件级**（`-l/-L` 只列文件名）。

### 核心要点清单

- 日志排错：`-n -C 5 "error" app.log`（行号 + 上下文）
- 过滤注释：`-iv "^#"` 反向匹配
- 代码搜索：`-rw "symbol" ./src/` 递归整词
- 统计频次：`-ci "fail"` 忽略大小写计数
- 提取字段：`-o` 只输出匹配片段（IP、端口等）
- 找含关键词的文件：`-rl "pattern" ./`

### 关键结论 / 流程

```
定范围(文件/管道/-r) → 定精度(-w/-F) → 定输出(-n/-C/-c/-l/-o)
```

## 1.1 核心选项

### 1.1.1 基础匹配

| 选项 | 用途 | 场景 |
|------|------|------|
| `-i` | 忽略大小写 | 日志、英文配置、不确定大小写的关键词搜索 |
| `-v` | 反向匹配（排除含目标行） | 过滤无效行、剔除注释/空行 |
| `-c` | 只输出匹配行数，不显示内容 | 统计关键词出现频次 |
| `-n` | 显示行号 | 定位代码/配置文件出错位置 |

### 1.1.2 上下文查看（查日志最常用）

| 选项 | 用途 | 说明 |
|------|------|------|
| `-A n` | 匹配行 + 后 n 行 | After |
| `-B n` | 匹配行 + 前 n 行 | Before |
| `-C n` | 匹配行 + 前后各 n 行 | Context |

场景：排查报错日志、查看完整调用上下文。

### 1.1.3 递归与文件检索

| 选项 | 用途 |
|------|------|
| `-r` | 递归搜索目录下所有文件（不跟进软链接） |
| `-R` | 递归搜索（跟进软链接） |

### 1.1.4 正则与精确匹配

| 选项 | 用途 | 示例 |
|------|------|------|
| `-w` | 整词匹配（只匹配独立单词） | `test` 不会匹配 `test123` |
| `-o` | 只输出匹配到的字符串本身 | 提取日志中的 IP、端口 |
| `-F` | 当作纯文本（关闭正则） | 搜索含 `. * ?` 等特殊符号的内容 |

### 1.1.5 文件名显示

| 选项 | 用途 |
|------|------|
| `-l` | 只输出包含匹配内容的文件名 |
| `-L` | 只输出不包含匹配内容的文件名 |

### 1.1.6 颜色高亮

```
--color=auto
```

多数系统默认别名已开启高亮。

## 1.2 高频组合用法

### 1.2.1 日志排错

```bash
grep -n -C 5 "error" app.log
```

查 error，显示行号 + 前后 5 行。

### 1.2.2 忽略大小写 + 过滤注释

```bash
grep -iv "^#" nginx.conf
```

忽略大小写，排除 `#` 开头的注释行。

### 1.2.3 项目全局搜代码

```bash
grep -rw "user_id" ./src/
```

递归、整词匹配，在 src 目录搜 `user_id`。

### 1.2.4 统计错误次数

```bash
grep -ci "fail" service.log
```

忽略大小写，统计失败行数。

### 1.2.5 提取所有 IP 地址

```bash
grep -o "[0-9]\+\.[0-9]\+\.[0-9]\+\.[0-9]\+" access.log
```

### 1.2.6 找出含目标内容的所有文件

```bash
grep -rl "timeout" ./
```

## 1.5 常见踩坑

- **二进制文件**：默认 grep 可能扫到 `/dev/`、`*.so` 产生乱码 → 加 `-I` 跳过二进制，或 `--include='*.c'`
- **递归软链死循环**：`-r` 不跟进软链，`-R` 会 → 目录有循环链接时用 `-r`
- **正则特殊字符**：搜索 `.`、`*` 等字面量 → 用 `-F` 或转义
- **大文件性能**：先 `grep -l` 定位文件再细查，避免 `grep -r` 无过滤扫全盘
