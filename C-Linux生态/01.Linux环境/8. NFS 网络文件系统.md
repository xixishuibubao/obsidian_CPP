# 1. NFS 网络文件系统

Ubuntu 双机（如 9950X 台式 + 12900H 笔记本）通过 **NFS** 共享 **工具链、sysroot、SDK**，使各节点 **绝对路径一致**，满足 [[B-构建与脚本/01.构建工具/6. 分布式编译|分布式编译]]（distcc/icecc）远程编码头文件与交叉 gcc 路径的要求。临时挂载语法见 [[C-Linux生态/01.Linux环境/6. Linux 目录结构|Linux 目录结构]] `/mnt`。

## 【核心精简提纲】

### 记忆主干

**NFS = Server export + Client 同路径挂载**。分布式编译场景：**源码可留主编译机本地**；**工具链 + sysroot 必须 NFS 同路径**（如 `/srv/distcc`）。Server 推荐常开台式机（9950X）；build 产物 `.o` **不要**放 NFS。

### 核心要点清单

- Server：`nfs-kernel-server`，`/etc/exports` 仅内网网段；开发机可用 `no_root_squash`（仅限可信 LAN）
- Client：`nfs-common`，`fstab` 用 `nfs4`、`defaults,_netdev,nofail`，**挂载点各节点完全相同**；**开机自动挂载**后新机器仅需加一行 fstab 即可加入集群
- 布局：`/srv/distcc/toolchain`（交叉 gcc/SDK）、`/srv/distcc/sysroot`（`--sysroot`）
- 编译调优：exports 默认 `sync` 保一致；NFS 承载只读工具链，**build 目录在本地磁盘**
- 验证：helper 上 `ls /srv/distcc/sysroot/usr/include`；distcc 报缺头文件时先查 NFS 而非 distcc
- 排错：`exportfs -v`、`showmount -e`、`mount | grep nfs`、防火墙 2049

### 关键结论 / 流程

```plain
9950X 建 /srv/distcc → exports + enable nfs-server → 新节点 fstab 一行 → 重启自动挂载 → checklist → distccd 加入集群
```

## 【完整拓展详情】

## 1.1 适用场景与边界

### 为何分布式编译需要 NFS

distcc/icecc 把 **单源文件 compile** 发到远程 helper 执行。远程进程使用的 **编译器路径、`-I` 头文件、`--sysroot`** 必须与主编译机一致。若各节点本地安装路径不同（如 SDK 在 A 机 `/home/foo/sdk`、B 机 `/opt/sdk`），远程编译会 **找不到头文件** 或链到错误库。

NFS 解决：**一份目录树，多机同一挂载点**。

### 什么放 NFS、什么不放

| 内容 | 是否 NFS | 说明 |
|------|----------|------|
| 交叉工具链 / Rockchip SDK | **是** | 见 [[C-Linux生态/03.Linux开发/2. 交叉编译优化选项]] 中 `TOOLCHAIN=` |
| sysroot（`--sysroot`） | **是** | 目标架构头文件与 `.so` |
| 大型一致 `-dev` 包（可选） | 可选 | 减少各节点 `apt` 版本漂移 |
| 工程源码 | **默认否** | distcc 传源码或 pump；源码放主编译机本地即可 |
| build 目录 / `.o` / `.a` | **否** | NFS 上编译慢、易锁竞争；产物放本地 SSD |

### 拓扑建议（9950X + 12900H）

| 角色 | 机器 | 说明 |
|------|------|------|
| NFS Server | **9950X 台式**（常开、有线） | 存放 `/srv/distcc` 实体目录 |
| NFS Client | 9950X + 12900H | 均 `mount` 到 **相同** `/srv/distcc` |
| distccd / iceccd | 两台均可 | 与 NFS 角色正交 |

笔记本外出时可仅本地编译；回 LAN 后 `sudo mount -a` 恢复 NFS（若 fstab 已配 `nofail`，开机不会因 Server 不可达而卡死）。

**NFS Server 本机（9950X）**：`/srv/distcc` 是 **本地目录**，无需通过 NFS 挂载自身；Client 才需要 fstab。

## 1.2 目录布局约定（与 distcc 集群绑定）

在 **Server 本机** 创建并填入内容（示例路径，可按习惯改用 `/opt/cross`，但 **所有节点必须一致**）：

```bash
sudo mkdir -p /srv/distcc/{toolchain,sysroot,devel}
# toolchain：解压 SDK 或 symlink 到 aarch64-linux-gnu-gcc
# sysroot：交叉根文件系统，供 --sysroot=/srv/distcc/sysroot
# devel（可选）：统一存放额外头文件
```

Rockchip 示例（NFS 共享交叉工具链）：

```bash
# 将 SDK 工具链目录复制或绑定到 NFS 可见路径（按实际 SDK 路径替换）
ln -sf /path/to/LuoBoCameraSDK/tools/linux/toolchain/aarch64-rockchip1240-linux-gnu \
  /srv/distcc/toolchain/aarch64-rockchip1240-linux-gnu
```

CMake toolchain 文件中 **只写 NFS 挂载点路径**，例如：

```cmake
set(CMAKE_SYSROOT /srv/distcc/sysroot)
set(CMAKE_C_COMPILER /srv/distcc/toolchain/aarch64-rockchip1240-linux-gnu/bin/aarch64-rockchip1240-linux-gnu-gcc)
```

## 1.3 Server 部署（exports / 防火墙）

### 安装与导出

```bash
sudo apt update
sudo apt install nfs-kernel-server

# 编辑 /etc/exports（网段按实际替换，仅内网）
# /srv/distcc  192.168.1.0/24(rw,sync,no_subtree_check,no_root_squash)
sudo nano /etc/exports
sudo exportfs -ra
sudo exportfs -v
```

**选项说明**：

| 选项 | 含义 | 开发机建议 |
|------|------|------------|
| `rw` | 读写 | 工具链更新时需要；只读工具链可用 `ro` |
| `sync` | 同步写盘 | 默认，避免 client 看见不一致头文件 |
| `no_subtree_check` | 减少 subtree 检查 | 单 export 大目录时常用 |
| `no_root_squash` | root 远程仍映射 root | SDK 安装脚本常需；**勿对公网/export 全网** |

### 防火墙（Ubuntu ufw）

```bash
sudo ufw allow from 192.168.1.0/24 to any port nfs
# 或放行 2049/tcp、2049/udp（视 nfs-utils 配置）
sudo ufw status
```

### 服务检查

```bash
sudo systemctl enable --now nfs-server
rpcinfo -p | grep nfs
showmount -e localhost
```

固定 Server IP（如 9950X `192.168.1.10`），Client `fstab` 写 IP 比 hostname 更稳（DHCP 环境下可选静态 IP 或 DNS）。

## 1.4 Client 挂载（fstab / 同路径验证）

**每台 Client**（12900H 及后续 helper；**不含**仅作 Server 的本机目录访问）：

```bash
sudo apt install nfs-common
sudo mkdir -p /srv/distcc
```

### 1.4.1 开机自动挂载（新节点加入集群）

目标：**机器重启或新设备接入 LAN 后，自动挂上 `/srv/distcc`**，再启动 distccd/iceccd 即可成为 helper，无需每次手动 `mount`。

#### fstab 推荐行（Client）

编辑 `/etc/fstab`，追加 **一行**（Server IP、网段按实际替换）：

```bash
sudo nano /etc/fstab
```

```plain
192.168.1.10:/srv/distcc  /srv/distcc  nfs4  defaults,_netdev,nofail,x-systemd.device-timeout=30  0  0
```

| 选项 | 作用 |
|------|------|
| `nfs4` | NFSv4（与 Server export 一致；若 Server 仅 v3 则改为 `nfs`） |
| `_netdev` | **等网络就绪后再挂载**，避免开机早期挂载失败 |
| `nofail` | Server 暂不可达时 **不阻塞整机启动**（12900H 外出、Server 未开时仍进系统） |
| `x-systemd.device-timeout=30` | 等待 Server 最多 30s（可选；防 DHCP/Wi‑Fi 慢导致超时） |

读写分离场景（工具链只读）可改为：

```plain
192.168.1.10:/srv/distcc  /srv/distcc  nfs4  ro,defaults,_netdev,nofail  0  0
```

应用并验证：

```bash
sudo systemctl daemon-reload
sudo mount -a
mount | grep /srv/distcc
```

**重启验证**（新节点必做）：

```bash
sudo reboot
# 登录后
mount | grep /srv/distcc
test -d /srv/distcc/toolchain && echo "NFS auto-mount OK"
```

#### Server 侧：开机自动 export

Server 除本地目录外，须保证 **nfs-server 随系统启动**（§1.3 已 `enable`，此处强调与 Client 配对）：

```bash
sudo systemctl enable nfs-server
sudo systemctl is-enabled nfs-server   # 应 enabled
```

建议 Server 使用 **固定 LAN IP**（如 9950X `192.168.1.10`），Client `fstab` 写 IP；若用 hostname，须保证 DNS/`/etc/hosts` 在挂载前已解析。

#### systemd 挂载单元（fstab 替代，可选）

与 fstab 等价，便于版本管理或按节点下发：

```ini
# /etc/systemd/system/srv-distcc.mount
[Unit]
Description=NFS distcc toolchain and sysroot
After=network-online.target
Wants=network-online.target
Before=distcc.service icecc.service

[Mount]
What=192.168.1.10:/srv/distcc
Where=/srv/distcc
Type=nfs4
Options=_netdev,nofail

[Install]
WantedBy=remote-fs.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable srv-distcc.mount
sudo systemctl start srv-distcc.mount
```

`Before=distcc.service` 保证 **先挂 NFS、再起 distccd**（若已自定义 distcc unit）。

#### distccd 与挂载顺序

Ubuntu 默认 `distcc.service` 可能在 NFS 之前启动。新节点若遇「distcc 已起但头文件仍缺」，任选其一：

**做法 A — 覆盖 drop-in（推荐）**

```bash
sudo systemctl edit distcc
```

```ini
[Unit]
After=remote-fs.target network-online.target
RequiresMountsFor=/srv/distcc
```

```bash
sudo systemctl daemon-reload
sudo systemctl restart distcc
```

**做法 B — 仅交叉编译集群**：在 `RequiresMountsFor` 路径存在后再手动 `enable distcc`；native 且不用 NFS 路径时可省略。

#### 新设备自动加入节点（最小步骤）

| 步骤 | 操作 |
|------|------|
| 1 | 同 LAN，能 `ping 192.168.1.10` |
| 2 | `apt install nfs-common distcc`（或 icecc） |
| 3 | `/etc/fstab` 添加上述一行 → `mount -a` |
| 4 | §1.4 checklist |
| 5 | 配置 `distccd` / `iceccd`，对端 `DISTCC_HOSTS` 增加本机 IP/slot |
| 6 | `reboot` 验证自动挂载 + distcc |

复制 fstab 模板到新机器即可，**无需在 helper 上再装一份 SDK**。

### 1.4.2 同路径验证 checklist

在 **每一台 helper** 上执行：

```bash
# 1. 挂载成功
test -d /srv/distcc/toolchain && test -d /srv/distcc/sysroot

# 2. 交叉编译器可执行（路径按实际 triple 替换）
test -x /srv/distcc/toolchain/aarch64-rockchip1240-linux-gnu/bin/aarch64-rockchip1240-linux-gnu-g++

# 3. sysroot 头文件可见
ls /srv/distcc/sysroot/usr/include 2>/dev/null || ls /srv/distcc/sysroot/include

# 4. 与 Server 内容一致（抽查）
md5sum /srv/distcc/toolchain/aarch64-rockchip1240-linux-gnu/bin/aarch64-rockchip1240-linux-gnu-gcc
```

Server 与 Client 上同一文件 checksum 应相同。通过后参见 [[B-构建与脚本/01.构建工具/6. 分布式编译|分布式编译]] §1.2 启用 distcc/icecc。

## 1.5 编译 workload 调优与只读策略

| 项 | 建议 | 原因 |
|----|------|------|
| exports `sync` vs `async` | 开发默认 **sync** | 多机同时读 SDK 时避免缓存不一致；追速度可试 `async` 并承担风险 |
| `rsize`/`wsize` | `1048576`（jumbo frame 允许时） | 大 SDK 文件读吞吐 |
| fstab `_netdev` | 推荐 | 网络就绪后再挂载，避免开机挂死 |
| fstab `nofail` | Client 推荐 | Server 未开时仍可进系统；回 LAN 后 `mount -a` |
| `RequiresMountsFor` | distcc drop-in | 保证 NFS 先于 distccd |
| 工具链目录 | 稳定后可改 exports **`ro`** | 防误改；更新时临时改 `rw` 或 Server 本地改 |
| build 目录 | **本地** `~/proj/build` | 见 §1.1；NFS 不适合大量小 `.o` 随机写 |

NFS 只解决 **路径一致**；GCC 主版本、`-march` 等仍见 [[B-构建与脚本/01.构建工具/1. g++|g++]] 与 [[C-Linux生态/03.Linux开发/2. 交叉编译优化选项|交叉编译优化选项]]。

## 1.6 故障排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 开机未挂载 | 无 fstab / Wi‑Fi 晚于 distcc | 查 `mount \| grep distcc`；补 fstab §1.4.1；`systemctl edit distcc` 加 `After=remote-fs.target` |
| `mount.nfs: Connection timed out` | Server 未启或不在 LAN | `nofail` 下系统仍会启动；Server 上线后 `sudo mount /srv/distcc` |
| `mount.nfs: access denied` | exports 网段不匹配、防火墙 | 查 `/etc/exports`、ufw、`showmount -e server` |
| `Stale file handle` | Server 重启或 export 变更 | Client `sudo umount -f /srv/distcc && sudo mount -a` |
| 权限 denied 写 toolchain | `root_squash` | 开发 LAN 用 `no_root_squash` 或 Server 本地改文件 |
| distcc `No such file or directory`（头文件） | NFS 未挂、路径不一致 | **在 helper 上** `ls` 报错路径，非只在主编译机查 |
| 编译极慢 | build 目录误放 NFS | 把 `-o` 输出改到本地 build 树 |
| `rpcinfo: can't contact portmapper` | nfs-server 未启 | `systemctl start nfs-server` |

常用命令：

```bash
# Server
sudo exportfs -v
showmount -a

# Client
nfsstat -m
dmesg | tail | grep -i nfs
```

## 1.7 扩展第三台 helper

新增机器只需：

1. 安装 `nfs-common`，按 **§1.4.1** 写入 `fstab`（或 `srv-distcc.mount`）并 `reboot` 验证自动挂载
2. 跑 §1.4.2 checklist
3. 安装 distccd/iceccd（建议 `RequiresMountsFor=/srv/distcc`），加入 [[B-构建与脚本/01.构建工具/6. 分布式编译|分布式编译]] 集群 `DISTCC_HOSTS` 或 icecc scheduler

无需在 helper 上再复制一份 SDK。
