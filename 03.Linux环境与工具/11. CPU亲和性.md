# 1. CPU亲和性（核心绑定）

CPU 亲和性（Affinity）用于**限制进程或线程在指定 CPU 核心上运行**，是嵌入式 Linux 多核调度与性能优化的重要手段。

## 1.1 单核与多核环境下的行为差异

CPU 亲和性的本质是**限制任务的运行核心范围**。不同核心数的环境下行为差异显著：

| 场景 | 亲和性是否有概念？ | 亲和性是否有实际作用？ |
|------|------------------|----------------------|
| **物理单核 CPU**（如 STM32、老旧 x86） | 概念存在，内核仍维护亲和性掩码 | 完全无效（只有 1 个核心可选） |
| **逻辑单核**（多核 CPU 屏蔽其他核心） | 概念存在 | 无效（与物理单核一致） |
| **多核 CPU** | 概念存在 | 有效，可避免跨核心迁移 |

### 1.1.1 单核环境下亲和性掩码的强制归一化

无论通过 `taskset` 还是 `sched_setaffinity` 设置何种掩码，内核都会自动归一化为**仅包含唯一核心的掩码**：

```bash
# 尝试设置为全核心掩码
taskset -p 0xf 1234
# 查询实际掩码
taskset -p 1234
> pid 1234's current affinity mask: 1  # 内核归一化为 0b0001
```

### 1.1.2 系统调用不会报错的陷阱

单核环境下调用 `sched_setaffinity`、`pthread_setaffinity_np` 等系统调用**不会返回错误**——内核会自动忽略不存在的核心，将掩码调整为可用核心，并返回成功：

```c
#include <stdio.h>
#include <sys/sched.h>

int main() {
    cpu_set_t mask;
    CPU_ZERO(&mask);
    CPU_SET(1, &mask);  // 尝试绑定到核心 1（单核环境中不存在）

    int ret = sched_setaffinity(0, sizeof(mask), &mask);
    if (ret == -1) {
        perror("sched_setaffinity failed");
        return 1;
    } else {
        printf("sched_setaffinity succeeded\n");  // 仍会输出
    }

    // 查询实际掩码
    CPU_ZERO(&mask);
    sched_getaffinity(0, sizeof(mask), &mask);
    printf("Core count: %d\n", CPU_COUNT(&mask));  // 输出 1
    return 0;
}
```

## 1.2 绑定粒度：线程 > 进程 > 整个程序

Linux 中，CPU 亲和性的控制粒度从细到粗为：**线程 → 进程 → 整个程序**。优先级：**线程级绑定 > 进程级绑定**。

| 绑定粒度 | 本质 | 优先级 | 适用场景 |
|----------|------|--------|----------|
| **线程** | 系统支持的最小粒度，给单个线程设置亲和性掩码 | 最高（覆盖进程设置） | 多线程精细化调度：计算线程绑核心 0~1，IO 线程绑核心 2~3；驱动中断线程绑专用核心；实时任务的关键线程绑高优核心 |
| **进程** | 给进程的主线程设置默认掩码，子线程未单独设置则继承 | 中（被线程覆盖） | 单线程程序整体绑定；多线程程序的默认调度范围 |
| **整个程序** | 工程概念，通过脚本/启动参数/统一函数批量设置所有进程/线程 | 最低（本质是批量设置） | 部署阶段统一配置；系统级资源隔离 |

### 1.2.1 进程与线程的亲和性继承关系

**进程是资源的容器，线程是调度的基本单位**。CPU 亲和性是调度属性，本质上是线程的属性。

- 若**线程未单独设置亲和性**，则**继承进程主线程的亲和性掩码**
- 若**线程单独设置了亲和性**，则**使用自身掩码，与进程默认设置无关**

```c
#include <stdio.h>
#include <pthread.h>
#include <sys/sched.h>

void *thread_func(void *arg) {
    cpu_set_t thread_mask, proc_mask;
    CPU_ZERO(&thread_mask);
    pthread_getaffinity_np(pthread_self(), sizeof(thread_mask), &thread_mask);
    CPU_ZERO(&proc_mask);
    sched_getaffinity(0, sizeof(proc_mask), &proc_mask);

    printf("线程核心数 = %d，进程核心数 = %d\n",
           CPU_COUNT(&thread_mask), CPU_COUNT(&proc_mask));
    printf("继承进程设置：%s\n",
           CPU_COUNT(&thread_mask) == CPU_COUNT(&proc_mask) ? "是" : "否");
    pthread_exit(NULL);
}

int main() {
    // 进程级绑定：主线程绑定到核心 0
    cpu_set_t proc_mask;
    CPU_ZERO(&proc_mask);
    CPU_SET(0, &proc_mask);
    sched_setaffinity(0, sizeof(proc_mask), &proc_mask);

    // 线程 1：不单独设置，继承进程设置
    pthread_t t1;
    pthread_create(&t1, NULL, thread_func, NULL);
    pthread_join(t1, NULL);

    // 线程 2：单独设置，绑定到核心 1，覆盖进程设置
    pthread_t t2;
    pthread_create(&t2, NULL, thread_func, NULL);
    cpu_set_t t2_mask;
    CPU_ZERO(&t2_mask);
    CPU_SET(1, &t2_mask);
    pthread_setaffinity_np(t2, sizeof(t2_mask), &t2_mask);
    pthread_join(t2, NULL);

    return 0;
}
```

编译运行（多核环境）：
```bash
gcc affinity_demo.c -o affinity_demo -lpthread
./affinity_demo
```

输出：
```
进程：已绑定到核心 0，核心数 = 1
线程：自身核心数 = 1，进程核心数 = 1    ← 线程 1 继承进程设置
继承进程设置：是
线程：自身核心数 = 1，进程核心数 = 1    ← 线程 2 覆盖进程设置
继承进程设置：否
```

## 1.3 代码兼容性陷阱

### 陷阱 1：依赖单核环境亲和性无效性，移植到多核后性能下降

单核环境下开发的代码未设置亲和性，移植到多核环境后，任务会被内核频繁调度到不同核心，导致跨核心迁移、缓存失效：

```c
#include <stdio.h>
#include <sys/sched.h>
#include <unistd.h>

void set_affinity_if_multicore() {
    int core_num = sysconf(_SC_NPROCESSORS_ONLN);
    if (core_num > 1) {
        cpu_set_t mask;
        CPU_ZERO(&mask);
        CPU_SET(0, &mask);
        sched_setaffinity(0, sizeof(mask), &mask);
        printf("多核环境：绑定到核心 0\n");
    } else {
        printf("单核环境：亲和性无需设置\n");
    }
}
```

### 陷阱 2：硬编码亲和性掩码，限制多核并行性

单核环境开发时为测试硬编码掩码 `0b0001`，移植到多核环境后任务被限制在核心 0，无法利用多核并行性：

```c
void set_multithread_affinity() {
    int core_num = sysconf(_SC_NPROCESSORS_ONLN);
    cpu_set_t mask;
    CPU_ZERO(&mask);

    if (core_num > 1) {
        for (int i = 0; i < core_num; i++)
            CPU_SET(i, &mask);
        printf("多核环境：允许在所有 %d 个核心上运行\n", core_num);
    } else {
        CPU_SET(0, &mask);
        printf("单核环境：绑定到核心 0\n");
    }
    sched_setaffinity(0, sizeof(mask), &mask);
}
```

## 1.4 嵌入式开发实践建议

| 场景 | 推荐绑定粒度 | 原因 |
|------|-------------|------|
| 单线程程序 | 进程级绑定 | 简单方便，效果与线程级一致 |
| 多线程程序 | 线程级绑定 | 灵活高效，可实现精细化调度 |
| 驱动开发 | 线程级绑定 | 中断处理线程必须单独设置亲和性，保证实时性 |
| 实时任务调度 | 线程级绑定 | 将实时控制线程绑定到高优核心，保证响应时间 |
| 部署阶段 | 整个程序级绑定 | 统一配置，实现系统级资源隔离 |

**特殊场景：单核带超线程（Hyper-Threading）的 CPU**

物理单核但逻辑双核，亲和性有实际作用（可限制任务在哪个逻辑核心上运行），应**视为多核环境**处理。

**核心原则**：
- **最小粒度原则**：尽量使用线程级绑定，实现最精细化的调度控制
- **优先级原则**：线程级绑定 > 进程级绑定，线程单独设置后以线程为准
- **工程实用原则**：根据程序实际需求选择最合适的绑定粒度
